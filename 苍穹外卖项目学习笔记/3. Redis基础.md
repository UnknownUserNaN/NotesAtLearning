1. Redis：
	- 概念：Redis是一个基于**内存**（MySQL是基于硬盘的）的key-value（MySQL是基于二维表的）结构数据库。Redis 是互联网技术领域使用最为广泛的**存储中间件**。Redis是用C语言开发的一个开源的高性能键值对(key-value)数据库，官方提供的数据是可以达到100000+的QPS（每秒内查询次数）。它存储的value类型比较丰富，也被称为结构化的NoSql数据库。
	- **主要特点：**
		- 基于内存存储，读写性能高
		- 适合存储热点数据（热点商品、资讯、新闻）
		- 企业应用广泛
2. NoSql（Not Only SQL），不仅仅是SQL，泛指**非关系型数据库**。NoSql数据库并不是要取代关系型数据库，而是关系型数据库的补充。
	- **关系型数据库(RDBMS)：**
		- Mysql ⬅️
		- Oracle
		- DB2
		- SQLServer
	- **非关系型数据库(NoSql)：**
		- Redis ⬅️
		- Mongo db
		- MemCached
3. Redis数据类型（value的数据类型）：
	- 字符串(string)：普通字符串，Redis中最简单的数据类型
	- 哈希(hash)：也叫散列，类似于Java中的HashMap结构（也就是value本身又是一个key-value）
	- 列表(list)：按照插入顺序排序，可以有重复元素，类似于Java中的LinkedList
	- 集合(set)：无序集合，没有重复元素，类似于Java中的HashSet
	- 有序集合(sorted set/zset)：集合中每个元素关联一个分数(score)，根据分数升序排序，没有重复元素
4. Redis常用命令：
	1. 字符串操作命令
		- **SET** key value 设置指定key的值
		- **GET** key 获取指定key的值
		- **SETEX** key seconds value 设置指定key的值，并将 key 的过期时间设为 seconds 秒（EX-Expire）
		- **SETNX** key value 只有在 key 不存在时设置 key 的值（NX-Not Exist）
	2. 哈希操作命令
		Redis hash 是一个string类型的 field 和 value 的映射表，hash特别适合用于存储对象，常用命令：
		- **HSET** key field value 将哈希表 key 中的字段 field 的值设为 value（H-Hash）
		- **HGET** key field 获取存储在哈希表中指定字段的值（H-Hash）
		- **HDEL** key field 删除存储在哈希表中的指定字段（H-Hash）
		- **HKEYS** key 获取哈希表中所有字段（H-Hash）
		- **HVALS** key 获取哈希表中所有值（H-Hash）

	3. 列表操作命令
		Redis 列表是简单的字符串列表，按照插入顺序排序，常用命令：
		- **LPUSH** key value1 [value2] 将一个或多个值插入到列表头部（L-Left）（注意，各个value1的id在高处，value2的id在低处，想象队列移动但id不动就可以了）
		- **LRANGE** key start stop 获取列表指定范围内的元素（L-Left）
		- **RPOP** key 移除并获取列表最后一个元素（R-Right）（注意，也存在**LPOP**和**RPUSH**）
		- **LLEN** key 获取列表长度（L-List）
		- **BRPOP** key1 [key2 ] timeout 移出并获取列表的最后一个元素， 如果列表没有元素会阻塞列表直到等待超时或发现可弹出元素为止

	4. 集合操作命令
		Redis set 是string类型的无序集合。集合成员是唯一的，这就意味着集合中不能出现重复的数据，常用命令：
		- **SADD** key member1 [member2] 向集合添加一个或多个成员（S-Set）
		- **SMEMBERS** key 返回集合中的所有成员（S-Set）
		- **SCARD** key 获取集合的成员数（S-Set，CARD-Cardinal集合基数/元素总数）
		- **SINTER** key1 [key2] 返回给定所有集合的交集（S-Set，INTER-Intersection）
		- **SUNION** key1 [key2] 返回所有给定集合的并集（S-Set）
		- **SREM** key member1 [member2] 移除集合中一个或多个成员（S-Set，REM-Remove）

	5. 有序集合操作命令
		Redis有序集合是string类型元素的集合，且不允许有重复成员。每个元素都会关联一个double类型的分数。常用命令：
		- **ZADD** key score1 member1 [score2 member2] 向有序集合添加一个或多个成员（Z-Z-set）
		- **ZRANGE** key start stop [WITHSCORES] 通过索引区间返回有序集合中指定区间内的成员（Z-Z-set）
		- **ZINCRBY** key increment member 有序集合中对指定成员的分数加上增量 increment（Z-Z-set，INCR-Increase）
		- **ZREM** key member [member ...] 移除有序集合中的一个或多个成员（Z-Z-set，REM-Remove）
		或者，在一定程度上，可以把有序集合想象成以实数（score）索引成员（member）的数据类型

	6. 通用命令
		Redis的通用命令是不分数据类型的，都可以使用的命令：
		- KEYS pattern 查找所有符合给定模式( pattern)的 key
			- KEYS * 查找当前数据库中所有的key
			- KEYS Set* 查找所有名字形如“Set1”“Set2”的key
		- EXISTS key 检查给定 key 是否存在（返回1表示存在，返回0表示不存在）
		- TYPE key 返回 key 所储存的值的类型
		- DEL key 该命令用于在 key 存在是删除 key

			```java
			package com.sky.config;
			
			import lombok.extern.slf4j.Slf4j;
			import org.springframework.context.annotation.Bean;
			import org.springframework.context.annotation.Configuration;
			import org.springframework.data.redis.connection.RedisConnectionFactory;
			import org.springframework.data.redis.core.RedisTemplate;
			import org.springframework.data.redis.serializer.StringRedisSerializer;
			
			@Configuration
			@Slf4j
			public class RedisConfiguration {
			
			    @Bean
			    public RedisTemplate redisTemplate(RedisConnectionFactory redisConnectionFactory){
			        log.info("开始创建redis模板对象...");
			        RedisTemplate redisTemplate = new RedisTemplate();
			        //设置redis的连接工厂对象
			        redisTemplate.setConnectionFactory(redisConnectionFactory);
			        //设置redis key的序列化器
			        redisTemplate.setKeySerializer(new StringRedisSerializer());
			        return redisTemplate;
			    }
			}
			```
			**解释说明：**
			当前配置类不是必须的，因为 Spring Boot 框架会自动装配 RedisTemplate 对象，但是默认的key序列化器为JdkSerializationRedisSerializer，导致我们存到Redis中后的数据和原始数据有差别，故设置为StringRedisSerializer序列化器。

1. Redis最多的应用其实是充当缓存。当服务有许多用户访问时，每一次都去数据库中查询显然来不及，也会让SQL服务器不堪重负。因此通常会在Redis中存储部分查询内容的缓存，当查询请求到来时，先去Redis中做查询，若有则取出对应结果，若无再去数据库中查询，然后再在Redis中存储一个副本。
	通过Redis来缓存数据，减少数据库查询操作。

2. Spring Cache：
	- 介绍：Spring Cache 是一个框架，实现了基于注解的缓存功能，只需要简单地加一个注解，就能实现缓存功能。Spring Cache 提供了一层抽象，底层可以切换不同的缓存实现，例如：
		- EHCache
		- Caffeine
		- Redis(常用)
	- 常用注解：
		
		| **注解**         | **说明**                                                      |
		| -------------- | ----------------------------------------------------------- |
		| @EnableCaching | 开启缓存注解功能，通常加在启动类上                                           |
		| @Cacheable     | 在方法执行前先查询缓存中是否有数据，如果有数据，则直接返回缓存数据；如果没有缓存数据，调用方法并将方法返回值放到缓存中 |
		| @CachePut      | 将方法的返回值放到缓存中                                                |
		| @CacheEvict    | 将一条或多条数据从缓存中删除                                              |
		在spring boot项目中，使用缓存技术只需在项目中导入相关缓存技术的依赖包，并在启动类上使用@EnableCaching开启缓存支持即可。
1. Spring Expression Language（SpEL）：
	Spring Cache 注解（`@Cacheable`, `@CachePut`, `@CacheEvict`）支持使用 **SpEL**（Spring Expression Language）来动态定义缓存 **key**、**条件**（condition）以及 **排除条件**（unless）。
	1. 可用的 SpEL 根对象
		
		|属性|描述|使用示例|
		|---|---|---|
		|`#root.method`|当前方法对象（`java.lang.reflect.Method`）|`#root.method.name`|
		|`#root.target`|当前被调用的目标对象|`#root.target`|
		|`#root.caches`|当前方法对应的缓存数组|`#root.caches[0].name`|
		|`#root.methodName`|当前方法名（字符串）|`#root.methodName`|
		|`#root.targetClass`|当前目标对象的类|`#root.targetClass`|
		|`#root.args`|方法参数数组|`#root.args[0]`|
		|**`#result`**|**方法返回值**（特定场景可用）|`#result.id`（result是关键字）|
		|**参数名**|直接使用方法参数名（需保留参数名）|`#userId`, `#book.id`|
		|`a0`, `p0` …|参数索引方式（无参数名时）|`a0`, `p1`, `a1` 等|
		 **注意**：
		 - `#result` 仅在 **`@Cacheable` 的 `unless`**、**`@CachePut`** 以及 **`@CacheEvict`（`beforeInvocation=false`）** 中可用。
		-  直接在 `condition` 中使用 `#result` **无效**（此时方法尚未执行）。
	1. 常用场景写法
		1. 自定义 Key（`key`）
			```java
			@Cacheable(cacheNames = "books", key = "#isbn")
			public Book findByIsbn(String isbn) { ... }
			// 组合多个参数
			@Cacheable(cacheNames = "books", key = "#author + ':' + #isbn")
			public Book find(String author, String isbn) { ... }
			// 使用参数对象的属性
			@Cacheable(cacheNames = "books", key = "#book.id")
			public Book save(Book book) { ... }
			// 静态方法调用
			@Cacheable(key = "T(com.example.CacheKeys).generateKey(#id, #type)")
			public Data getData(Long id, String type) { ... }
			// 默认 key 生成策略：若不指定 key，则默认使用所有参数（或 SimpleKey）
			```
		2. 条件缓存（`condition`）
			```java
			// 仅当参数满足条件时才缓存
			@Cacheable(cacheNames = "users", condition = "#id > 1000")
			public User getUser(Long id) { ... }
			// 基于参数属性的条件
			@Cacheable(cacheNames = "books", condition = "#book.price < 50")
			public Book process(Book book) { ... }
			```
		3. 排除条件（`unless`）
			```java
			// 返回值满足条件时不缓存
			@Cacheable(cacheNames = "users", unless = "#result == null")
			public User getUser(Long id) { ... }
			@Cacheable(cacheNames = "books", unless = "#result.price > 100")
			public Book findExpensive() { ... }
			```
		4. `@CachePut` 与 `@CacheEvict` 中的特殊用法
			```java
			// @CachePut 总是执行方法，并更新缓存，可使用 #result
			@CachePut(cacheNames = "users", key = "#user.id")
			public User updateUser(User user) {
			    // ... 更新逻辑
			    return user; // #result 即为返回对象
			}
			// @CacheEvict 在方法执行后根据返回值清除缓存
			@CacheEvict(cacheNames = "books", key = "#result.id", beforeInvocation = false)
			public Book deleteBook(Long id) { ... }
			```
