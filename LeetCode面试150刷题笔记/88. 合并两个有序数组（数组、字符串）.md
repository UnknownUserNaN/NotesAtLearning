给你两个按 **非递减顺序** 排列的整数数组 `nums1`\* \*和 `nums2`，另有两个整数 `m` 和 `n` ，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你 **合并** `nums2`\* \*到 `nums1` 中，使合并后的数组同样按 **非递减顺序** 排列。

\*\*注意：\*\*最终，合并后数组不应由函数返回，而是存储在数组 `nums1` 中。为了应对这种情况，`nums1` 的初始长度为 `m + n`，其中前 `m` 个元素表示应合并的元素，后 `n` 个元素为 `0` ，应忽略。`nums2` 的长度为 `n` 。

\*\*进阶：\*\*你可以设计实现一个时间复杂度为 `O(m + n)` 的算法解决此问题吗？

***

最简单的方法自然是将第二个数组中的数据接入到第一个数组中后，然后调用内置排序算法排序。但这样的方法因为使用了排序，复杂度应为`$O((m+n)\log(m+n))$`，这实际上还是有点高的。

```cpp
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
        for(int i=m; i<nums1.size(); i++){
            nums1[i] = nums2[i-m];
        }
        sort(nums1.begin(), nums1.end());
	}
}
```

如何进一步优化上面的算法呢？实际上，显然能想到，该问题实际上存在一个复杂度为`$O(m+n)$`的方法。我们只需要遍历一遍第一个数组的空间，就能完成本文的任务。

由于nums1和nums2是基本有序的，因此只需要不断遍历num1的每个“间隔”，查询nums2中相应的数是否适合插入在这里即可。num1中一共有m个数，因此只需要分别查看m+1个间隔是否符合要求即可。

基于这个思路，于是编写了下面的代码：

```cpp
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
        // 遍历nums1的m+1个间隔
        int j = 0; // nums2的索引，也在记录已插入多少个nums2里的数
        for (int i = 0; i <= m; i++) {
            while (true) {
                if (i == 0 && nums1[i + j] >= nums2[j]) {
                    nums1.pop_back();
                    nums1.insert(nums1.begin() + j, nums2[j]);
                } else if (i == m && nums1[i + j] <= nums2[j]) {
                    nums1.pop_back();
                    nums1.insert(nums1.begin() + i + j, nums2[j]);
                } else if (nums1[i + j - 1] <= nums2[j] &&
                           nums1[i + j] >= nums2[j]) {
                    nums1.pop_back();
                    nums1.insert(nums1.begin() + i + j, nums2[j]);
                } else
                    break;

                j++;
                if (j >= n)
                    break;
            }
        }
    }
};
```

这段代码在本地的IDE中运行正常，并且可以得到正常结果。但是在Leetcode的在线IDE中运行出了错误，ChatGPT认为这是因为上面的程序通过pop和insert方法变更了数组的长度，导致Leetcode的在线编译器在执行更为严格的语法检查时抛出了错误。ChatGPT随后给出了一个无需变更数组长度的程序，它与我的方法最大的不同是，它是从数组末端开始遍历并填充内容的：

```cpp
class Solution {
public:
    void merge(vector<int>& nums1, int m, vector<int>& nums2, int n) {
        int p1 = m - 1;  // nums1 的有效末尾
        int p2 = n - 1;  // nums2 的末尾
        int p = m + n - 1;  // nums1 总空间的末尾
        
        // 从后往前合并
        while (p1 >= 0 && p2 >= 0) {
            if (nums1[p1] > nums2[p2]) {
                nums1[p] = nums1[p1];
                p1--;
            } else {
                nums1[p] = nums2[p2];
                p2--;
            }
            p--;
        }

        // 如果 nums2 还有剩余，继续填充
        while (p2 >= 0) {
            nums1[p] = nums2[p2];
            p2--;
            p--;
        }
    }
};
```

这样的方法显然要聪明得多。这是因为，从nums1的末端开始遍历，可以避开使用insert和pop的操作。从数组末端开始遍历时，遍历总是从数组末尾的0元素开始，这些0元素可以轻松地被覆盖掉，而无需像从数组始端开始遍历时，为了不影响开头的数组，不得不采用insert方法。

本题的**逆向思维**是非常难做到的。正向遍历要么需要使用临时数组，要么需要使用insert操作，而逆向遍历则完全不需要进行上述步骤。
