给你两个字符串 `haystack` 和 `needle` ，请你在 `haystack` 字符串中找出 `needle` 字符串的第一个匹配项的下标（下标从 0 开始）。如果 `needle` 不是 `haystack` 的一部分，则返回  `-1`\*\* \*\*。

***

思路仍然很简单。逐个枚举`haystack`中的每个位置，然后如果该位置与`needle`的首个字符相同，则立刻比较下个字符，直到整个字符串匹配完成。如果不能完全匹配，则直接结束当前匹配状态，枚举下个位置。基于这个思路就可以写出下面的代码：

```cpp
class Solution {
public:
    int strStr(string haystack, string needle) {
        if(haystack.size() < needle.size()) 
            return -1; // 特殊情况
            
        int idx = 0; // 能匹配上的字符索引
        for (int i = 0; i < haystack.size() - needle.size() + 1; i++) { // 枚举haystack中的每个位置
            while (haystack[i + idx] == needle[idx]) {
                idx++;
                if (idx >= needle.size()) {
                    // 已经将所有的字符匹配完成
                    return i;
                } else if (i + idx >= haystack.size()) {
                    // 遍历索引不合理了
                    return -1;
                }
            }
            idx = 0; // 未能匹配上，将匹配上的字符索引归零
        }
        return -1;
    }
};
```

这是一个非常朴素的想法。其时间复杂度为`$O(m(n-m))$`，其中`$m$`为`needle`的长度，`$n$`为`haystack`的长度；空间复杂度为`$O(1)$`。

我以为这已经是最简单的算法了，没想到看了答案，才发现有更简单的算法，能将时间复杂度降到`$O(m+n)$`，不过代价是空间复杂度增加到了`$O(m)$`。这就是大名鼎鼎的**KMP算法**。
