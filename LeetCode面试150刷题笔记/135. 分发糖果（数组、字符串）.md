`n` 个孩子站成一排。给你一个整数数组 `ratings` 表示每个孩子的评分。

你需要按照以下要求，给这些孩子分发糖果：

*   每个孩子至少分配到 `1` 个糖果。
*   相邻两个孩子评分更高的孩子会获得更多的糖果。

请你给每个孩子分发糖果，计算并返回需要准备的 **最少糖果数目** 。

***

将题目的要求用数学的语言表达出来，总是有帮助的。

如果用数组`$p[i]$`表示给每个孩子的糖果分配量，那么要求的第一条意味着对于所有的`$i\in[0,N)$`，都有`$p[i]\geq1$`。

要求第二条要表示成数学语言有点复杂。如果数组`$r[i]$`表示每个孩子的评分，那么对于任意两个相邻的孩子`$i$`和`$i-1$`（`$i\geq1$`），若有`$r[i]<r[i-1]$`，则应有`$p[i]<p[i-1]$`；反之，若`$r[i]>r[i-1]$`，则应有`$p[i]>p[i-1]$`。将这两条综合起来，这相当于说，我们希望对所有的`$i\in[1,N)$`，都有`$\left(r[i]-r[i-1]\right)\left(p[i]-p[i-1]\right)>0$`。

题目的要求，是希望我们可以设法求出数组`$p[i]$`的最小值，也就是`$\displaystyle \min \sum_{i=0}^{N-1}{p[i]}$`。

注意到这里出现了数组差分的形式，这启发我们从差分的角度设计算法：

我们可以先设法构造一个数组满足第二条要求，之后再给数组“整体抬升”到1以上，就能满足要求了。

我们先假设给第一个孩子1颗糖果。然后到第二个孩子，求此时他评分较前者的差分，也就是`$r[1]-r[0]$`。如果有`$r[1]-r[0]>0$`，那么我们应当将糖果数量的差分满足`$p[1]-p[0]\geq1>0$`；反之则应有`$p[1]-p[0]\leq-1<0$`。对于评分差分为0，即`$r[1]-r[0]=0$`的情况，本着分配糖数越少越好的原则，我们同样希望`$p[1]-p[0]\leq-1<0$`。

我们可以试试直接令`$p[1]-p[0]=\pm1$`，根据这个规律依次类推，就可以构造出满足第二条要求的分配结果。当然这样构造之后，会出现某些孩子手里糖果数为负数的情况，这样其实问题不大，我们可以最后将整个数组整体“抬升”到1以上，这样就同时满足了两个要求。

但需要注意的是，上面这种想法分配得到的糖果数不是最少的。试想有这样的一个数组：`[100, 0, 99, 98, 97, ..., 1, 0]`，按照上面的规则，首先我们会得到一个`[1, 0, 1, 0, -1, -2, ..., -97, -98]`这样的结果，然后“整体抬升”后，就会变成`[100, 99, 100, 99, 98, ..., 2, 1]`的分配结果。这显然不是最少的，因为最左侧的分配结果显然可以是`[2, 1, 100, 99, ...]`，这同样也满足条件。

复盘上面的问题，我们发现，问题出在评分数组中存在一个先下降，紧接着上升的结构。我们给出在中心的孩子分配了过多的糖果。实际上，可以断定处在这个结构中心的孩子，他的分配结果完全可以是最小的1，这是最省糖果的一种分配方法。因此我们在遍历时，除了关注与前者的差分之外，还要关注与后者的差分。

那么与上面结构对称的结构呢？也就是评分先上升，再下降的结构呢？根据我们前面的遍历规则，处在这个结构中心的孩子一定会比两侧的孩子多一块糖，这倒是完全合理的，因此这里不需要我们额外地考虑。

但问题在于，上面的思路很难应用到程序的遍历过程中。在遍历到具有“先下降、后上升”的结构之前，我们根本没办法确定其余的位置应该是多少。
