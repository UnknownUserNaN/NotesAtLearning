在一条环路上有 `n` 个加油站，其中第 `i` 个加油站有汽油 `gas[i]`\* \*升。

你有一辆油箱容量无限的的汽车，从第\* *`i`* *个加油站开往第* *`i+1`* *个加油站需要消耗汽油 `cost[i]`* \*升。你从其中的一个加油站出发，开始时油箱为空。

给定两个整数数组 `gas` 和 `cost` ，如果你可以按顺序绕环路行驶一周，则返回出发时加油站的编号，否则返回 `-1` 。如果存在解，则 **保证** 它是 **唯一** 的。

***

我们先看题目中的例子：`gas = [1,2,3,4,5], cost = [3,4,5,1,2]`

来到一个加油站可以加油，离开加油站会耗油，我们当然希望来到一个加油站后可以“净”加油，这时候我们就需要进出一次加油站的油量变化值。这促使我们计算一个新的数组`diff[i]=gas[i]-cost[i]`：：

`gas = [1,2,3,4,5]`\
`cost = [3,4,5,1,2]`\
`diff = [-2, -2, -2, 3, 3]`

如果我们选定好了一个起点，那么沿途的油量变化其实就是对数组diff的部分和。比如要求从0号加油站到3号加油站的油量变化，我们就需要求`-2-2-2=-6`。也就是说，要求从`$i$`号加油站到`$j$`号加油站的油量变化，其实就是需要求数组`diff`从`$i$`到`$j-1$`的这几个数的和。注意到部分和可以用两个前缀和的差得到，这促使我们求出数组`diff`的前缀和`cumsum`：

`gas = [1,2,3,4,5]`\
`cost = [3,4,5,1,2]`\
`diff = [-2, -2, -2, 3, 3]`\
`cumsum = [-2, -4, -6, -3, 0]`

本题中涉及到了循环求和，这里为了实现方便，我们把`diff`和`cumsum`各自多重复一遍：

`diff   = [-2, -2, -2,  3, 3,| -2, -2, -2,  3, 3]`\
`cumsum = [-2, -4, -6, -3, 0,| -2, -4, -6, -3, 0]`

本题实际上是要找到一个索引`$i$`，使得在上面的重复后的数组中，对所有的`$1\leq j\leq n$`（`$n$`为加油站个数），都有`$cumsum[i+j]-cumsum[i]>0$`。

那么不难注意到，如果我们直接将上面的`$i$`选为上面`cumsum`的最小值点，那么显然对后面的`$n$`个值，都满足上面的不等式。这样，我们找到`$i$`之后，便可以选择在`$i+1$`处开始。

当然上面所有的推理全部基于可以绕环路行驶一周的假设。实际上，如果数组`cumsum`的末尾值不是0，那么根据比较偏“能量守恒”的思路，获取的总汽油量大于消耗的总汽油量，所以肯定存在一个解使得汽车可以跑完全程。

基于上面的思路，就可以写出接下来的程序：

```cpp
class Solution {
public:
    int canCompleteCircuit(vector<int>& gas, vector<int>& cost) {
        // 求数组的前缀和，并设法找到其最小值点
        int diff = 0, cumsum = 0;
        int min_cumsum = INT_MAX, index = 0;
        for (int i = 0; i < gas.size(); i++) {
            diff = gas[i] - cost[i];
            cumsum += diff;
            if (cumsum < min_cumsum) {
                min_cumsum = cumsum;
                index = i;
            }
        }

        // 遍历完数组，确定其是否有解
        if (cumsum >= 0)
            return (index + 1)%gas.size(); // 有解，返回最小值点后一位
        else
            return -1; // 无解，返回-1
    }
};
```

一次直接AC。这个故事告诉我们，**当数组存储了一个连续状态的时候，设法求这个数组的部分和和差分是非常有必要的，因为这实际上对应着积分和微分，这恰好是数学上分析的手段**。

官方题解的思路比较复杂，这里就不展开了。评论区有一位大哥的思路与我的想法很类似，这里把他的想法也贴上来：

> 这个算法只从第一个加油站遍历一圈即可，对于中间油量可以出现负数的情况，可以想象出发前向别人借了足够的油量，走完一圈只要能还上就行了。这样只需要一层循环就可以。

```cpp
class Solution {
    public int canCompleteCircuit(int[] gas, int[] cost) {
        // 最低可能的油量，可能为负，表示出发前借的油
        int minBalance = 0;  
       // 达到最低油量的加油站索引，在此加油站出发
       // 油量不会低于最低值，所以此加油站即为题目所求
        int minIndex = 0;  
        int n = gas.length;
        int balance = 0;
        for(int i = 0; i < gas.length; i++) {
            // 计算后的balance为到达下一加油站 i + 1的油量，如果创了新低则记录之
            balance = balance + gas[i] - cost[i];
            if(balance < minBalance) {
                minBalance = balance;
                minIndex = i + 1;
            }
            
        }
        // 从第一个加油站出发，如果油量恢复到出发前以上，代表可能跑完一圈
        // 在中间出现负值也没有关系，可以想象出发前向别人借了一些油，最后
        // 能还上就可以了
        return balance >= 0 ? minIndex: -1;
    }
}
```
