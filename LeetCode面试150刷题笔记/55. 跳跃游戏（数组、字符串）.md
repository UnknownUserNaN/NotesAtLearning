给你一个非负整数数组 `nums` ，你最初位于数组的 **第一个下标** 。数组中的每个元素代表你在该位置可以跳跃的最大长度。

判断你是否能够到达最后一个下标，如果可以，返回 `true` ；否则，返回 `false` 。

***

可以递归地考虑这个问题。首先，如果最后一步可以到达末尾，那么这个位置上限制的最大跳跃长度一定不小于这一步的长度。这样，我们只需要从后往前遍历数组，如果这一位置上的数值不小于距离末尾的距离，那么这个位置就是一个备选项。那么接下来我们就只需要考虑，如何到达这一个“备选项”的位置，那么接下来就只需要递归地重复上面的步骤就可以了。

考虑递归的实际实现方式，上面的递归过程可以改写成栈或者树的形式。这实际上是一个深度搜索的过程。

```cpp
class Solution {
public:
    bool sliceCheck(vector<int>& nums, int end) {
        // 特殊情况
        if (end == 0)
            return true;

        // 主循环
        for (int i = end - 1; i >= 0; i--) {
            if (nums[i] >= end - i) {
                // 这一位置到终点的距离满足要求，于是递归地查看有没有到这一位置的方法
                if (sliceCheck(nums, i) == true)
                    return true;
            }
        }
        return false;
    }

    bool canJump(vector<int>& nums) {
        return sliceCheck(nums, nums.size() - 1);
    }
};
```

上面的程序在运行一些比较短的数组时还是可以的，但是在运行一些稍微长一些的数组时，运行时间就会成倍地增加。因此上面的递归思路只是写起来的时候比较容易，实际执行时往往不是最优的方法。因此，上面的思路还要进一步优化。

注意我们上面思路中的其中一点，我们首先是寻找一步能到达终点的点，如果该点的数值大于到终点的步数，则认为该点是可行的。结合上面递归的思路，我们实际上可以这么做：不断从后向前遍历数组，如果发现有一点可以到达终点，则将此点加入备选项的数组。此后的遍历中，除了检查能否到达终点，还检查能否到达备选项的位置。如此循环，直到遍历到数组的末尾。

这种方法就是上面递归方法的改写版本，但实际运行时显然不够”经济“。我们需要注意到这样的一个事实，如果我们留意上面的”备选项“数组，在上面由后向前遍历的过程中，加入数组的索引值实际上是越来越小的。当我们依次判断遍历到的点是否能到达终点和备选点时，实际上只有最小的备选点起了作用，因为如果遍历过程中的点能到达最小的备选点，那么这个点也一定能到达终点。由于每个位置实际上只规定了跳跃的**最大距离**，而非距离，这使得整个由后往前的过程其实并不像一个树形结构，也就是说遍历的过程中**不会走到“死路”**。

注意到这一点，我们就可以改写上面的算法了：

```cpp
class Solution {
public:
    bool canJump(vector<int>& nums) {
        int min_stop = nums.size() - 1;
        for (int i = nums.size() - 2; i >= 0; i--) {
            if (nums[i] >= min_stop - i) {
                min_stop = i;
            }
        }
        if (min_stop == 0)
            return true;
        else
            return false;
    }
};
```

这样就直接AC了。

后来看了题解，才发现实际上这道题也可以正着遍历：

> **解题思路：**
>
> *   如果某一个作为 起跳点 的格子可以跳跃的距离是 3，那么表示后面 3 个格子都可以作为 起跳点
>
> <!---->
>
> *   可以对每一个能作为 起跳点 的格子都尝试跳一次，把 能跳到最远的距离 不断更新
>
> <!---->
>
> *   如果可以一直跳到最后，就成功了

代码也很简洁：

```cpp
class Solution {
public:
    bool canJump(vector<int>& nums) {
        int k = 0;
        for (int i = 0; i < nums.size(); i++) {
            if (i > k) return false;
            k = max(k, i + nums[i]);
        }
        return true;
    }
};
```
