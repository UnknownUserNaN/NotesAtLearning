给你一个整数数组 `citations` ，其中 `citations[i]` 表示研究者的第 `i` 篇论文被引用的次数。计算并返回该研究者的 **`h`**\*\*\* \*\*\***指数**。

根据维基百科上 [h 指数的定义](https://baike.baidu.com/item/h-index/3991452?fr=aladdin)：`h` 代表“高引用次数” ，一名科研人员的 `h`\*\* 指数 \*\*是指他（她）至少发表了 `h` 篇论文，并且 **至少 \******有 `h` 篇论文被引用次数大于等于 `h` 。如果 `h`***\*\* \*有多种可能的值，\*\*`h` 指数 \*\*是其中最大的那个。

***

先对输入的数组进行排序会比较简单。对于题目给出的例子，`citations = [3,0,6,1,5]`，排序之后变成`[0,1,3,5,6]`。我们可以考虑从这个数组的起始位置开始遍历，这时检查的是有5篇论文的引用次数大于5的情况。由于整个数组是已经排序好的，所以我们只需要检查当前位置的引用是否满足当前遍历值即可。如果该情况成立，则直接说明其H指数为5；如果该检查不成立，则继续向后遍历，直至遍历到最后。

根据上面的思路可以写出如下的程序：

```cpp
class Solution {
public:
    int hIndex(vector<int>& citations) {
        // 先对数组进行排序
        sort(citations.begin(), citations.end());

        // 再依次遍历数组
        int h = citations.size();
        for (int i = 0; i < citations.size(); i++) {
            if (citations[i] >= h)
                break;
            else
                h--;
        }
        return h;
    }
};
```

这一次提交直接AC了，并且整个程序的耗时也居于前列。但是这个程序的时间复杂度是`$O(n\log n)$`，空间复杂度为`$O(\log n)$`。根据Leetcode的特点，这个题目应该还有`$O(n)$`时间的做法。

上面的算法也可以改用哈希表统计频次、然后处理频次来做。这样做虽然避免了排序，但难免需要耗费更多的额外空间：

```cpp
class Solution {
public:
    int hIndex(vector<int>& citations) {
        map<int, int> freq; // 统计论文引用数频次的哈希表

        // 统计频次
        for (int cite : citations) {
            if (cite >= citations.size())
                freq[citations.size()]++; // 高于论文总数的论文引用数据全部整合
            else
                freq[cite]++;
        }

        // 根据哈希表中的频次，由小到大遍历频次
        int h = citations.size();
        int sum_freq = 0;
        for (; h >= 0; h--) {
            sum_freq += freq[h];
            if (sum_freq >= h)
                break;
        }

        return h;
    }
};
```

该算法的复杂度可判定为`$O(n)$`，但因为需要遍历两次数组（相当于`$O(2n)$`），因此运行时间较优化良好的C++内置排序算法反而更高。而时间复杂度降低的代价是，空间复杂度从前面的`$O(\log n)$`上升到`$O(n)$`。

做到这里，我以为这已经是最优的算法了，但是看了官方题解，才发现还有只需要使用常数空间的算法。这个方法就是实际上就是经典的二分法，官方题解中称之为“二分搜索”法。

首先不难注意到，如果输入数组的长度是`$n$`，那么显然要求的H指数的取值区间为`$[0,n]$`。这时我们便可以通过二分法猜测这个解。首先猜H指数为`$\displaystyle \lfloor\frac{n}{2}\rfloor$`，然后遍历整个数组（这需要`$O(n)$`的时间），查看有多少个引用数值大于`$\displaystyle\lfloor\frac{n}{2}\rfloor$`。如果满足条件的引用数值的数量大于`$\displaystyle \lfloor\frac{n}{2}\rfloor$`，那么说明满足H指数的区间应当进一步地在区间`$\left[\displaystyle\lfloor\frac{n}{2}\rfloor,n\right]$`中搜索，反之，则应在`$\left[0,\displaystyle\lfloor\frac{n}{2}\rfloor\right]$`的区间中搜索。然后继续猜测，然后继续遍历整个数组，重复上述过程，搜索区间收敛到一个整数为止。整个二分法的搜索过程需要`$O(\log n)$`的时间，因此上述整个过程的时间复杂度为`$O(n\log n)$`，高于之前的哈希表方法，但是却只需要使用常数级的空间。

下面是官方题解采用二分法编写的程序：

```cpp
class Solution {
public:
    int hIndex(vector<int>& citations) {
        int left=0,right=citations.size();
        int mid=0,cnt=0;
        while(left<right){
            // +1 防止死循环
            mid=(left+right+1)>>1;
            cnt=0;
            for(int i=0;i<citations.size();i++){
                if(citations[i]>=mid){
                    cnt++;
                }
            }
            if(cnt>=mid){
                // 要找的答案在 [mid,right] 区间内
                left=mid;
            }else{
                // 要找的答案在 [0,mid) 区间内
                right=mid-1;
            }
        }
        return left;
    }
};
```

值得一提的是，我们需要明白为什么此题可以采用二分法。回想课本上采用二分法寻找函数零点的过程，其中最重要的一个条件是，函数在此区间单调，且零点必然存在在该区间。“函数在该区间单调”，意味着零点必然有一侧函数值均大于0，有一侧的函数值必然小于0，而这个“大于0”和“小于0”恰恰可以指引我们向着零点的方向前进。

回到这个问题，H指数指的是\*\*“有h篇论文的引用次数至少为h”的值的最大值\*\*。并且，小于H指数的值亦满足\*\*“有h篇论文的引用次数至少为h”**的性质**，**而大于H指数的值则不满足**“有h篇论文的引用次数至少为h”**的性质，在目标答案附近两种不同的表现，恰好对应着前面函数零点搜索的例子中“一侧大于0”“一侧小于0”的特点，我们可以通过检查当前数组满不满足**“有h篇论文的引用次数至少为h”\*\*的性质，来指引我们逐步缩小搜索区间。

这个题目启发了我们：**如果要在某个区间搜索某个值，且在该值两侧的区间具有不同的性质，我们便可以采用二分法来查找这个值。**
