给定整数数组 nums 和整数 k，请返回数组中第 k 个最大的元素。

请注意，你需要找的是数组排序后的第 k 个最大的元素，而不是第 k 个不同的元素。

你必须设计并实现时间复杂度为 O(n) 的算法解决此问题。

---
这个问题最好的办法是使用“堆”这个数据结构实现。
首先，堆在本质上是一棵完全二叉树，并且其每个结点满足“堆序性”，即每个结点的值都小于（或大于）其左右孩子。若都小于其左右孩子，则这个堆称为“小根堆”（根是较小者）；若都大于其左右孩子，则这个堆称为”大根堆“。
基于堆序性，我们不难得到一条堆的重要性质：小（大）根堆的根结点的值一定是整个堆中的最小（大）值。这个性质有什么作用呢？因为堆本质上是一棵完全二叉树，而完全二叉树可以按层次遍历形成一个连续数组，即完全二叉树与连续数组一一对应。因此，如果我们可以将一个数组处理成一个满足堆序性的堆，那么这个数组的第一个元素一定是这个数组的最大值（或者最小值）。这就是所谓“优先队列”的实现原理（C++中的容器是`priority_queue`），它的底层就是通过堆来实现的。
有了这条性质，我们不难猜出这道题的基本思路：设法把题中的数组处理成大根堆，这样数组的第一个元素一定是数组中的最大值。然后我们将这个最大值“弹出”数组，继续将数组处理成大根堆，这样数组的第一个元素就变成了原数组的第二大的值。随后重复上述操作k次，查看数组中的第一个元素，便可以得到数组中的第k个最大元素。
那么问题来了，该如何将一个任意的数组处理成满足堆序性的堆呢？首先，我们需要知道一个重要性质，如果一个连续数组可以被视为完全二叉树，那么数组中下标为$i$的结点，其左孩子的下标一定为$2i+1$，右孩子的下标一定为$2i+2$。因此，我们可以遍历数组中的每个元素，也就是遍历树中的每一个结点，检查它与它的两个孩子之间的大小关系，除非它的两个孩子的下标已经超出了数组范围。也就是我们要遍历所有下标满足$2i+1<L$（其中$L$为数组的长度）的元素，即所有满足$\displaystyle i>\left\lfloor \frac{L-1}{2}\right\rfloor=\frac{L}{2}-1$的元素。如果这些元素中有一者不满足堆序性，我们便将该结点与其两个孩子中的较大者或较小者进行交换，究竟是较小者还是较大者要看是小根堆还是大根堆，总之要让当前双亲-两个孩子之间满足堆序性的关系。
基于上面的思路，我们便可以手动实现一个大根堆，并完成本题的要求：
```cpp
class Solution {
public:
    void maxHeapify(vector<int>& a, int i, int heapSize) {
        // 定义大根堆，i为要进行下滤操作的元素（非叶子结点）
        // 下面的l和r分别定义了左孩子、右孩子
        // largest为当前根节点、左孩子和右孩子中的最大者
        int l = i * 2 + 1, r = i * 2 + 2, largest = i;  // 初始时默认根节点最大
        if (l < heapSize && a[l] > a[largest]) { // 左孩子在堆中（存在左孩子），且大于根节点
            largest = l; // 将较大者更新为根节点
        } 
        if (r < heapSize && a[r] > a[largest]) { // 右孩子在堆中（存在右孩子），且大于根节点
            largest = r; // 将较大者更新为根节点
        }
        if (largest != i) { // 如果三者中最大的不是根节点，则说明不满足堆序性，要进行下滤
            swap(a[i], a[largest]); // 将要操作的非叶子结点（较小者）与要操作的节点进行交换，完成下滤
            maxHeapify(a, largest, heapSize); // 对交换后的结点递归地执行下滤操作
        }
    }

    void buildMaxHeap(vector<int>& a, int heapSize) {
        for (int i = heapSize / 2 - 1; i >= 0; --i) { // 注意遍历顺序，是从数组较大者向前遍历
            maxHeapify(a, i, heapSize); // 对每个非叶子结点进行下滤操作，这样可以把数组中的最大值移到根节点
        } 
    }

    int findKthLargest(vector<int>& nums, int k) {
        int heapSize = nums.size();
        buildMaxHeap(nums, heapSize); // 从当前的数组中建立堆（建立大根堆，这样数组末尾的数自动）
        // 建立完大根堆之后，其根节点必为数组中的最大值
        for (int i = nums.size() - 1; i >= nums.size() - k + 1; --i) { // 该循环只需要执行k次，也就是只需要弹出k个根节点就可以了
            swap(nums[0], nums[i]); // 将当前数组中的最大值移到末端“保存”起来
            --heapSize; // 减小堆的大小，相当于将刚才的最大值弹出堆
            maxHeapify(nums, 0, heapSize); // 对根节点进行下滤操作，使得最大值重新被交换到根节点
        }
        return nums[0];
    }
};
```
整个程序中有几点需要特别注意：
1. 遍历数组使之满足堆序性时，应当是从元素较大的一侧开始遍历，这样才能让数组中的最大值“上浮”到二叉树的根部。
2. 为了堆序性交换一个元素时，应当继续“跟踪”被交换的这个元素，直至其不再被交换为止。这样才能保证数组中的最大值能“上浮”到二叉树的根部。
3. 将堆顶元素“弹出”堆，实际上只需要令其与数组的最后一位进行交换，随后再也不处理这一位即可。这样可以利用数组的空间“存储”最大值，免去使用额外空间的麻烦。
4. 实际上，如果我们将上面的k设定为数组的整个长度，我们便可以将整个数组排序成一个从小到大排列的数组。这就是经典的“堆排序”。想要让小数在前，就要建立大根堆，然后把最大的数“抛”到数组后面去。由于堆排序的过程中只对数组中的元素进行交换，因此堆排序的一个最大特点是它能实现数组的**原地排序**。
堆排序方法的时间复杂度为$O(n\log n)$（本题中是$O(n+k\log n)$），空间复杂度为$O(\log n)$（仅使用递归的栈空间）。
当然，[官方题解](https://leetcode.cn/problems/kth-largest-element-in-an-array/solutions/307351/shu-zu-zhong-de-di-kge-zui-da-yuan-su-by-leetcod-2/)也提到，本题也可以用快速排序完成。标准的快排过程是这样的：
> 首先我们来回顾一下快速排序，这是一个典型的分治算法。我们对数组 a[l⋯r] 做快速排序的过程是（参考《算法导论》）：
> 分解： 将数组 a[l⋯r] 「划分」成两个子数组 a[l⋯q−1]、a[q+1⋯r]，使得 a[l⋯q−1] 中的每个元素小于等于 a[q]，且 a[q] 小于等于 a[q+1⋯r] 中的每个元素。其中，计算下标 q 也是「划分」过程的一部分。
> 解决： 通过递归调用快速排序，对子数组 a[l⋯q−1] 和 a[q+1⋯r] 进行排序。
> 合并： 因为子数组都是原址排序的，所以不需要进行合并操作，a[l⋯r] 已经有序。
> 上文中提到的 「划分」 过程是：从子数组 a[l⋯r] 中选择任意一个元素 x 作为主元，调整子数组的元素使得左边的元素都小于等于它，右边的元素都大于等于它， x 的最终位置就是 q。

快排不止一种写法，分区方式不同，写出的快排程序也不一样。一种是Lomuto分区，它通常选取数组末尾元素作为基准，小于基准的值全部前置，大于基准的值会被自然前置。它可以保证分区后的基准值位置一定是正确的（因为它最终强制把基准值放到了两个分区中间的位置），因此随后对子数组进行快排的方法也可以省去一个元素（即基准值）的处理：
```cpp
// 快速排序函数（Lomuto分区实现）
void quickSort(vector<int>& nums, int l, int r) {
	// 1. 递归终止条件：当子数组长度为1或空时，无需排序
    if (l >= r) return;

	// 2. 初始化分区参数：
    // - i 初始在左边界左侧（l-1）
    // - j 初始在右边界右侧（r+1）
    // - 基准值 x 选中间元素（避免极端情况如全排序数组导致最坏时间复杂度）
    int x = nums[r];  // 基准值为右侧元素

    // Lomuto 分区循环：
    int i = l - 1;    // 指向小于区的末尾
    for (int j = l; j < r; j++) {
        if (nums[j] < x) {        // 只处理严格小于基准的情况
            i++;
            swap(nums[i], nums[j]); 
        }
    }
    swap(nums[i + 1], nums[r]);    // 将基准放回正确位置
    int pivotIndex = i + 1; 

    // 递归排序左右子数组（注意边界的差异）
    quickSort(nums, l, i - 1);  // 左半部分不包含基准
    quickSort(nums, i + 2, r);  // 右半部分不包含基准
}
```
然后是效率更高的Hoare分区方法，基本思路是在左侧区间找到大于等于基准的不合理的值，在右侧区间找到小于等于基准的不合理的值，然后将两者进行交换。随后两个指针不断向中心移动，直至合拢。效率更高的代价是，基准值的最终位置不一定是正确的，只能保证两个分区的大小关系是正确的。
```cpp
// 快速排序函数（Hoare 分区方案实现）
void quickSort(vector<int>& nums, int l, int r) {
    // 1. 递归终止条件：当子数组长度为1或空时，无需排序
    if (l >= r) return;

    // 2. 初始化分区参数：
    // - i 初始在左边界左侧（l-1）
    // - j 初始在右边界右侧（r+1）
    // - 基准值 x 选中间元素（避免极端情况如全排序数组导致最坏时间复杂度）
    int i = l - 1, j = r + 1;
    int x = nums[(l + r) >> 1]; // 位运算代替 (l + r) / 2，等价且更高效（快排的效率与基准的选择高度相关）

    // 3. Hoare 分区循环：双指针从两端向中间移动
    while (i < j) {
        // 3.1 移动左指针 i：跳过所有小于 x 的元素，直到找到 >=x 的元素
        do i++; while (nums[i] < x);

        // 3.2 移动右指针 j：跳过所有大于 x 的元素，直到找到 <=x 的元素
        do j--; while (nums[j] > x);

        // 3.3 如果指针未交错，交换左右指针的元素（确保左边 <=x，右边 >=x）
        if (i < j) swap(nums[i], nums[j]);
    }

    // 4. 递归排序左右子数组：
    // - 分区点为 j（因为当 i >= j 时，j 是左半部分的最右端）
    // - 左子数组：[l, j]（所有元素 <=x）
    // - 右子数组：[j+1, r]（所有元素 >=x）
    quickSort(nums, l, j);
    quickSort(nums, j + 1, r);
}
```
上面的过程中需要注意第3步。这种双指针法能够快速实现基准两侧的“基本有序”。
需要注意的是，经历完第3步之后，我们的基准值被移动到了下标为$j$的位置，只有基准值的位置是准确的，随后实际上无需再移动基准值的位置。随后为了实现整个数组的有序，实际上还要再对区间$[l,j]$和$[j+1, r]$递归地应用快排。但我们基于快排完成本题的话，其实只需要关注第k个最大元素所在的那个子区间即可。基于此可以写出下面的程序：
```cpp
class Solution {
public:
    int quickselect(vector<int> &nums, int l, int r, int k) {
        if (l == r)
            return nums[k];
        int partition = nums[l], i = l - 1, j = r + 1;
        while (i < j) {
            do i++; while (nums[i] < partition);
            do j--; while (nums[j] > partition);
            if (i < j)
                swap(nums[i], nums[j]);
        }
        if (k <= j)return quickselect(nums, l, j, k);
        else return quickselect(nums, j + 1, r, k);
    }

    int findKthLargest(vector<int> &nums, int k) {
        int n = nums.size();
        return quickselect(nums, 0, n - 1, n - k);
    }
};
```
官方题解称这个算法可以达到$O(n)$的时间复杂度，我暂且蒙古。空间复杂度达到$O(\log n)$已经是没有问题的。
