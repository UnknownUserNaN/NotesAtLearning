给你一个 无重复元素 的整数数组 candidates 和一个目标整数 target ，找出 candidates 中可以使数字和为目标数 target 的 所有 不同组合 ，并以列表形式返回。你可以按 任意顺序 返回这些组合。

candidates 中的 同一个 数字可以 无限制重复被选取 。如果至少一个数字的被选数量不同，则两种组合是不同的。 

对于给定的输入，保证和为 target 的不同组合数少于 150 个。

提示：candidates 的所有元素 互不相同

---
首先需要注意的一点是，本题中的同一个数字可以无限制地被重复选取，这是本题与此前的回溯问题略微不同的一点。
但是这部分不同也并不会造成太大的困难。首先先考虑本问题的简单版本，每个数字都只能被选取一次。这时沿用之前回溯法的思路，构造一个深度与数组`candidates`相同的二叉树，每个深度为`i`的结点决定是否将整数`candidates[i]`加入到求和中。如果发现加入该整数后超过了目标，则改为尝试下一个结点。为了能提升算法的运行效率，我们最好以从大到小的顺序遍历`candidates`。
现在加上每个数字可以使用无数次的条件，不难发现，这个条件实际上在我们每次决定加入`candidates[i]`时多了一个是否要继续在本结点“停留”的判断。本题与此前的回溯题目的最大区别就在这个判断。这个判断使得我们会面临一个问题，我们判断答案是有顺序的，因此得出的答案实际上是各种“序列”，但是答案只考虑“组合”。也就是说，如果我们沿用前面的思路，会产生类似于`[2, 2, 3]`和`[2, 3, 2]`，甚至还有`[3, 2, 2]`，我们的方法会给出这三种结果，但实际上这三种结果只可以被视为一种——我们会给出多余的答案。
那么如何规避这一种情况呢？方法就是改为从小到大的顺序遍历`candidates`，这样程序就只会给出`[2, 2, 3]`这一种答案。考虑我们“构造”的二叉树，从小到大的遍历顺序保证结果数组中的数值一定是非降序排列的（深度遍历在到达叶子节点之前不会返回其父节点），因此类似`[2, 3, 2]`甚至`[3, 2, 2]`的结果就可以得到规避。
按照回溯法的一般模板，很容易写出如下的程序：
```cpp
class Solution {
public:
    void backtracking(vector<vector<int>>& ret, vector<int>& can,
                      vector<int>& com, int cur, int sum, int target) {
        if (sum == target) {
            ret.push_back(com);
            return;
        }

        for (int i = cur; i < can.size(); i++) {
            // 以从小到大的顺序遍历
            if (sum + can[i] > target)
                continue;

            com.push_back(can[i]);
            backtracking(ret, can, com, i, sum + can[i], target);
            com.pop_back();
        }
    }

    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        vector<vector<int>> ret;
        vector<int> com;
        backtracking(ret, candidates, com, 0, 0, target);
        return ret;
    }
};
```
[官方题解](https://leetcode.cn/problems/combination-sum/solutions/406516/zu-he-zong-he-by-leetcode-solution/)的写法略有不同，但是思路是一致的：
```cpp
class Solution {
public:
    void dfs(vector<int>& candidates, int target, vector<vector<int>>& ans, vector<int>& combine, int idx) {
        if (idx == candidates.size()) {
            return;
        }
        if (target == 0) {
            ans.emplace_back(combine);
            return;
        }
        // 直接跳过
        dfs(candidates, target, ans, combine, idx + 1);
        // 选择当前数
        if (target - candidates[idx] >= 0) {
            combine.emplace_back(candidates[idx]);
            dfs(candidates, target - candidates[idx], ans, combine, idx);
            combine.pop_back();
        }
    }

    vector<vector<int>> combinationSum(vector<int>& candidates, int target) {
        vector<vector<int>> ans;
        vector<int> combine;
        dfs(candidates, target, ans, combine, 0);
        return ans;
    }
};
```
该算法的空间复杂度容易分析，因为其完全取决于数组`com`的最大长度，大致为$O(target)$（也就是备选数组`candidates`中存在数字1时的情况）。但是该算法的时间复杂度有些难以分析，因为上面回溯算法所对应的二叉树结点个数是较为难以计算的（因为我们实际上对一棵二叉树进行了“剪枝”），我们只能给出一个非常松的上界估计，为$O(n\cdot 2^n)$，其中$n$为`candidates`的长度。这个上界实际上对应了我们不做“剪枝”，二叉树为“完全二叉树”的情况。
值得一提的是，上面的算法还有一定的优化空间，因为上面的剪枝条件实际上是非常“松”的，我们还是会对二叉树中的某些结点做无效的“遍历”。但这一块就不是这个算法题所要求的难度了。
