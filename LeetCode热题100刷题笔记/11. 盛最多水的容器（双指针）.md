给定一个长度为 `n` 的整数数组 `height` 。有 `n` 条垂线，第 `i` 条线的两个端点是 `(i, 0)` 和 `(i, height[i])` 。

找出其中的两条线，使得它们与 `x` 轴共同构成的容器可以容纳最多的水。

返回容器可以储存的最大水量。

\*\*说明：\*\*你不能倾斜容器。

***

一开始使用了遍历性质的解法，`$O(n^2)$`复杂度，结果毫无意外地超时了：

```cpp
class Solution {
public:
    int calcArea(vector<int> &height, int left, int right){
        int min_height = min(height[left], height[right]);
        int area = min_height*(right-left);
        return area;
    }

    int maxArea(vector<int>& height) {
        int max_area = 0;
        for(int i=0; i<height.size()-1; i++){
            for(int j=i+1; j<height.size(); j++){
                int area = calcArea(height, i, j);
                if(area > max_area){
                    max_area = area;
                }
            }
        }
        return max_area;
    }
};
```

后来在官方的提示下才想到复杂度更低的做法。即**一开始设定好左指针和右指针，分别位于数组的最左侧和最右侧，然后每次循环中不断移动对应高度更低一侧的指针，直至两指针相遇**。遍历过程中不断计算此时的水量，并始终保留更低者。

```cpp
class Solution {
public:
    int calcArea(vector<int> &height, int left, int right){
        int min_height = min(height[left], height[right]);
        int area = min_height*(right-left);
        return area;
    }

    int maxArea(vector<int>& height) {
        int left = 0, right = height.size()-1;
        int area = calcArea(height, left, right);
        int max_area = area;

        while(right > left){
            if(height[right] > height[left]){
                // 右侧高于左侧，则移动左侧指针
                left++;
            }
            else{
                // 反之，移动右侧指针
                right--;
            }

            // 计算此时Area
            area = calcArea(height, left, right);
            if(area > max_area){
                max_area = area;
            }
        }

        return max_area;
    }
};
```

官方的写法与上面的写法大同小异。但是为什么这种做法是正确的？官方给出了一段较为数学的证明：

现在假设左指针`$a$`和右指针`$b$`分别在数组的两侧。

不失一般性地，我们可以假设`$h(a)<h(b)$`，此时的水量可以计算为：`$Area=min[h(a), h(b)]\cdot(b-a)=(b-a)h(a)$`。

\*\*如果此时我们固定住左指针，那么此种情况下，无论右指针在哪里，水量都不会再比上面的`$Area$` 更大了。\*\*这里不妨假设右指针向左移动了`$n$`位，此时的水量为：`$Area^\prime=min[h(a),h(b-n)]\cdot(b-a-n)=(b-a-n)h(a)<(b-a)h(a)=Area$`。

因此，此时**为了寻找到更大的`$Area$`，我们移动右指针是没有意义的，因此只能移动左指针**。

移动完左指针之后，我们就又会遇到和上面一样的问题，此时只能仿照上面的策略，继续移动高度更低的一侧的指针。自此不断循环往复，直到两指针相遇。

当然值得注意的是，**上面的搜索策略并不是最优的，因为我们不能保证每一次移动高度更低的一侧的指针都能获取到更大的`$Area$`**。但是，这样的算法比起最开始的暴力算法`$O(n^2)$`，显然要快速许多，因为它的复杂度是`$O(n)$`，双指针只需要遍历数组一次即可找到问题答案。

这个题目很有启发价值。它提醒我们，做算法题有时候需要一定的数学思维。此外，双指针分别设定在数组的两侧，也是一种非常常见的思路。
