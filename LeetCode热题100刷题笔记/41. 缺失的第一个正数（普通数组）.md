给你一个未排序的整数数组 `nums` ，请你找出其中没有出现的最小的正整数。

请你实现时间复杂度为 `O(n)` 并且只使用常数级别额外空间的解决方案。

***

如果不要求使用`$O(1)$`的空间，本题会非常简单：只要给整个数组建立一个哈希表，然后从小到大依次查表即可。这样的算法时间和空间的复杂度均为`$O(n)$`。

```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        // 建立哈希表
        map<int, int> num2index;
        for (int i = 0; i < nums.size(); i++) {
			if(nums[i] > 0)
            	num2index[nums[i]] = i;
        }

        // 从小到大查询哈希表
        int ans = 1;
        while (true) {
            auto ret = num2index.find(ans);
            if (ret == num2index.end())
                break;
            ans++;
        }

        return ans;
    }
};
```

但是本题要求使用常数级的额外空间，而又只能遍历一次数组（`$O(n)$`的时间复杂度），所以此题较为困难。

我苦思冥想寻找这道题的`$O(1)$`空间的算法，抓耳挠腮几个小时未果，最后万般无奈之下看了官方题解。结果万万没想到，官方题解前言里的这段话给我干无语了：

> 「真正」满足时间复杂度为 O(N) 且空间复杂度为 O(1) 的算法是不存在的，但是我们可以退而求其次：利用给定数组中的空间来存储一些状态。也就是说，如果题目给定的数组是不可修改的，那么就不存在满足时空复杂度要求的算法；但如果我们可以修改给定的数组，那么是存在满足要求的算法的。

我是万万没想到，所谓的`$O(1)$`空间的做法，竟然是从输入的数组空间里抠。这里的经验提醒了我们，通常算法的输入数组的空间复杂度是不作计算的。**如果我们有一个“常数空间”的算法，那么这个算法往往是借用了输入数组的空间完成了算法。**

这里我们采用非官方题解的解法，设法将输入数组整理成我们希望的哈希表的形式。这种方法又叫做“原地哈希”。

首先需要注意一个很显然的事实，根据题意，如果输入数组的长度为`$N$`，那么很显然，我们需要返回的最小正整数一定是在`$\left[1,N+1\right]$`的区间中。如果我们遍历完数组中的`$N$`个数，都没有发现缺失的正整数，那么缺失的正整数一定是`$N+1$`。

然后我们需要回顾此前哈希表的思路，哈希表这个结构为我们提供了一个最大的方便，那就是它允许我们乱“序”存入各个数值，但允许我们按顺序遍历。也就是说，我们往哈希表中存储数值时，可以是按3,1,5,9,……这样的顺序写入，但我们遍历存入的各个数时，却可以按照1,2,3,4,……这样的顺序依次查询。

现在我们不能使用哈希表这种额外空间了，那么我们便要想办法将输入的数组“改造成”支持顺序遍历的结构。我们希望，当我们做完改造操作时，整个数组都是”基本有序“的。如果数组中出现了1，那么它最好出现在数组的最开头；如果数组中出现了2，那么它最好出现在数组的第二位……以此类推，我们希望整数`$i$`能够出现在`$i-1$`的位置上。这样当我们从头开始遍历数组时，就相当于完成了从`$1$`到`$N$`依次查询哈希表的操作。

但现在我们只能使用常数级的额外空间，我们该怎么将输入数组整理成上面“基本有序”的形式呢？这时候参考前面“轮转数组”的思路，我们可以使用“交换”这一操作。

我们首先依次遍历整个数组。对于每个被遍历到的元素`$i$`，我们希望它能被置于第`$i-1$`位上。怎么将它移动到这一位置上呢？只需要将它与第`$i-1$`位上的元素交换就可以了。我们在遍历过程中，对每一个这样的元素执行这样的交换操作，这样遍历完一次数组，整个数组就达到了“基本有序”的状态。

这样操作的好处是，接下来，我们只需要复刻一遍对哈希表的查询操作。依次遍历`$[1,N]$`中的数，如果利用该数减一之后的值作为下标（索引）访问数组时，对应位置上不是该数，那么说明数组中缺失的就是该数。

基于此我们就可以写出下面的程序：

```cpp
// author:rmokerone
class Solution {
public:
    int firstMissingPositive(vector<int> &nums) {
        for (int i = 0; i < nums.size(); i++) {
            while (nums[i] != i + 1) {
                if (nums[i] <= 0 || nums[i] > nums.size() || nums[i] == nums[nums[i] - 1])
                    break;
                // 将nums[i] 放置到对应位置上[1,2,3...]
                int idx = nums[i] - 1;
                nums[i] = nums[idx];
                nums[idx] = idx + 1;
            }
        }
        for (int i = 0; i < nums.size(); i++) {
            if (nums[i] != (i + 1)) {
                return (i + 1);
            }
        }
        return (nums.size() + 1);
    }
};
```

回顾整个思考的过程，可以看出，本题实际上是考察我们“原地哈希”的能力。也就说，本题实际上希望我们能够将数组整理成哈希表这种数据结构——说得再深一点，就是自己能根据情况设计哈希函数，然后基于数组实现出哈希表。

还有一点需要注意，如果题目考察的是**原地算法**，那么类似倒序、交换之类的操作会非常有用，因为这些操作既容易对应操作，又容易保留原来的信息，并且还不占用额外的空间（或者说，不占用非常数级的额外空间）。
