给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

\*\*注意：\*\*答案中不可以包含重复的三元组。

***

这个题除了暴力解法暂时想不到什么更简便的方法了。即使是有“双指针”的提示，也想不到这两个指针该如何设置。万般无奈，只好看了题解。

题解的思路非常令人醍醐灌顶。题解首先假设了我们采用了经典的三重循环的暴力解法，考虑了三元组的表示形式。如果`$a\leq b \leq c$`，那么三元组`$(a,b,c)$`和`$(c,b,a)$`还有其他的4种排列都是合理的。那么这里我们不妨选择**从小到大排列**的一种（从大到小排列也可以，只要三元组各数有顺序即可）。这是因为，一旦输出的三元组确定了排序顺序（比如由小到大的顺序），那么当元组的首项`$a$`确定后，此后的元素`$b$`只需要在大于`$a$`的范围中搜寻即可，第三个元素`$c$`只需要在大于`$b$`的范围中搜寻即可。这样的想法启发了我们将输入数组排序，基于三重循环的思路和哈希表表示（可以节省一次循环），可以写出下面的代码：

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        // 先将输入的数组排序
        sort(nums.begin(), nums.end());

        // 创建一个哈希表，自带降重功能
        unordered_map<int, int> num2index;
        for(int i=0; i<nums.size(); i++){
            num2index[nums[i]] = i;
        }

        // 采用循环方法
        vector<vector<int>> ret;
        int first, second, third;
        // 按顺序选取第一个元素
        for(int i=0; i<nums.size(); i++){
            if(i != 0 && nums[i] == nums[i-1]){
                continue;
            }
            first = nums[i];

            // 按顺序选取第二个元素
            for(int j=i+1; j<nums.size(); j++){
                if(j != i+1 && nums[j] == nums[j-1]){
                    continue;
                }
                second = nums[j];

                // 采用哈希表寻找第三个元素，寻找速度更快
                third = 0 - first - second;
                if(third >= second && third >= first){
                    // 按前面的分析，第三个元素值要大于前两者
                    auto it = num2index.find(third);
                    if(it != num2index.end() && it->second != i && it->second != j){
                        // 哈希表中可以找到，且不与前两者的下标重复，因此将其加入到答案列表中
                        ret.push_back({first, second, third});
                    }
                }
            }
        }

        return ret;
    }
};
```

这样的算法时间复杂度大致为`$O(n^2)$`，如果使用原始的三重循环的方法则会是复杂度更高的`$O(n^3)$`。但是上面的算法并不简洁，因为实际上还是遍历了两次整个数组，而且还多了许多不必要的遍历。采用哈希表减少一次循环遍历的时间并不经济，因为这增加了无关的空间占用。

接下来我们顺着题解的思路继续简化：

上面的算法实际上摸到了双指针做法的边。当第一个元素确定下来之后，剩下的实际上是在剩余的数组中搜寻是否有两个数之和等于指定值的问题，也就是所谓的“两数之和”问题。\*\*对于一个已经排序好的数组，我们实际上可以让一个剩下的两个数一个从小到大，一个从大到小以相逆的方向进行遍历。\*\*如果这两个数值恰好满足要求，则填入答案；否则缩小右侧的数值，直至右侧的数值向左移动时越过第二个数值为止。

下面是官方题解：

```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        int n = nums.size();
        sort(nums.begin(), nums.end());
        vector<vector<int>> ans;
        // 枚举 a
        for (int first = 0; first < n; ++first) {
            // 需要和上一次枚举的数不相同
            if (first > 0 && nums[first] == nums[first - 1]) {
                continue;
            }
            // c 对应的指针初始指向数组的最右端
            int third = n - 1;
            int target = -nums[first];
            // 枚举 b
            for (int second = first + 1; second < n; ++second) {
                // 需要和上一次枚举的数不相同
                if (second > first + 1 && nums[second] == nums[second - 1]) {
                    continue;
                }
                // 需要保证 b 的指针在 c 的指针的左侧
                while (second < third && nums[second] + nums[third] > target) {
                    --third;
                }
                // 如果指针重合，随着 b 后续的增加
                // 就不会有满足 a+b+c=0 并且 b<c 的 c 了，可以退出循环
                if (second == third) {
                    break;
                }
                if (nums[second] + nums[third] == target) {
                    ans.push_back({nums[first], nums[second], nums[third]});
                }
            }
        }
        return ans;
    }
};
```

整个题其实对我们比较有启发的一点是，当我们意识到我们进行了许多不必要的遍历时，**我们其实可以通过双指针的方法改写遍历**，从而减少不必要的遍历次数，进而减少时间复杂度。双指针方法对于数组这种离散一维数据结构非常有用。

此外，**双指针似乎特别倾向于希望数组是有序的**。因为有序的数组，通常才能保证从两侧开始的双指针“不重不漏”，使得每次指针移动时不会漏掉可行解，即每一步的移动都能缩小解空间。
