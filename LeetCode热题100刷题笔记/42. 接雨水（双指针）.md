给定 `n` 个非负整数表示每个宽度为 `1` 的柱子的高度图，计算按此排列的柱子，下雨之后能接多少雨水。

***

这是遇到的第一个Hard题。即使半年之前使用Python结合题解完成过一次这道题，但是现在已经忘掉了Python的做法和思路了。没想出特别好的办法，只能去看题解。
这里直接沿用B站的讲解视频的思路：[数组实现](https://www.bilibili.com/video/BV1TSPeeGEHC/)
对于每个位置，当前位置能接多少雨水，取决于其左侧区域中最高的柱子和右侧区域中最高的柱子。找到其左右两侧最高的柱子后，两柱子高度的较小者减去当前位置的高度，就可获知当前位置可接的雨水量。
有了这样的观察，代码也能很快写出来：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        vector<int> left_max(n), right_max(n);

        // 从左向右遍历，找最高的柱子
        left_max[0] = height[0];
        for (int i = 1; i < n; i++)
            left_max[i] = max(height[i], left_max[i - 1]);

        // 从右向左遍历，找最高的柱子
        right_max[n - 1] = height[n - 1];
        for (int i = n - 2; i >= 0; i--)
            right_max[i] = max(height[i], right_max[i + 1]);

        // 遍历每个位置，计算当前位置能接的雨水数
        int ans = 0;
        for (int i = 0; i < n; i++) {
            ans += (min(left_max[i], right_max[i]) - height[i]);
        }
        return ans;
    }
};
```
采用numeric库中的partial_sum算法可以简化上面代码的编写：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        vector<int> left_max(n), right_max(n);

        // 从左向右遍历，找最高的柱子
        partial_sum(height.begin(), height.end(), left_max.begin(), ranges::max);

        // 从右向左遍历，找最高的柱子
        partial_sum(height.rbegin(), height.rend(), right_max.rbegin(), ranges::max);

        // 遍历每个位置，计算当前位置能接的雨水数
        int ans = 0;
        for (int i = 0; i < n; i++) {
            ans += (min(left_max[i], right_max[i]) - height[i]);
        }

        return ans;
    }
};
```
上面的做法虽然可以实现$O(3n)=O(n)$ 的时间复杂度，但是上面的做法却需要$O(2n)=O(n)$ 的空间。既然这道题被归类到“双指针”之下，那么这个方法自然可以采用双指针完成，也就是用$O(1)$的空间完成。
这里主要参考了[Leetcode官方题解](https://leetcode.cn/problems/trapping-rain-water/solutions/692342/jie-yu-shui-by-leetcode-solution-tuvc/)的思路：
>注意到下标 i 处能接的雨水量由 leftMax[i] 和 rightMax[i] 中的最小值决定。由于数组 leftMax 是从左往右计算，数组 rightMax 是从右往左计算，因此可以使用双指针和两个变量代替两个数组。
>维护两个指针 left 和 right，以及两个变量 leftMax 和 rightMax，初始时 left=0,right=n−1,leftMax=0,rightMax=0。指针 left 只会向右移动，指针 right 只会向左移动，在移动指针的过程中维护两个变量 leftMax 和 rightMax 的值。
>当两个指针没有相遇时，进行如下操作：
>使用 height[left] 和 height[right] 的值更新 leftMax 和 rightMax 的值；
>如果 height[left]<height[right]，则必有 leftMax<rightMax，下标 left 处能接的雨水量等于 leftMax−height[left]，将下标 left 处能接的雨水量加到能接的雨水总量，然后将 left 加 1（即向右移动一位）；
>如果 height[left]≥height[right]，则必有 leftMax≥rightMax，下标 right 处能接的雨水量等于 rightMax−height[right]，将下标 right 处能接的雨水量加到能接的雨水总量，然后将 right 减 1（即向左移动一位）。
>当两个指针相遇时，即可得到能接的雨水总量。

这个改进后的思路，也就是这个观察，确实很难用言语表达清楚。
总之，有了这个观察之后，很容易就可以写出下面的代码：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int ans = 0;
        int left = 0, right = height.size() - 1; // 左右两指针的初始位置
        int left_max = 0, right_max = 0;         // 左右两指针遍历时遇到的高度

        while (left < right) {
            // 遍历过程中维护left_max和right_max的值
            left_max = max(left_max, height[left]);
            right_max = max(right_max, height[right]);

            // 判断指针的移动逻辑，并同时计算雨水量
            if (height[left] < height[right]) {
                // ans += (min(left_max, right_max)-height[left]);
                ans += left_max - height[left];
                left++;
            } else {
                // ans += (min(left_max, right_max)-height[right]);
                ans += right_max - height[right];
                right--;
            }
        }
        return ans;
    }
};
```
当然还有另一种类似于双指针的思路。可以首先通过一次遍历找到最高的柱子height_max以及它的位置，那么在这个柱子的左侧，所有的数组right_max均为height_max，而在这个柱子的右侧，所有的数组left_max均为height_max，这样就相当于固定住了`min(left_max, right_max)`中的一者。这样，只需要分别遍历该最高柱子的左侧和右侧。
这个思路相较上面更为直观，但是需要多一次遍历找到最高的柱子height_max，时间复杂度略高。写出的代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int ans = 0;

        // 找到最高的柱子所在位置
        int max_idx =
            max_element(height.begin(), height.end()) - height.begin();

        // 从左向右遍历到最高的柱子
        int left_max = 0;
        for (int i = 0; i < max_idx; i++) {
            left_max = max(height[i], left_max);
            ans += left_max - height[i];
        }

        // 从右向左遍历到最高的柱子
        int right_max = 0;
        for (int i = height.size() - 1; i > max_idx; i--) {
            right_max = max(height[i], right_max);
            ans += right_max - height[i];
        }

        return ans;
    }
};
```
值得一提的是上面用到了algorithm库中的max_element函数实现了更为快速和方便的最大值搜寻。

最后，这道题其实还有一种思路。前面都是都是在位置上纵向切割数组，而有一种思路是横向切割数组，即假想横着剖开整个柱子，如果截面中出现了“中空”，那么说明这个”中空“处是需要蓄水的。如此不断沿用此法，直至遍历到柱子的最高高度。这个思路也很简单，但是这个方法最终的时间复杂度不仅与数组规模$n$有关，还与数组的最大值$height_{max}$有关，即$O(n\cdot height_{max})$，这个思路的时间复杂度实在太高。所以这里不再给出相关程序。
