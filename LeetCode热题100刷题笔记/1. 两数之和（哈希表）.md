给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出 \*\*和为目标值 \*\**`target`*  的那 **两个** 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用两次相同的元素。

你可以按任意顺序返回答案。

***

我的做法：

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        map<int, int> num2index; // 建立数值到对应索引的广义表

        // 建立数值到对应索引的广义表
        for(int i=0; i<nums.size(); i++){
            num2index[nums[i]] = i;
        }

        // 遍历输入数组里的每个元素，并寻找与其配对的元素
        for(int i=0; i<nums.size(); i++){
            auto iter = num2index.find(target-nums[i]);
            if(iter != num2index.end()){
                if(iter->second == i) continue;
                else return {i, iter->second};
            }
        }

        return {0, 0};
    }
};
```

官方做法中**使用unordered\_map来实现哈希表结构**。虽然map容器也可以实现哈希表结构，但是前者由于底层采用红黑树实现，因此其在遍历全部元素时可以按从小到大的顺序输出，但这也同时增加了通过键值查询值的时间，而**unordered\_map则无此特点，因此其按键值查询时用时更短**。

很多涉及到查询的情况，使用哈希表会更好。

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& nums, int target) {
        unordered_map<int, int> hashtable;
        for (int i = 0; i < nums.size(); ++i) {
            auto it = hashtable.find(target - nums[i]);
            if (it != hashtable.end()) {
                return {it->second, i};
            }
            hashtable[nums[i]] = i;
        }
        return {};
    }
};
```

一般情况下使用哈希表的思路是：先遍历输入数组中的每个元素，并建立数值到索引下标的哈希表；然后再遍历一遍数组中的每个元素，并查找哈希表中是否有满足匹配的元素。

但这里官方做法更为巧妙，将上面两步融合成一步，一边建立哈希表一边查询是否有满足要求的元素。若查询不到相应元素，则将其加入到哈希表中；否则，问题即解决，直接返回当前的两个元素下标即可。

除此之外，一些C++语法也要注意：

1.绝大多数的find方法，返回的都是相应元素的迭代器。如果查询不到，一般返回的都是相应容器的end迭代器（因为其实际上不指向任何元素）。

2.返回vector\<int>数组也可以使用大括号{}来简便表示。
