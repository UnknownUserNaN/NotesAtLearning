给定一个整数数组 `nums`，将数组中的元素向右轮转 `k`\* *个位置，其中 `k`* \*是非负数。

**进阶：**

*   尽可能想出更多的解决方案，至少有 **三种** 不同的方法可以解决这个问题。
*   你可以使用空间复杂度为 `O(1)` 的 \*\*原地 \*\*算法解决这个问题吗？

***

这个题题面要求很简单，但是细看函数实现其实有一个“小坑”，就是要求实现的函数实际上是没有返回值的。也就是说，这个函数要对传入的数组本身进行操作。

不管什么坑，上来先写一个最容易想到的，而且时间和空间复杂度均为`$O(n)$`的方法：

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        k = k%nums.size(); // 实际k有时会大于数组长度
        vector<int> ret;
        for(int i=nums.size()-k; i<nums.size(); i++)
            ret.push_back(nums[i]);
        for(int i=0; i<nums.size()-k; i++)
            ret.push_back(nums[i]);
        nums = ret;
    }
};
```

也可以增加一倍空间，然后把多余的元素删掉：

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int l = nums.size();
        k = k % l;
        nums.insert(nums.end(), nums.begin(), nums.begin() + l - k);
        nums.erase(nums.begin(), nums.begin() + l - k);
    }
};
```

但是，这些做法都无可避免地要使用额外的空间。如何实现“原地”的算法呢？

本题的提示中提到了**reverse**这个词，也就是倒序的做法。提示中重点提示了我们可以倒转**部分**的数组内容，从而实现目标的结果。这个提示已经相当接近最完美的答案了。

我们拿样例中`[1,2,3,4,5,6,7]`，且`k=3`的情况来推敲应该怎么做。我们的最终目标是，将这个数组通过部分的倒转操作，使之变成`[5,6,7,1,2,3,4]`。

观察结束时与最开始时的区别，可以发现`k=3`把数组分成了`[1,2,3,4]`和`[5,6,7]`两个部分，而且最后元素`7`和`1`相邻了。

如何使这两者相邻呢？显然，只需要各自倒转`[1,2,3,4]`和`[5,6,7]`这两个部分，这样就可以得到`[4,3,2,1,7,6,5]`。

到这里，离着目标已经相当接近了，我们发现这里各自倒转两次后的结果恰好与我们的目标差一个整体的倒序！这是因为两个子数组的部分倒序让原来的升序变成了降序，而我们要得到升序的结果，因此需要再给每个元素倒转一次，使之回归升序。所以是再做一次整体的倒序，我们就可以得到目标的数组了。这就是本题所要求的“原地”的算法。

当然，实际上，上面“部分倒序”和“整体倒序”的操作可以互相交换。但无论如何，最终的算法非常简洁：

```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int l = nums.size();
        k = k % l;
        reverse(nums.begin(), nums.end() - k);
		reverse(nums.end() - k, nums.end());
		reverse(nums.begin(), nums.end());
    }
};
```

官方题解用了另一种方式解释了这种倒转的思路：

> 该方法基于如下的事实：当我们将数组的元素向右移动 k 次后，尾部 kmodn 个元素会移动至数组头部，其余元素向后移动 kmodn 个位置。
>
> 该方法为数组的翻转：我们可以先将所有元素翻转，这样尾部的 kmodn 个元素就被移至数组头部，然后我们再翻转 \[0,kmodn−1] 区间的元素和 \[kmodn,n−1] 区间的元素即能得到最后的答案。

官方题解还提到了一种名为“环状替换”的方法，该方法从需要额外空间的交换法延伸而来。既然我们需要额外空间来完成交换，那我们为什么不将这个额外空间压缩到只剩单个变量呢？事实上这确实可以做到，只不过需要一种特殊的循环方法。由于此种方法太过繁琐，这里不再赘述。
