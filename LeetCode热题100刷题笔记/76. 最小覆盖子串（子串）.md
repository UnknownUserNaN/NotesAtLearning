给你一个字符串 `s` 、一个字符串 `t` 。返回 `s` 中涵盖 `t` 所有字符的最小子串。如果 `s` 中不存在涵盖 `t` 所有字符的子串，则返回空字符串 `""` 。

**注意：**

*   对于 `t` 中重复字符，我们寻找的子字符串中该字符数量必须不少于 `t` 中该字符数量。
*   如果 `s` 中存在这样的子串，我们保证它是唯一的答案。

\*\*进阶：\*\*你能设计一个在 `o(m+n)` 时间内解决此问题的算法吗？

***

不得不说，最后的“进阶”要求有点“剧透”了。这样低的一个时间范围，很显然需要使用哈希表类似的结构来降低时间复杂度。而想到哈希表，就想到了此前使用哈希表统计字符频次的类似做法。注意到这里的两个字符串中只会出现英文字母，考虑大小写也只有26种可能，因此这里完全可以使用一个长为`$26\times2=52$`的哈希表来存储频次信息。

我们可以不断遍历字符串s，一边遍历一遍记录其中各字符的频次，并将其记录到哈希表中。如果遍历过程中，发现s的各字符频次均大于t的各字符频次，那么便可以确定此时的s可以根据此前的信息“构造”出子串。

这样的想法很妙。但是需要注意的是，上述的条件一旦达成，此后的遍历过程中，子串都是可以被构造的了。

这里可以引入此前的前缀和思想。某一个开始于`$i$`，结束于`$j$`的子串，其频次可以通过遍历至`$j$`和遍历至`$i$`的两个频次作差得到。而如果s的某一子串的各字符频次恰好全部大于t的各字符频次，那么这个子串显然可以“覆盖”字符串t。如果两个字符串的频次相差最小，那么这个子串显然是所需的最短的子串。

这时我们可以结合提示，设置两个指针，一个左指针，一个右指针。首先将左指针固定在开始，右指针逐步移动到恰好可以构造子串的程度。然后开始移动左指针，如果可以构成符合题意的子串，则继续移动左指针，否则移动右指针。移动期间不断维护遇到的最短的子串，直至右指针移动到数组末尾。

基于上面的思路，可以写出如下的代码：

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        // 统计字符串t中的词频
        unordered_map<char, int> target_freq;
        for (int i = 0; i < t.length(); i++) {
            target_freq[t[i]]++;
        }

        // 设置左右指针，并按规则移动
        int left = 0, right = 0;
        unordered_map<char, int> window_freq;
        string ans = "";
        while (right < s.length()) {
            // 检查左右两指针的词频之差是否大于目标词频
            bool flag = true;
            for (auto each : target_freq) {
                char c = each.first;
                if (window_freq[c] < each.second) {
                    // 如果某一字符对应词频之差小于要求，则说明不能构造所需子串
                    flag = false;
                    break;
                }
            }

            // 根据上述结果决定如何移动左右指针
            if (flag) {
                // 构成的子串满足词频统计条件，则进一步移动左指针
                if (ans == "" || right - left < ans.length()) {
                    ans = s.substr(left, right - left);
                }
                window_freq[s[left]]--;
                left++;
            } else {
                // 构成的子串不满足词频统计条件，则进一步移动右指针
                window_freq[s[right]]++;
                right++;
            }
        }

        return ans;
    }
};
```

但是经过算例验证，这段程序有一个逻辑问题，就是在右指针移动到数组末尾时，没有继续尝试通过左指针收缩窗口。这是不合适的，应当在发现窗口可以构成合适的子串时，冻结住右指针然后不断地移动左指针。于是修改了上面的程序：

```cpp
class Solution {
public:
    bool isSatisfied(unordered_map<char, int>& target_freq,
                     unordered_map<char, int> window_freq) {
        // 检查左右两指针的词频之差是否大于目标词频
        bool flag = true;
        for (auto each : target_freq) {
            char c = each.first;
            if (window_freq[c] < each.second) {
                // 如果某一字符对应词频之差小于要求，则说明不能构造所需子串
                flag = false;
                break;
            }
        }
        return flag;
    }

    string minWindow(string s, string t) {
        // 统计字符串t中的词频
        unordered_map<char, int> target_freq;
        for (int i = 0; i < t.length(); i++) {
            target_freq[t[i]]++;
        }

        // 设置左右指针，并按规则移动
        int left = 0, right = 0;
        unordered_map<char, int> window_freq;
        string ans = "";
        while (left <= right) {
            // 根据上述结果决定如何移动左右指针
            if (isSatisfied(target_freq, window_freq)) {
                // 构成的子串满足词频统计条件，则不断移动左指针，直到左指针不满足条件为止
                do {
                    if (ans == "" || right - left < ans.length()) {
                        ans = s.substr(left, right - left);
                    }
                    window_freq[s[left]]--;
                    left++;
                } while (isSatisfied(target_freq, window_freq));
            } else {
                // 构成的子串不满足词频统计条件，则进一步移动右指针
                if (right >= s.size()) // 防越界处理
                    break;
                window_freq[s[right]]++;
                right++;
            }
        }

        return ans;
    }
};
```

但很遗憾，上面的程序虽然能够正常运行，但是运行时间超出要求了。这是因为，每进行一次循环，都要调用一次isSatisfied函数遍历一遍哈希表，这通常是比较耗费时间的。

按照ChatGPT的修改意见，可以在第一次时就确定好对应字符的匹配个数，然后在循环的过程中维护这个匹配个数。当所有字符能够完全匹配上时，就说明可以构造题目要求的子串了。而这种方法的优势是，可以在遍历过程中不断维护，每次可以用近乎`$O(1)$`的时间判断能否构成子串。

下面我直接贴上ChatGPT修改后的版本：

```cpp
class Solution {
public:
    string minWindow(string s, string t) {
        // 统计字符串 t 中的字符频数
        unordered_map<char, int> target_freq;
        for (char c : t) {
            target_freq[c]++;
        }
        
        // 当前窗口中满足条件的字符种数
        int required = target_freq.size();
        int formed = 0;
        
        // 记录窗口中的字符频数
        unordered_map<char, int> window_freq;
        
        // 结果保存格式：窗口长度，左边界，右边界
        int ans_len = INT_MAX, ans_left = 0, ans_right = 0;
        
        int left = 0, right = 0;
        while (right < s.size()) {
            // 将当前字符加入窗口
            char c = s[right];
            window_freq[c]++;
            
            // 如果当前字符的频数达到了目标要求，则计数加1
            if (target_freq.count(c) && window_freq[c] == target_freq[c])
                formed++;
            
            // 当窗口满足要求时，尝试收缩窗口
            while (left <= right && formed == required) {
                // 更新答案
                if (right - left + 1 < ans_len) {
                    ans_len = right - left + 1;
                    ans_left = left;
                    ans_right = right;
                }
                
                // 移除左侧字符
                char ch = s[left];
                window_freq[ch]--;
                if (target_freq.count(ch) && window_freq[ch] < target_freq[ch])
                    formed--;
                
                left++;
            }
            
            // 继续扩展窗口
            right++;
        }
        
        return ans_len == INT_MAX ? "" : s.substr(ans_left, ans_len);
    }
};
```
