给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。

如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

---
这个题要求必须要设计时间复杂度为$O(\log n)$的算法解决问题。看到这个要求，就很容易想到二分法。在严格升序数组中，查找元素的位置，可以非常简单地使用二分法。但是本题是**非递减**数组，这意味着要查找的元素可能会连续出现多次，甚至整个数组都是。因此我们还需要在二分法的基础上做一些改进：在我们使用二分法查找到元素后，立即向两侧寻找相同元素，找到相同元素则继续扩张窗口，直到遇到不同元素为止。
在面试时，初次写出的程序如下：
```cpp
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        int l = 0, r = nums.size() - 1;
        while (r - l > 1) {
            int idx = (l + r) / 2;
            if (nums[idx] == target) {
                l = idx;
                r = idx;
                while (nums[l] == target)
                    l--;
                while (nums[r] == target)
                    r++;
                return {l + 1, r - 1};
            } else if (nums[idx] < target)
                l = idx;
            else if (nums[idx] > target)
                r = idx;
        }
        return {-1, -1};
    }
};
```
这个程序有很多问题。首先是循环条件有问题，其次是二分法的搜索区间有问题，再就是大方向上的问题——不应该在找到元素后再线性扩张范围。如果数组内全是目标元素，那么这个算法的时间复杂度会达到$O(n)$，较为低效。
当然前两个问题也需要处理。面试官当时提醒，这个二分法的搜索区间是$\left[l,r\right]$，是个闭区间，这会导致在收缩搜索区间时，处理边界问题会比较棘手。上面的程序就遇到了这个问题。最终面试官建议可以考虑使用开区间表示搜索范围——哪怕是半开半闭区间都会好很多。
只可惜当时面试的时候面试官没有给改正程序的机会。后面按照面试官的意思，尝试把搜索区间改为左闭右开区间$\left[l,r\right)$，就好很多了：
```cpp
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        int n = nums.size();
        if (n == 0) return {-1, -1}; // 处理空数组情况
        
        int l = 0, r = n;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (nums[mid] == target) {
                l = idx;
                r = idx;
                while (nums[l] == target && l > 0)
                    l--;
                while (nums[r] == target && r < n - 1)
                    r++;
                return {l + 1, r - 1};
            } else if (nums[mid] < target) {
                l = mid + 1; // 调整左边界
            } else {
                r = mid - 1; // 调整右边界
            }
        }
        return {-1, -1};
    }
};
```
但是，事后看了相关的视频之后，才发现这种二分查找的写法是非常”不健康“的。按直觉来说，当二分法需要调整左右区间位置的时候，应该是`l=mid`或者`r=mid`这种形式才符合直觉，用`l=mid+1`和`r=mid-1`这种形式反而容易造成误解。为了能够沿用这种符合直觉的书写方式，我们需要将查找区间改为开区间$\left( l, r \right)$：
```cpp
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        int n = nums.size();
        if (n == 0) return {-1, -1}; // 处理空数组情况
        
        int l = -1, r = n; // (-1, n)的搜索区间恰好对应整个数组[0, n-1]的范围
        while (l + 1 < r) {
            int mid = (l + r) / 2;
            if (nums[mid] == target) {
                l = idx;
                r = idx;
                while (nums[l] == target && l > 0)
                    l--;
                while (nums[r] == target && r < n - 1)
                    r++;
                return {l + 1, r - 1};
            } else if (nums[mid] < target) {
                l = mid + 1; // 调整左边界
            } else {
                r = mid - 1; // 调整右边界
            }
        }
        return {-1, -1};
    }
};
```

上面的程序已经能够解决问题了，但是如前面所说，它在最差情况下的时间复杂度退化到了$O(n)$。这是我们需要解决的。实际上改进思路也很简单，只需要分别使用二分法查找我们要返回的区间左右端点就可以了。本题要求寻找的“元素的第一个和最后一个位置”，实际上是“第一个等于该元素的位置“和”第一个不等于该元素的位置减一“。当然这里还是会涉及到很多复杂的边界条件问题，这里再次贴上视频中的截图：
![[Pasted image 20250416012008.png]]
基于此，我们可以写出运行效率更高的程序。这里直接贴上[官方题解](https://leetcode.cn/problems/find-first-and-last-position-of-element-in-sorted-array/solutions/504484/zai-pai-xu-shu-zu-zhong-cha-zhao-yuan-su-de-di-3-4/)的做法，他们直接写了一个函数来专门完成二分查找：
```cpp
class Solution { 
public:
    int binarySearch(vector<int>& nums, int target, bool lower) {
        int left = 0, right = (int)nums.size() - 1, ans = (int)nums.size();
        while (left <= right) {
            int mid = (left + right) / 2;
            if (nums[mid] > target || (lower && nums[mid] >= target)) {
                right = mid - 1;
                ans = mid;
            } else {
                left = mid + 1;
            }
        }
        return ans;
    }

    vector<int> searchRange(vector<int>& nums, int target) {
        int leftIdx = binarySearch(nums, target, true);
        int rightIdx = binarySearch(nums, target, false) - 1;
        if (leftIdx <= rightIdx && rightIdx < nums.size() && nums[leftIdx] == target && nums[rightIdx] == target) {
            return vector<int>{leftIdx, rightIdx};
        } 
        return vector<int>{-1, -1};
    }
};
```
当然，上面的程序中其实还有一个小问题，就是数值溢出问题。问题出在`mid=(left+right)/2`这一行，原因在于如果`left`和`right`的值过大，`left`和`right`相加后会造成值溢出，从而导致其不会产生正确结果。为了尽可能地避免上述问题的发生，通常这一个语句会改写成：`mid=left+(right-left)/2`，也就是改为让`left`加一个比较小的数。
此外，对照上面的图可以看出，实际上这个思路可以很容易地调用C++的库函数来完成。其中，`lower_bound`函数用来寻找**第一个不小于value的元素**，`upper_bound`函数用来查找**第一个大于 value 的元素**。这两个函数都在`algorithm`库中。
```cpp
class Solution {
public:
    vector<int> searchRange(vector<int>& nums, int target) {
        int leftIdx =
            lower_bound(nums.begin(), nums.end(), target) - nums.begin();
        int rightIdx =
            upper_bound(nums.begin(), nums.end(), target) - nums.begin() - 1;
        if (leftIdx <= rightIdx && rightIdx < nums.size() &&
            nums[leftIdx] == target && nums[rightIdx] == target)
            return vector<int>{leftIdx, rightIdx};
        return vector<int>{-1, -1};
    }
};
```
这是本题的一个非常快的解决方式，还可以避开上面二分查找的边界问题，以及中点坐标的数值溢出问题。
