给定一个二叉树的根节点 root ，和一个整数 targetSum ，求该二叉树里节点值之和等于 targetSum 的 路径 的数目。

路径 不需要从根节点开始，也不需要在叶子节点结束，但是路径方向必须是向下的（只能从父节点到子节点）。

---
仍然可以用递归思路解决。以某节点为根的二叉树具有的符合条件的路径总和数，等于以该结点为根的路径总和数，加上其左右二叉树中具有的路径总和数。后者好计算，直接“丢”给递归就可以了。前者如果继续沿用递归思路的话，其实也好计算。以该结点为根的路径总和数，等于分别以该结点的左孩子和右孩子的路径总和数之和（不过目标数值需要调整）。
有了这个思路，程序就很好写了：
```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left),
 * right(right) {}
 * };
 */
class Solution {
public:
    int pathSum_root(TreeNode* root, long targetSum) {
        // 该函数用来计算以当前结点为根的路径的数目
        long sum_root, sum_left, sum_right; // 使用int有时会数值溢出

        if (root->left == nullptr)
            sum_left = 0;
        else
            sum_left = pathSum_root(root->left, targetSum - (root->val));

        if (root->right == nullptr)
            sum_right = 0;
        else
            sum_right = pathSum_root(root->right, targetSum - (root->val));

        if (root->val == targetSum)
            sum_root = 1;
        else
            sum_root = 0;

        return sum_root + sum_left + sum_right;
    }

    int pathSum(TreeNode* root, int targetSum) {
        if (root == nullptr)
            return 0;

        long sum_root, sum_left, sum_right;

        sum_root = pathSum_root(root, targetSum);

        if (root->left == nullptr)
            sum_left = 0;
        else
            sum_left = pathSum(root->left, targetSum);

        if (root->right == nullptr)
            sum_right = 0;
        else
            sum_right = pathSum(root->right, targetSum);

        return sum_root + sum_left + sum_right;
    }
};
```
官方题解的做法与上面的做法大同小异。但是需要指出的是，上面的方法虽然空间复杂度能达到$O(n)$，但是时间复杂度却比较高，达到了$O(n^2)$。所以上面的方法虽然能够AC，但是结果并不理想。
究其根本，还是因为我们进行了重复的遍历。上面程序中的`pathSum`和`pathSum_root`都需要向下遍历，但是两者的遍历结果并不“互通”。如果能让两者的遍历结果“互通”，那么能算法的时间复杂度能得到一个较大的降低。
联系本文的做法，不难想到前缀和算法。如果能在遍历过程中记录遍历至此的前缀和，那么就能省去多次遍历的计算量。这个过程很容易很容易跟深度优先的算法结合在一起。
下面给出[官方题解](https://leetcode.cn/problems/path-sum-iii/solutions/1021296/lu-jing-zong-he-iii-by-leetcode-solution-z9td/)的思路：
> 我们定义节点的前缀和为：由根结点到当前结点的路径上所有节点的和。我们利用先序遍历二叉树，记录下根节点 root 到当前节点 p 的路径上除当前节点以外所有节点的前缀和，在已保存的路径前缀和中查找是否存在前缀和刚好等于当前节点到根节点的前缀和 curr 减去 targetSum。
> - 对于空路径我们也需要保存预先处理一下，此时因为空路径不经过任何节点，因此它的前缀和为 0。
> - 假设根节点为 root，我们当前刚好访问节点 node，则此时从根节点 root 到节点 node 的路径（无重复节点）刚好为 root→p_1→p 2→…→p k→node，此时我们可以已经保存了节点 p 1,p 2,p 3,…,p k的前缀和，并且计算出了节点 node 的前缀和。
> - 假设当前从根节点 root 到节点 node 的前缀和为 curr，则此时我们在已保存的前缀和查找是否存在前缀和刚好等于 curr−targetSum。假设从根节点 root 到节点 node 的路径中存在节点 p_i 到根节点 root 的前缀和为 curr−targetSum，则节点 p +1到 node 的路径上所有节点的和一定为 targetSum。
> 我们利用深度搜索遍历树，当我们退出当前节点时，我们需要及时更新已经保存的前缀和。

官方题解给出的代码也很简洁：
```cpp
class Solution {
public:
    unordered_map<long long, int> prefix;

    int dfs(TreeNode *root, long long curr, int targetSum) {
	    // 参数curr用来保存遍历至此的前缀和
        if (!root) {
            return 0;
        }

        int ret = 0;
        curr += root->val; // 更新遍历至此的前缀和
        // 检查已经遍历过的路径上有没有前缀和为curr-targetSum的部分
        // 若有，则两个前缀和相减后（即curr-(curr-targetSum)）对应的片段就是满足条件的路径
        if (prefix.count(curr - targetSum)) { 
            ret = prefix[curr - targetSum];
        }

        prefix[curr]++; // 记录对应的前缀和
        ret += dfs(root->left, curr, targetSum);
        ret += dfs(root->right, curr, targetSum);
        prefix[curr]--; // 遍历结束，要退出对应节点，所以要删除遍历过的前缀和信息

        return ret;
    }

    int pathSum(TreeNode* root, int targetSum) {
        prefix[0] = 1;
        return dfs(root, 0, targetSum);
    }
};
```
这个方法最差情况下的空间复杂度达到了$O(n)$，较之前没有变动，但是时间复杂度成功降低到了$O(n)$，因为只需要对整个二叉树进行**一次**深度遍历就能完成任务。
