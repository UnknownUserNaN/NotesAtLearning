给定一个二叉树的根节点 root ，返回 它的 中序 遍历 。

---
遍历二叉树就是通过某条线路对二叉树的各个结点进行一次访问，访问的方法有三种分为前序（前根）遍历、中序（中根）遍历、后序（后根）遍历，层序遍历它们的遍历顺序如下所示：
> 前序遍历:根节点=》根节点的左子树=》根节点的右子树
> 中序遍历:根节点的左节点=》根节点=》根节点的右子树
> 后序遍历:根节点的左节点=》根节点的右节点=》根节点

这三种遍历方式有两种写法，一种是递归写法，一种是循环写法。递归写法非常简单：
```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode() : val(0), left(nullptr), right(nullptr) {}
 *     TreeNode(int x) : val(x), left(nullptr), right(nullptr) {}
 *     TreeNode(int x, TreeNode *left, TreeNode *right) : val(x), left(left),
 * right(right) {}
 * };
 */
class Solution {
public:
    vector<int> recursiveTrace(TreeNode*& root, vector<int>& trace) {
        if (root != nullptr) {
            if (root->left != nullptr)
                recursiveTrace(root->left, trace);
            trace.emplace_back(root->val);
            if (root->right != nullptr)
                recursiveTrace(root->right, trace);
        }
        return trace;
    }

    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> ans;
        recursiveTrace(root, ans);
        return ans;
    }
};
```
另一种是循环写法，这种写法实际上是把刚才递归方法中隐式的栈给明示了出来：
```cpp
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        stack<TreeNode*> st;
        vector<int> ans;
        while(root != nullptr || !st.empty()){
            // 先走到二叉树最左侧的结点
            while(root != nullptr){
                st.push(root);
                root = root->left;
            }
            // 开始逐步回溯
            root = st.top(); // 回访当前最左侧节点的根节点
            st.pop();
            ans.emplace_back(root->val); 
            root = root->right;
        }
        return ans;
    }
};
```
这两种写法实际上是等价的，其时间复杂度均为$O(n)$，空间复杂度均为$O(n)$，其中$n$为二叉树结点的总数。
然而，[官方题解](https://leetcode.cn/problems/binary-tree-inorder-traversal/solutions/412886/er-cha-shu-de-zhong-xu-bian-li-by-leetcode-solutio/)中提及了一种能将空间复杂度降至$O(1)$的算法（代价是修改树的结构），名为Morris算法。该种方法特别适合用来完成二叉树的中序遍历。
Morris 遍历算法整体步骤如下（假设当前遍历到的节点为 x）：
> 1. 如果 x 无左孩子，先将 x 的值加入答案数组，再访问 x 的右孩子，即 x=x.right。
> 2. 如果 x 有左孩子，则找到 x 左子树上最右的节点（即左子树中序遍历的最后一个节点，x 在中序遍历中的前驱节点），我们记为 predecessor。根据 predecessor 的右孩子是否为空，进行如下操作：
>     如果 predecessor 的右孩子为空，则将其右孩子指向 x，然后访问 x 的左孩子，即 x=x.left。
> 	如果 predecessor 的右孩子不为空，则此时其右孩子指向 x，说明我们已经遍历完 x 的左子树，我们将 predecessor 的右孩子置空（恢复树结构），将 x 的值加入答案数组，然后访问 x 的右孩子，即 x=x.right。
> 3. 重复上述操作，直至访问完整棵树。

实际上，整个Morris算法的核心在于**利用叶子结点的空闲指针**。由于中序遍历需要首先走到树的最左侧，因此左孩子通常是不可用的，因此只能利用右孩子的空闲指针。这些空闲指针使得遍历完子树的所有左孩子之后，能够成功返回到当前子树的根节点完成后续遍历。
Morris算法的核心在于，找到了哪些结点能用来完成“回传”。Morris算法将这些结点称之为“predecessor”，这些结点**通常是**（并不总是）最终的叶子结点，然后利用其空闲指针，使得算法在遍历到此处时，可以恰好返回到所在子树的根节点。Morris算法的解决方法是，修改其右指针，将其指回对应子树的根节点。
也可以站在一个更基本的思路回看整个Morris算法的原始思路。按照中序遍历的顺序要求，如果不使用栈结构记录当前结点的双亲，那么为了能够在遍历完之后能及时返回到双亲，必须在这些会”卡死“遍历的结点处提前创建好可以让程序返回双亲的指针。实际上，将这种思路如法炮制到前序遍历和后序遍历，就能得到Morris算法的其他形式。
基于以上思路，可以写出对应的代码：
```cpp
class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        vector<int> ans;
        TreeNode *p = root, *pre = nullptr;

        while (p != nullptr) {
            if (p->left == nullptr) {
                // x无左孩子
                ans.emplace_back(p->val);
                p = p->right;
            } else {
                // x有左孩子，先找predecessor（当前左子树的最右侧结点）
                pre = p->left;
                while (pre->right != nullptr && pre->right != p)
                    // 注意第二个条件，这里防止访问回传指针形成死循环
                    pre = pre->right;

                // 找到了predecessor
                if (pre->right == nullptr) {
                    pre->right = p;       // 它的右孩子还没用过，所以利用它创建回传指针
                    p = p->left; // 开始访问左子树
                } else {
                    // 它的右孩子已经用来创建回传指针了，说明此时已经遍历完了x的左子树
                    // 此时按照中序遍历的规律，该往答案数组中填入根节点了
                    ans.emplace_back(p->val); // 填入根节点
                    p = p->right;             // 访问回传指针，跳转回根节点
                    pre->right = nullptr;     // 恢复回传指针（恢复树结构）
                }
            }
        }

        return ans;
    }
};
```
