给你一个链表，删除链表的倒数第 n 个结点，并且返回链表的头结点。

进阶：你能尝试使用一趟扫描实现吗？

---
最简单的思路当然是遍历一遍数组，取得链表的长度，然后再从头开始遍历，找到要删除的结点。但这显然不经济，因为题目告诉我们，这个问题显然有一次扫描就解决的方法。
题目的提示说到，可以使用两个指针，并且两个指针有一定的延迟（delay）。这是一个非常“天才”的想法。假设我们要删除倒数第$n$个节点，我们可以一先一后地从头释放两个相同速度的指针，快指针走出$n$步之后再释放慢指针。这样，当快指针走到链表末尾后，慢指针恰好指向需要删除的结点。
基于此可以很快写出程序：
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        ListNode* prehead = new ListNode(0, head); // 新建一个什么都不做的头节点
        ListNode *fast = prehead, *slow = prehead;
        int counter = n;
        while (fast->next != nullptr) {
            fast = fast->next;
            if (counter != 0)
                counter--;
            else
                slow = slow->next;
        }
        slow->next = slow->next->next;
        return prehead->next;
    }
};
```
值得一提的是，为了防止某些恼人的边界条件（比如链表长度只有1并且要求你删掉这唯一的元素），上面的程序中增加了一个空的头结点。增加的新结点并不会改变算法的运行结果，但是却可以帮助避免上面的边界情况。
很幸运地，官方题解也是采用了这样的思路。不过官方题解提示了这种方法的“隐患”，即容易造成内存泄露：
>特别地，在某些语言中，由于需要自行对内存进行管理。因此在实际的面试中，对于「是否需要释放被删除节点对应的空间」这一问题，我们需要和面试官进行积极的沟通以达成一致。
```cpp
class Solution {
public:
    ListNode* removeNthFromEnd(ListNode* head, int n) {
        ListNode* dummy = new ListNode(0, head);
        ListNode* first = head;
        ListNode* second = dummy;
        for (int i = 0; i < n; ++i) {
            first = first->next;
        }
        while (first) {
            first = first->next;
            second = second->next;
        }
        second->next = second->next->next;
        ListNode* ans = dummy->next;
        delete dummy;
        return ans;
    }
};
```
