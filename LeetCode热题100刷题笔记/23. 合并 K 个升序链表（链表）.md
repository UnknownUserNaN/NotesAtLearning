给你一个链表数组，每个链表都已经按升序排列。

请你将所有链表合并到一个升序链表中，返回合并后的链表。

---
一个最简单的思路是，对这$k$个链表进行两两合并。两两合并的方法参考前面21题的做法。
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* merge2(ListNode* l1, ListNode* l2) {
        ListNode* preHead = new ListNode(-1);

        ListNode* prev = preHead;
        while (l1 != nullptr && l2 != nullptr) {
            if (l1->val < l2->val) {
                prev->next = l1;
                l1 = l1->next;
            } else {
                prev->next = l2;
                l2 = l2->next;
            }
            prev = prev->next;
        }

        prev->next = l1 == nullptr ? l2 : l1;

        return preHead->next;
    }

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        // 特殊情况处理
        if (lists.empty())
            return nullptr;
        if (lists.size() == 1)
            return lists[0];

        // 每两个进行一次合并
        ListNode *p1 = lists[0], *p2 = lists[1];
        for (int i = 1; i < lists.size(); i++) {
            p2 = lists[i];
            p1 = merge2(p1, p2);
        }

        return p1;
    }
};
```
当然，这样的时间复杂度是$O(k^2n)$（合并两个链表的复杂度是$O(n)$，但这个$n$会随着合并而递增），显然不是最简。实际上，可以参考二分法（分治法），将输入的数组中的链表不断地两两合并，直至链表中只剩下一个链表为止：
```cpp
class Solution {
public:
    ListNode* merge2(ListNode* l1, ListNode* l2) {
        ListNode* preHead = new ListNode(-1);

        ListNode* prev = preHead;
        while (l1 != nullptr && l2 != nullptr) {
            if (l1->val < l2->val) {
                prev->next = l1;
                l1 = l1->next;
            } else {
                prev->next = l2;
                l2 = l2->next;
            }
            prev = prev->next;
        }

        prev->next = l1 == nullptr ? l2 : l1;

        return preHead->next;
    }

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        // 特殊情况处理
        if (lists.empty())
            return nullptr;
        else if (lists.size() == 1)
            return lists[0];

        // 每两个进行一次合并
        while (lists.size() > 1) {
            int n = lists.size();
            for (int i = 0; i < n / 2; i++) {
                lists[i] = merge2(lists[i], lists[i + 1]);
                lists.erase(lists.begin() + i + 1);
            }
        }
        return lists[0];
    }
};
```
这样，可以将时间复杂度降低到$O(kn\log k)$，额外空间复杂度降到$O(1)$（实际的空间复杂度为$O(k)$，因为动用了输入的数组）。官方题解中的分治法则采用了递归的方法，因此需要额外的空间，额外空间复杂度来到了$O(\log k)$：
```cpp
class Solution {
public:
    ListNode* mergeTwoLists(ListNode *a, ListNode *b) {
        if ((!a) || (!b)) return a ? a : b;
        ListNode head, *tail = &head, *aPtr = a, *bPtr = b;
        while (aPtr && bPtr) {
            if (aPtr->val < bPtr->val) {
                tail->next = aPtr; aPtr = aPtr->next;
            } else {
                tail->next = bPtr; bPtr = bPtr->next;
            }
            tail = tail->next;
        }
        tail->next = (aPtr ? aPtr : bPtr);
        return head.next;
    }

    ListNode* merge(vector <ListNode*> &lists, int l, int r) {
        if (l == r) return lists[l];
        if (l > r) return nullptr;
        int mid = (l + r) >> 1;
        return mergeTwoLists(merge(lists, l, mid), merge(lists, mid + 1, r));
    }

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        return merge(lists, 0, lists.size() - 1);
    }
};
```
这种通过递归减少代码编写量的方法很适合参考。这里还可以注意一个细节，**当要对一个数组或者一个链表进行递归式的操作时，为递归函数设定操作范围（如上面代码中的`l`和`r`）是很实用的一个技巧**。
除此之外，官方题解还给出了优先队列的解法。其具体的思路是，每次循环时，将各个链表头部待连接的结点移动到要输出的链表末尾。优先队列可以保证，将各个链表乱序入队后，可以按一定会的顺序出队。这样，我们先将各个链表的头结点放入优先队列，优先队列会自动给这些结点排序；入队完毕后，就从队列中弹出一个元素，这个元素必然是这些结点中值最小的，于是将该结点连接到要输出的链表上，如果该结点还有后继，则紧接着将该结点的后继入队。如此循环往复，便可完成各个链表的合并。
当然，虽然思路很直接，但是实现起来还是需要使用C++的一些高级语法。比如我们需要新定义一个数据类型，使其包含结点的值，结点的指针，和给该数据类型排序的方法（供优先队列使用）。具体的实现可以参照下面官方题解中的程序：
```cpp
class Solution {
public:
    struct Status {
        int val;
        ListNode *ptr;
        bool operator < (const Status &rhs) const {
            return val > rhs.val;
        }
    };

    priority_queue <Status> q;

    ListNode* mergeKLists(vector<ListNode*>& lists) {
        for (auto node: lists) {
            if (node) q.push({node->val, node}); // 先将每个链表的头结点都放入优先队列
        }
        ListNode head, *tail = &head; 
        while (!q.empty()) {
            auto f = q.top(); q.pop();
            tail->next = f.ptr; 
            tail = tail->next;
            if (f.ptr->next) q.push({f.ptr->next->val, f.ptr->next});
        }
        return head.next;
    }
};
```
上述做法仍能维持$O(kn\log k)$的复杂度，但是却要使用$O(k)$的额外空间来存储优先队列。综合来看，其性能仍不如上面的分治法。
