你是一个专业的小偷，计划偷窃沿街的房屋。每间房内都藏有一定的现金，影响你偷窃的唯一制约因素就是相邻的房屋装有相互连通的防盗系统，如果两间相邻的房屋在同一晚上被小偷闯入，系统会自动报警。

给定一个代表每个房屋存放金额的非负整数数组，计算你 不触动警报装置的情况下 ，一夜之内能够偷窃到的最高金额。

---
设输入数组`nums`的长度为`n`，该题要求的结果使用动态规划数组表示应为`dp[n-1]`。我们首先从简单的例子开始尝试寻找这个问题的一般性质：
- `n==1`时，显然只能选那一间屋子。
- `n==2`时，显然是在这两间屋子中选一间金额更大的。
- `n==3`时，有两种选择方式，即`[0, 2]`和`[1]`这两个解，我们需要在这两种方案中选一个金额更大的。为了方便动态规划，我们将`[0, 2]`这个选项视为`n==1`时的结果与`nums[2]`的叠加，然后将其与`n==2`时的情况进行对比，选出二者中的较大值。
其实到这里，我们就能稍微体会出这个问题该如何考虑动态规划了。当我们计算`n==i`的情况时，考虑`nums[i-1]`的状态，它有“偷”和“不偷”两种选择。如果选择“偷”，那就意味着我们同时需要放弃`nums[i-2]`的房子，我们只能在`nums[0]`到`nums[i-3]`中规划，这就是`n==i-2`的情况；如果我们选择“不偷”，那么就意味着我们保留了偷`nums[i-2]`的可能性，这就是`n==i-1`的情况。具体选择“偷”还是“不偷”，则是看这两种选择的哪一个预期收益最大。
所以，经过这层推理，我们不难得出下面的状态转移方程：
$$dp[i]=\max\left(dp[i-2]+nums[i-1], dp[i-1]\right)\quad i>2$$
至于$i\leq2$的边界情况，则可以参照一开始的分析完成。
现在可以很容易地写出程序了：
```cpp
class Solution {
public:
    int rob(vector<int>& nums) {
        if (nums.size() == 1)
            return nums[0];
        if (nums.size() == 2)
            return max(nums[0], nums[1]);
        vector<int> dp(nums.size());
        for (int i = 0; i < nums.size(); i++) {
            if (i == 0)
                dp[i] = nums[0];
            else if (i == 1)
                dp[i] = max(nums[0], nums[1]);
            else
                dp[i] = max(dp[i - 2] + nums[i], dp[i - 1]);
        }
        return dp[nums.size() - 1];
    }
};
```
该程序的时间复杂度为$O(n)$，空间复杂度也为$O(n)$。
