整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 在下标 3 处经旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

提示：
- nums 中的每个值都 独一无二
- 题目数据保证 nums 在预先未知的某个下标上进行了旋转

---
此题归类在二分查找之下，很显然本题需要通过二分查找来解决问题。但本题的棘手之处在于输入的数组破坏了二分查找“钟意”的单调性，轮转数组导致数组是分段单调的。如果能将这个轮转点找到，问题就能化归成原来的二分查找形式。
如何找到这个轮转点呢？最简单的办法是遍历整个数组，如果当前元素小于其前一个元素，那么它显然就是轮转点（也就是题目中的nums[0]）。但是这样遍历，整个算法的时间复杂度将达到$O(n)$，不符合题目要求。
到这里，思路陷入了僵局。无法，只得去求助题解。
[灵茶山艾府](https://leetcode.cn/problems/search-in-rotated-sorted-array/solutions/1987503/by-endlesscheng-auuh/)的题解非常好，为本题提供了两种思路：一种思路是两次二分查找，第一次查找找到数组中的最小值（这里参见153. 寻找旋转排序数组中的最小值），该最小值就是轮转点的位置，然后再设法在相应区间进行查找即可；第二种思路则是只进行一次二分查找，一边进行二分一边判断对应区间到底哪一个是有序的，然后再在对应区间中寻找。这里我们重点讲第二种方法。
首先需要明白轮转数组的一个特点，整个单调的数组经过轮转之后，变成了两个区间上分段单调的数组，并且左侧单调区间的最小值（实际上就是`nums[0]`），始终大于右侧单调区间的最大值（也就是`nums[n-1]`)。
此外，我们要谨记一个原则，二分查找之所以要求数组的单调性，是因为中点处的取值与目标值的大小能够“指引”我们目标值在中点的哪一侧。以单调递增的数组为例，如果中点值大于目标值，则说明目标值在中点值左侧，搜索区间右侧端点应当收缩；反之应该收缩左侧端点。需要注意的是，上述过程实际上并不“强求”数组的单调性，**如果我们能通过一些条件判断出目标值在中点值的哪一侧，那么二分法依然可以正常进行。**
本题便是这样的例子。设此时的搜索区间中点`mid`已经确定，其中点值为$nums[mid]=x$。假若目标值存在在数组中，那么只有两个可能：一是中点值和目标值分别在不同的单调区间，二是中点值和目标值在相同的单调区间中。
后者是容易处理的，因为这直接划归到了最原始的单调区间中的二分查找。前者稍微棘手一点，但是，如果我们能确定中点值（或者目标值）在左侧单调区间还是右侧单调区间，那就可以得知目标值在中点值的哪一侧，从而可以借此收缩搜索区间。
这一步可以借用前面的推理，左侧单调区间中的数值（哪怕是最小值），也都大于右侧单调区间的最大值（也就是`nums[n-1]`）。我们可以借此关系确定目标值是在哪个单调区间，如果`x>nums[n-1]`，那么`x`必然在左侧单调区间，否则（也就是`x<=nums[n-1]`）则在右侧单调区间。目标值也可以采用类似的判断方式，最终划归到上面分析的两种情况。
上面整个推理的核心步骤是利用边界值的特点确定目标值与中点值的位置关系。以最右侧的边界值为例，由于轮转数组局部单调递增的特点，其左侧只能出现比它还小的数，如果出现了比它大的数，则该数必然不与最右侧边界值处于同一个单调递增区间。
基于上面的这些推理，可以得到程序：
```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {
        int n = nums.size();
        if (n == 0)
            return -1;

        int l = -1, r = n; // 开区间写法
        while (l + 1 < r) {
            int mid = l + (r - l) / 2;
            if (nums[mid] == target)
                return mid;

            // 先检查mid在左侧单调区间还是右侧单调区间
            if (nums[mid] > nums[n - 1]) {
                // mid在左侧单调区间
                if (target <= nums[n - 1])
                    // target在右侧单调区间，因此target在mid的右侧，区间应当向右收缩
                    l = mid;
                else {
                    // target和mid一样都在左侧单调区间
                    if (target > nums[mid])
                        l = mid;
                    else
                        r = mid;
                }
            } else {
                // mid在右侧单调区间
                if (target <= nums[n - 1]) {
                    // target也在右侧单调区间
                    if (target > nums[mid])
                        l = mid;
                    else
                        r = mid;
                } else
                    // target在左侧单调区间，因此target在mid的左侧，区间应当向左收缩
                    r = mid;
            }
        }
        return -1;
    }
};
```
实际上这个问题不止这一种区分方式，比如我在评论区就见到了这种写法：
```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {
        int n = nums.size();
        if (n == 0)
            return -1;

        int l = -1, r = n; // 开区间写法
        while (l + 1 < r) {
            int mid = l + (r - l) / 2; // 防止数据溢出的写法
            if (nums[mid] == target)
                return mid;

            // 首先判断目标和中点不在同一半区间的情况
            if (target <= nums[n - 1] && nums[mid] > nums[n - 1])
                // 目标在右半区间（局部有序特点），中点在左半区间
                l = mid; // 向着目标所在的右半区间收缩
            else if (target > nums[n - 1] && nums[mid] <= nums[n - 1])
                // 目标不在右半区间，则必然在左半区间，中点在右半区间
                r = mid; // 向着目标所在的左半区间收缩
            else {
                // 目标和中点在同一半区间
                if (nums[mid] < target)
                    // 中点在目标左侧，则应当将搜索区间向右收缩
                    l = mid;
                else
                    // 中点在目标右侧，则应当将搜索区间向左收缩
                    r = mid;
            }
        }
        return -1;
    }
};
```

```cpp
class Solution {
public:
    int search(vector<int>& nums, int target) {
        int n = (int)nums.size();
        if (!n) {
            return -1;
        }
        if (n == 1) {
            return nums[0] == target ? 0 : -1;
        }
        int l = 0, r = n - 1;
        while (l <= r) {
            int mid = (l + r) / 2;
            if (nums[mid] == target) return mid;
            if (nums[0] <= nums[mid]) {
                if (nums[0] <= target && target < nums[mid]) {
                    r = mid - 1;
                } else {
                    l = mid + 1;
                }
            } else {
                if (nums[mid] < target && target <= nums[n - 1]) {
                    l = mid + 1;
                } else {
                    r = mid - 1;
                }
            }
        }
        return -1;
    }
};
```
