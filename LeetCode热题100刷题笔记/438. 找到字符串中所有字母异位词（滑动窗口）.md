给定两个字符串 `s` 和 `p`，找到 `s`\*\* **中所有 `p`** \*\*的 \*\*异位词 \*\*的子串，返回这些子串的起始索引。不考虑答案输出的顺序。

***

此题仍然归在“滑动窗口”的分类之下，这启发我们仍然使用滑动窗口的方法来考虑此题。

最容易想到的办法就是先排序字符串p，然后再在s中设置一个滑动窗口，滑动窗口内生成子串并排序。如果窗口内子串与目标子串相同则生成一个下标，否则滑动窗口到下一个位置。

这样的程序很快就写出来了，但是毫无意外地执行超时了：

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        // 将目标字符串排序
        sort(p.begin(), p.end());

        // 不断滑动窗口生成子串，并不断与目标子串比较
        vector<int> ans;
        if (p.length() > s.length())
            return ans;
        for (int i = 0; i < s.length() - p.length() + 1; i++) {
            string window = s.substr(i, p.length());
            sort(window.begin(), window.end());
            if (window == p)
                ans.push_back(i);
        }

        return ans;
    }
};
```

很明显，超时的主要原因在于上面的算法过于依赖排序算法了。窗口每滑动一次，就要调用一次排序算法，这在窗口较大的时候往往是非常耗时的。但是找到了问题症结，却仍然想不出能替代排序的方案。无奈只得看了题解。

题解对我最大的一个启发就是，题解**注意到了本题中涉及的均是小写英文字符**。这意味着本文只涉及到26个英文字符，因此完全可以**通过一个长度为26的数组来记录窗口中各字符的频次信息**。在窗口每次移动的时候，都设法维护这个频次信息。窗口每滑动一次，就减少滑出字符的频次，增加滑入字符的频次，然后检查该频次是否与目标字符串p的频次相等。我根据题解思路编写的代码如下：

```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        // 将目标字符串排序
        sort(p.begin(), p.end());

        // 例外情况
        if (p.length() > s.length())
            return vector<int>();

        // 数组初始化
        vector<int> ans;
        vector<int> sCount(26);
        vector<int> pCount(26);

        // 统计起始位置处的各字母频次
        for(int i=0; i<p.length(); i++){
            sCount[s[i]-'a']++;
            pCount[p[i]-'a']++;
        }
        if(sCount == pCount){
            ans.push_back(0);
        }
        
        // 开始滑动窗口，并维护sCount的值
        for (int i = 1; i < s.length() - p.length() + 1; i++) {
            sCount[s[i-1]-'a']--;
            sCount[s[i+p.length()-1]-'a']++;

            if(sCount == pCount){
                ans.push_back(i);
            }
        }

        return ans;
    }
};
```

当然，这里还有一个可以优化的细节：在窗口开始滑动的时候，我们实际上只关心两个频次之间的差距。因此，假如窗口在起始位置时，两个频次之间的差为0，那么，当窗口滑离起始位置时，只要这个频次之差能再次回归到0，那么就可以确定当前窗口满足题意了。

基于这个思路，实际上可以不必维护sCount和pCount两个数组，而转而维护两者的差这一个数组。不过这都是一点细节上的差异了，对于整个算法的时间复杂度影响不大。

回顾整个做题的全过程，个人认为有两点需要注意：

**1.当需要判断异位词的时候，往往要反应到它对应着“相同的字符频次信息”，可以考虑用排序后的字符串作为它的唯一“标识”，也可以考虑用此题中的固定长度的数组；**

**2.当题目中涉及的元素的“取值范围”比较小时（比如本题只涉及26个小写英文字母的情况），考虑使用一个固定长度的“索引表”来记录信息往往会有奇效。**
