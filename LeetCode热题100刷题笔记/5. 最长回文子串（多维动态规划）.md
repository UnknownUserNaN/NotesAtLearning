给你一个字符串 s，找到 s 中最长的 回文 子串。

提示： s 仅由数字和英文字母组成

---
本题中要求的子串是连续的，那么字符串`s`中所有可能的子串均可以通过枚举其左右区间`[l, r]`来实现。但这样做的时间复杂度很有可能会达到$O(n^3)$，非常不经济。通过多维动态规划方法能够通过“空间换时间”的做法降低时间复杂度到$O(n^2)$。
我们建立一个二维的动态规划数组`dp[i][j]`，其大小为`n*n`，每一个元素的值为该区间`[i, j]`上的子串是否是回文的（布尔值）。显然，这个`n*n`大小的数组只有`i<=j`的部分是有意义的。
我们先确定最简单的边界条件：当`i==j`时我们可以很容易确定`dp[i][j]=s[i]`，因为此时的子串只有一个字符，显然是回文的，于是`dp[i][j]=true`。当`i==j-1`时，如果`s[i]==s[j]`，那么显然这个子串也是回文的，否则则不是回文的，于是`dp[i][j]=(s[i]==s[j])`。
随后，我们考虑在上面两种情况两侧增加字符，也就是将区间扩展为`[i-1, j+1]`。显然，如果`dp[i][j]==true`，那么如果这时我们在两侧加上的字符`s[i-1]`和`s[j+1]`满足`s[i-1]=s[j+1]`，那么就能确定此时区间上的子串也是回文的。但如果上述两个情况有一者不满足，那么显然这个子串也是不回文的。综合上述条件，我们便可以确定，`dp[i-1][j+1]=(dp[i][j])&&(s[i-1]==s[j+1]`，写成更方便操作的形式，也就是`dp[i][j]=(dp[i+1][j-1])&&(s[i]==s[j]`。
填充完整个`dp`数组之后，要获得最长的回文子串，我们只需要遍历整个数组，找到两个坐标值相差最大的、取值为`true`的元素即可。当然，效率更高的做法是，我们一边填充`dp`数组，一边记录坐标值相差最大的、取值为`true`的元素。找到这个元素之后，最终直接返回该元素的位置所对应的子串即可。
当然上面还需要注意一点，那就是数组更新的顺序。因为该数组中有相当大部分的内容依赖于边界条件的结果，所以我们要首先将容易确定的边界条件所对应的数组更新了，然后再更新剩余部分的数组。为了方便程序的编写，我们应当从区间长度较短的情况开始更新，然后逐步枚举对应长度的区间。我们枚举从`1`到`s`长度的区间长度`l`，然后枚举每个区间的左端点`i`，计算出右端点`j`。
基于上面的分析，可以写出下面的程序：
```cpp
class Solution {
public:
    string longestPalindrome(string s) {
        int n = s.size();
        int max_len = 0, max_posi = 0;
        vector<vector<bool>> dp(n, vector<bool>(n, false));

        for (int l = 1; l <= n; l++) {
            for (int i = 0; i <= n - l; i++) {
                int j = i + l - 1;
                if (l <= 2)
                    dp[i][j] = s[i] == s[j];
                else
                    dp[i][j] = dp[i + 1][j - 1] && (s[i] == s[j]);
                if (dp[i][j] && l > max_len) {
                    max_len = l;
                    max_posi = i;
                }
            }
        }

        return s.substr(max_posi, max_len);
    }
};
```
该程序的时间复杂度为$O(n^2)$，空间复杂度为$O(n^2)$。
[官方题解](https://leetcode.cn/problems/longest-palindromic-substring/solutions/255195/zui-chang-hui-wen-zi-chuan-by-leetcode-solution/)的写法与上面这种写法稍有不同。
```cpp
class Solution {
public:
    string longestPalindrome(string s) {
        int n = s.size();
        if (n < 2) {
            return s;
        }

        int maxLen = 1;
        int begin = 0;
        // dp[i][j] 表示 s[i..j] 是否是回文串
        vector<vector<int>> dp(n, vector<int>(n));
        // 初始化：所有长度为 1 的子串都是回文串
        for (int i = 0; i < n; i++) {
            dp[i][i] = true;
        }
        // 递推开始
        // 先枚举子串长度
        for (int L = 2; L <= n; L++) {
            // 枚举左边界，左边界的上限设置可以宽松一些
            for (int i = 0; i < n; i++) {
                // 由 L 和 i 可以确定右边界，即 j - i + 1 = L 得
                int j = L + i - 1;
                // 如果右边界越界，就可以退出当前循环
                if (j >= n) {
                    break;
                }

                if (s[i] != s[j]) {
                    dp[i][j] = false;
                } else {
                    if (j - i < 3) {
                        dp[i][j] = true;
                    } else {
                        dp[i][j] = dp[i + 1][j - 1];
                    }
                }

                // 只要 dp[i][L] == true 成立，就表示子串 s[i..L] 是回文，此时记录回文长度和起始位置
                if (dp[i][j] && j - i + 1 > maxLen) {
                    maxLen = j - i + 1;
                    begin = i;
                }
            }
        }
        return s.substr(begin, maxLen);
    }
};
```
除了上面这种经典的动态规划算法外，官方题解还给出了一种“中心拓展方法”。枚举每个长度为1和2的区间，然后通过两侧添加字符的方式扩张这个区间，并参考前面动态规划的方法，检查扩张后的区间是否也是回文的。在整个遍历的过程中，记录遇到的最长回文子串。这个算法的时间复杂度与上面的算法相同，但空间复杂度因为采用了循环方法而得以降低到$O(1)$。
```cpp
class Solution {
public:
    pair<int, int> expandAroundCenter(const string& s, int left, int right) {
        while (left >= 0 && right < s.size() && s[left] == s[right]) {
            --left;
            ++right;
        }
        return {left + 1, right - 1};
    }

    string longestPalindrome(string s) {
        int start = 0, end = 0;
        for (int i = 0; i < s.size(); ++i) {
            auto [left1, right1] = expandAroundCenter(s, i, i);
            auto [left2, right2] = expandAroundCenter(s, i, i + 1);
            if (right1 - left1 > end - start) {
                start = left1;
                end = right1;
            }
            if (right2 - left2 > end - start) {
                start = left2;
                end = right2;
            }
        }
        return s.substr(start, end - start + 1);
    }
};
```
然而除了上面这两种算法外，还有一种更为优化的算法，叫Manacher算法。该算法跟KMP算法一样，利用额外的空间减少重复访问某些字符的次数，从而降低运行时间。该算法的时间复杂度为$O(n)$，空间复杂度为$O(n)$，可以说是效果最好的算法了。
```cpp
class Solution {
public:
    int expand(const string& s, int left, int right) {
        while (left >= 0 && right < s.size() && s[left] == s[right]) {
            --left;
            ++right;
        }
        return (right - left - 2) / 2;
    }

    string longestPalindrome(string s) {
        int start = 0, end = -1;
        string t = "#";
        for (char c: s) {
            t += c;
            t += '#';
        }
        t += '#';
        s = t;

        vector<int> arm_len;
        int right = -1, j = -1;
        for (int i = 0; i < s.size(); ++i) {
            int cur_arm_len;
            if (right >= i) {
                int i_sym = j * 2 - i;
                int min_arm_len = min(arm_len[i_sym], right - i);
                cur_arm_len = expand(s, i - min_arm_len, i + min_arm_len);
            } else {
                cur_arm_len = expand(s, i, i);
            }
            arm_len.push_back(cur_arm_len);
            if (i + cur_arm_len > right) {
                j = i;
                right = i + cur_arm_len;
            }
            if (cur_arm_len * 2 + 1 > end - start) {
                start = i - cur_arm_len;
                end = i + cur_arm_len;
            }
        }

        string ans;
        for (int i = start; i <= end; ++i) {
            if (s[i] != '#') {
                ans += s[i];
            }
        }
        return ans;
    }
};
```
