已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：
若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]
注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

提示：
- nums 中的所有整数 互不相同
- nums 原来是一个升序排序的数组，并进行了 1 至 n 次旋转

---
这道题是在”33. 搜索旋转排序数组“题后完成的。有了第33题的加持，做这道题会容易不少。
二分查找的关键是，能否确定目标值与中点值的左右关系。这往往要用到题目中隐藏的数量关系。比如对于此题中涉及的轮转数组，其通常由左右两个严格递增区间组成，并且左侧单调区间的最小值（即`nums[0]`)必然大于右侧单调区间的最大值（即`nums[n-1]`）。这是一个我们判断中点值和目标值在哪一侧的单调区间的重要方式。
目标值与中点值的位置关系有且仅有两种可能：要么分属于不同的单调区间，要么同属于相同的单调区间。但本题的情况较为特殊，要求我们找到数组中的最小值。根据上面性质我们容易知道，最小值一定出现在右侧单调区间的最左侧。所以，前面的位置关系实际上可以直接划归成两种更为简单的情况，即：中点值在左侧区间，中点值在右侧区间。如果我们的搜索区间是开区间，那么当且仅当右端点`r`与左端点`l`分属不同区间，且`r-l<=1`时，才可以确定找到了数组的最小值（也就是`nums[r]`）。
基于上面的思考，可以很容易地写出如下的程序：
```cpp
class Solution {
public:
    int findMin(vector<int>& nums) {
        int n = nums.size();
        int l = -1, r = n; // 开区间写法
        int mid = l + (r - l) / 2;
        while (r - l > 1) {
            mid = l + (r - l) / 2;
            if (nums[mid] > nums[n - 1])
                // 中点值在左侧单调区间，应当收缩左端点
                l = mid;
            else
                // 中点值在右侧单调区间，应当收缩右端点
                r = mid;
        }
        return nums[r]; // 此时右端点的值必然是要找的最小值（因为只有右端点在右侧单调区间）
    }
};
```
程序很简单，并且可以达到所需的$O(\log n)$的时间复杂度。上面的程序里，搜索区间是开区间，官方题解采用了闭区间写法：
```cpp
class Solution {
public:
    int findMin(vector<int>& nums) {
        int low = 0;
        int high = nums.size() - 1;
        while (low < high) {
            int pivot = low + (high - low) / 2;
            if (nums[pivot] < nums[high]) {
                high = pivot;
            }
            else {
                low = pivot + 1;
            }
        }
        return nums[low];
    }
};
```
这里无非是一些细节上的差异，最终的时间复杂度都是相同的。
