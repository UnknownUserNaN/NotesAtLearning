给定一个整数数组 temperatures ，表示每天的温度，返回一个数组 answer ，其中 answer[i] 是指对于第 i 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 0 来代替。

---
本题要求使用栈的话就很容易想到思路。我们首先创造一个与输入数组同长度的数组。遍历数组的过程中，我们维护一个栈，这个栈将记录每个元素和它出现的位置。如果当前遍历到的元素大于栈顶元素，那么便可以将该栈顶元素对应位置的结果置为两个下标的差，然后将该栈顶元素弹出，当然还要继续重复此步骤，直至栈为空或者当前元素小于等于栈顶元素。这时我们便可以将该元素和它所在的位置入栈了。
当然，很有可能遍历完数组之后，栈还没有清空。这时我们只需要结果数组中的对应位置置0即可，因为这代表着我们没找到比它更大的数。当然，我们也可以在初始化时将所有位置都置为0，这样就可以把最后一步省掉。
基于上面的思考，可以很容易写出下面的程序：
```cpp
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        vector<int> ret(temperatures.size(), 0); // 输出数组与输入数组同形
        stack<pair<int, int>> stk; // 栈中存入数组元素和它对应的位置
        for (int i = 0; i < temperatures.size(); i++) {
            while (stk.size() != 0 && temperatures[i] > stk.top().first) {
                ret[stk.top().second] = i - stk.top().second;
                stk.pop();
            }
            stk.push({temperatures[i], i});
        }
        return ret;
    }
};
```
这个程序的时间复杂度为$O(n)$，空间复杂度为$O(n)$。感觉已经是最好的做法了，但是提交时发现运行时间达到了38ms，只击败了18.21%的人。不知为何。
[官方题解](https://leetcode.cn/problems/daily-temperatures/solutions/283196/mei-ri-wen-du-by-leetcode-solution/)的做法与上面的思路相同，区别在于，官方题解只把对应元素的下标入栈，而没有将元素本身一同入栈，因为知道了对应元素的下标实际上就等于知道了元素本身。这减少了不少空间消耗，但空间复杂度也是在$O(n)$级别。
```cpp
class Solution {
public:
    vector<int> dailyTemperatures(vector<int>& temperatures) {
        int n = temperatures.size();
        vector<int> ans(n);
        stack<int> s;
        for (int i = 0; i < n; ++i) {
            while (!s.empty() && temperatures[i] > temperatures[s.top()]) {
                int previousIndex = s.top();
                ans[previousIndex] = i - previousIndex;
                s.pop();
            }
            s.push(i);
        }
        return ans;
    }
};
```
实际运行起来，官方题解的做法要稍快一些，用时25ms，也是只击败了65.84%的人。不知为何。
另外，看完官方题解我才知道，原来上面使用栈的方法实际上是单调栈的做法。
> 如果我们需要在 O(n) 的时间复杂度内求出数组中各个元素右侧第一个更大的元素及其下标，然后一并得到其他信息，这时候往往需要单调栈。
