给定 n 个非负整数，用来表示柱状图中各个柱子的高度。每个柱子彼此相邻，且宽度为 1 。

求在该柱状图中，能够勾勒出来的矩形的最大面积。

---
首先先想办法把题目中要求的数据转换成严谨的数学语言。假设数组是`heights`，要计算它在下标为`[l,r]`上的“矩形面积”，那么这个面积`S`应当满足`S=(r-l+1)*min(heights[l:r])`。题目希望我们找到所有`l`和`r`情况下的`S`的最大值。
看到这个问题，我们会有一种朴素的直觉。假设我们固定住左端点`l=l0`，移动右端点`r`，如何构造找出这种情况下最大的`S`？很容易想到的就是，我们不断尝试性地向右移动右端点`r`，如果始终有`heights[r]>=heights[l0]`，那么很显然我们越向后移动`r`，得到的`S`面积就越大。这个面积增大的过程会一直持续，直到我们遇到第一个位置`r=r0`使得`heights[r0]<heights[l0]`。如果我们截止在`r=r0`时，此时的面积是`S1=(r0-l0+1)*heights[r0]`；如果我们截止在`r=r0-1`时，此时的面积是`S2=(r0-l0)*heights[l0]`。由于`r0-l0+1>r0-l0`，但是`heights[r0]<heights[l0]`，所以这两个时刻的面积究竟谁大谁小也不能完全确定，只能通过具体数值的运算才能确定。
现在我们好奇的是，在区间`[l0,r0]`上，我们能否构造出一个面积比`S1`或者`S2`大的矩形吗？假设有`l1`和`r1`满足`l0<=l1<r1<=r0`，那么在这两个限定的区间上构造出的矩形面积是`S3=(r1-l1+1)*min(heights[l1:r1])`。虽然有`r1-l1+1<r0-l0`，但是有`heights[l1:r1]>heights[l0]`，所以构造出比`S1`和`S2`更大的矩形是完全有可能的。
基于上面的思考，我们容易得到一种思路：我们枚举所有可能的左端点`l`，然后移动右端点`r`，直到找到第一个点使得`heights[r]<heights[l]`，这时便得到了固定左端点`l`的情况下能构造出的最大的矩形面积。我们在枚举`l`的过程中，不断维护计算得到的矩形面积的最大值，直至枚举完所有的`l`。
那么如何寻找找到第一个使得`heights[r]<heights[l]`的点呢？一个简单的想法是，每次在`r=l+1`处释放指针，然后不断向右移动`r`，遇到了满足`heights[r]<heights[l]`的点之后就计算此时最大的矩形面积（具体还是要在`S1`和`S2`的表达式中选一个较大的）。计算完成之后，向右移动`l`，然后重复上述步骤。这个算法的时间复杂度近似为$O(n^2)$，按出题人的“尿性”，这种复杂度显然太高了。
很显然，上面的步骤中有许多不必要的遍历：比如在`[l0,r0-1]`上，数组`heights`保持递增，那么在遍历到`heights[r]<heights[l0]`时，实际上我们也能**一同**确定，当`l`取在`[l0+1,r0-1]`上时，也有`heights[r]<heights[l]`。但是按照上面的朴素算法，我们在第一次得知`heights[r]<heights[l0]`时，我们并不能一同确定“当`l`取在`[l0+1,r0-1]`上时，也有`heights[r]<heights[l]`”。这显然是一个可以优化和改进的点。
如何在遍历的过程中利用这种“局部递增”的性质呢？结合本题在“栈”中的归类，我们便容易联想到单调栈。我们可以遍历数组`heights`元素，当数组为空或当前元素大于等于栈顶元素时，就将该元素入栈（实际上应该将该元素的下标入栈，因为通过下标能得知元素值，而且下标还能额外记录该元素的位置信息，方便计算矩形面积）；而当当前元素小于栈顶元素时，这预示着我们找到了固定栈顶元素为左端点的情况下，遇到的第一个小于左端点值的点，此时便可以计算构造的矩形面积最大值，随后我们可以弹出该栈顶元素，然后继续重复上述判断，直到栈为空或当前元素大于等于栈顶元素，这时再将该元素入栈。这样，利用单调栈这种额外的空间，我们可以将原来朴素算法的时间复杂度从$O(n^2)$降低到$O(n)$。
当然最后我们还有一些边界情况需要处理，那就是很有可能在遍历完成之后，栈是非空的。这意味着数组`heights`的末端有一段是局部递增的，那这很简单，我们只需要逐个计算以栈中元素为左端点、以数组末端为右端点所构成的矩形面积即可。过程中我们不断维护遭遇到的面积最大值，执行完上述过程后，相当于枚举了所有的左端点`l`下所能遇到的矩形面积最大值，然后记录了这些值中的最大值。
基于上面这些思考，我们很容易写出下面的程序：
```cpp
class Solution {
public:
    int largestRectangleArea(vector<int>& heights) {
        int ans = INT_MIN;
        stack<int> stk; // 单调栈，存储元素的下标
        for (int i = 0; i < heights.size(); i++) {
            while (!stk.empty() && heights[i] < heights[stk.top()]) {
                // 遇到了栈顶元素为左端点时，第一个可以构造的右矩形
                // 计算S1和S2，并维护遇到的面积最大值
                int S1 = (i - stk.top() + 1) * heights[i];
                int S2 = (i - stk.top()) * heights[stk.top()];
                ans = max(ans, max(S1, S2));

                // 弹出当前栈顶元素，继续判断
                stk.pop();
            }
            stk.push(i);
        }
        // 处理边界情况
        while (!stk.empty()) {
            int S = (heights.size() - stk.top()) * heights[stk.top()];
            ans = max(ans, S);
            stk.pop();
        }
        return ans;
    }
};
```
但是很遗憾，上面的程序是不对的。在运行样例`[2,1,2]`时，所能构造的矩形最大面积为3，但上面的程序返回的是2。问题其实也很容易查明，我们上面在尝试构造矩形的时候只尝试了以当前遍历到的数组元素**为左端点**，然后**向右侧扩张**这一种构造方式。实际上，我们完全可以尝试以当前遍历到的数组元素为矩形的“高”，**向左和向右扩张**，直到在两侧找到第一个小于当前“高”的元素。
所以，以某一个“柱子”为矩形的高，所能构造的最大矩形的面积，与其向左和向右遇到的第一个小于它高度的“柱子”有关。接下来改进算法的思路也将围绕这一点进行。我们进行三次遍历，前两次遍历分别以正向和逆向遍历`heights`，通过单调栈确定某一位置向左和向右遇到的第一个大于它的”柱子“的位置，并将其记录在数组中；第三次遍历分别以上面记录的数据，构造出以当前柱子为高、以数组中记录的数据为左端点和右端点（实际上要各自偏移一位）的最大值的矩形，过程中不断维护遇到的矩形面积最大值。这样就可以完成问题了：
```cpp
class Solution {
public:
    int largestRectangleArea(vector<int>& heights) {
        // 首先正向和反向遍历数组，确定当前位置向左和向右遇到的第一个小于它本身的值
        int l = heights.size();
        stack<int> stk;
        vector<int> right_min(l, l - 1); // 向右能遇到的值最大的值，注意这里填入的初始值
        for (int i = 0; i < l; i++) {
            while (!stk.empty() && heights[i] < heights[stk.top()]) {
                right_min[stk.top()] = i - 1; // 注意这里应当偏移一位
                stk.pop();
            }
            stk.push(i);
        }
        while (!stk.empty())
            stk.pop(); // 清空栈
        vector<int> left_min(l, 0); // 向左能遇到的值最大的值，注意这里填入的初始值
        for (int i = l - 1; i >= 0; i--) {
            while (!stk.empty() && heights[i] < heights[stk.top()]) {
                left_min[stk.top()] = i + 1; // 注意这里应当偏移一位
                stk.pop();
            }
            stk.push(i);
        }

        // 再次遍历整个数组，枚举以当前元素为高的最大矩形面积
        int ans = INT_MIN;
        for (int i = 0; i < l; i++) {
            int s = heights[i] * (right_min[i] - left_min[i] + 1);
            ans = max(ans, s);
        }
        return ans;
    }
};
```
容易分析，上面算法的时间复杂度为$O(3n)=O(n)$，空间复杂度为$O(3n)=O(n)$。
然而最终，这段程序的运行时间为43ms，只击败了11.2%的人。这说明这个算法还有一定的优化空间。
一个简单的直觉是，上面使用了两次遍历来确定某一“柱子”向左和向右遇到的第一个小于它的值，有没有办法在一次遍历中就把这两个值求出来呢？现在假设我们原本在向右遍历，寻找右侧第一个小于当前“柱子”的值，有没有办法在这个遍历的过程中一起把左侧第一个的小于当前“柱子”的值求出来呢？
其实是可以的。不难注意到，当我们向右遍历的时候，单调栈实际上会记录数组`heights`中具有局部递增性质的片段，也就是局部右侧大于左侧的片段。不难发现，当我们发现遍历时发现了这种局部递增性质的时候，当前“柱子”向左遇到的第一个小于它的元素一定是栈中的上一个元素。也就是说，当我们准备将某一个柱子的位置压入栈时，压入之前的栈顶元素恰好就是当前要被压入栈的“柱子”向左遇到的第一个小于它的值。这样我们便可以在一次遍历中填充起`left_min`和`right_min`这两个数组。
```cpp
class Solution {
public:
    int largestRectangleArea(vector<int>& heights) {
        // 遍历数组，确定当前位置向左和向右遇到的第一个小于它本身的值
        int l = heights.size();
        stack<int> stk;
        vector<int> left_min(l, 0); // 向左能遇到的值最大的值，注意这里填入的初始值
        vector<int> right_min(l, l - 1); // 向右能遇到的值最大的值，注意这里填入的初始值
        for (int i = 0; i < l; i++) {
            while (!stk.empty() && heights[i] < heights[stk.top()]) {
                right_min[stk.top()] = i - 1; // 注意这里应当偏移一位
                stk.pop();
            }
            left_min[i] = stk.empty() ? 0 : stk.top() + 1;
            stk.push(i);
        }

        // 再次遍历整个数组，枚举以当前元素为高的最大矩形面积
        int ans = INT_MIN;
        for (int i = 0; i < l; i++) {
            int s = heights[i] * (right_min[i] - left_min[i] + 1);
            ans = max(ans, s);
        }
        return ans;
    }
};
```
这次的运行时间提升到了18ms，但是仍然只击败了79.18%的人。
写到这里其实还能发现，我们甚至可以把第三次遍历也整合到这次的遍历中来，因为第三次遍历时的遍历方向也是与上面的遍历同向的。我们不难注意到，在我们试图给`right_min[stk.top()]`赋值的时候，实际上我们也已经知道了`left_min[stk.top()]`的值，因此我们可以算出`stk.top()`位置的矩形面积。过程中我们不断维护遇到的矩形面积最大值即可。
这个优化非常关键，因为这成功将算法的空间消耗从$O(3n)$降低到了$O(n)$——虽然仍然在$O(n)$的级别上。
```cpp
class Solution {
public:
    int largestRectangleArea(vector<int>& heights) {
        stack<int> stk;
        int ans = 0;
        heights.push_back(0); // 该行等价于在遍历完数组后处理栈中剩余的元素
        for (int r = 0; r < heights.size(); r++) { // right_min
            while (!stk.empty() && heights[r] < heights[stk.top()]) {
                int cur = stk.top(); // 我们要计算以该“柱子”为高的矩形面积
                stk.pop();
                int l = stk.empty() ? 0 : stk.top() + 1; // left_min
                ans = max(ans, (r - l) * heights[cur]); // 注意这里矩形面积的计算方法变了
            }
            stk.push(r);
        }
        return ans;
    }
};
```
然而如此精简的程序提交后运行时间反而提升到了19ms，也只击败了77.72%的人。不平。
