给你一个整数数组 nums ，找到其中最长严格递增子序列的长度。

子序列 是由数组派生而来的序列，删除（或不删除）数组中的元素而不改变其余元素的顺序。例如，[3,6,2,7] 是数组 [0,3,1,6,2,2,7] 的子序列。

进阶：你能将算法的时间复杂度降低到 O(n log(n)) 吗?

---
仍然是动态规划的思路。考虑建立动态规划数组`dp[i]`，表示`nums[0]`到`nums[i]`至此的子数组中，最长的递增子序列。`dp[i]`可以由其前方的若干个状态转移而来，如果有`nums[i]>nums[j]`，那么显然`dp[i]`便可以由`dp[j]`转移而来，即直接令`dp[i]=dp[j]+1`。但是，我们倾向于使用最长的递增子序列，因此实际上`dp[i]`需要在若干个满足条件的`dp[j]`中选一个最大的再加一。
基于此，我们可以写出下面的程序，其时间复杂度为$O(n^2)$。
```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        if (nums.size() == 1)
            return 1;

        vector<int> dp(nums.size());
        dp[0] = 1;
        for (int i = 1; i < nums.size(); i++) {
            int max_val = 0;
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i])
                    max_val = max(max_val, dp[j]);
            }
            dp[i] = max_val + 1;
        }
        return dp[nums.size() - 1];
    }
};
```
然而该程序最终无法通过全部用例，在如下的用例上“掉”了下来：
```text
[1,3,6,7,9,4,10,5,6]
```
正确的输出应当是6，但该程序输出为5。经检查可以发现，我们似乎默认在最后一个位置上一定会产生最长的子数组。这实际上是不正确的。如果数组的最大值提前出现了（就像上面的用例中那样），那么该位置很有可能就会取到整个数组的最长子序列。其后的元素，由于其值不大于数组中的最大值，因此即使能构成合理的子序列，也未必能满足“最长”的性质。
解决方案是，最终输出时不采用`dp`数组的最后一位，而是取整个`dp`数组的最大值来输出。
```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        if (nums.size() == 1)
            return 1;

        vector<int> dp(nums.size());
        dp[0] = 1;
        for (int i = 1; i < nums.size(); i++) {
            int max_val = 0;
            for (int j = 0; j < i; j++) {
                if (nums[j] < nums[i])
                    max_val = max(max_val, dp[j]);
            }
            dp[i] = max_val + 1;
        }
        return *max_element(dp.begin(), dp.end());
    }
};
```
这样的修改就AC了。该程序的时间复杂度为$O(n^2)$，空间复杂度为$O(n)$。
本题提示还有$O(n\log n)$的做法，很容易就能想到这个$\log n$应该是把上面程序中遍历小于`i`的部分的数组改成了二分搜索。
那这个二分搜索该如何实现呢？我尝试了一些思路，都达不到效果，最后只能诉诸[官方题解](https://leetcode.cn/problems/longest-increasing-subsequence/solutions/147667/zui-chang-shang-sheng-zi-xu-lie-by-leetcode-soluti/)了：
> 考虑一个简单的贪心，如果我们要使上升子序列尽可能的长，则我们需要**让序列上升得尽可能慢**，因此我们希望**每次在上升子序列最后加上的那个数尽可能的小**。
> 基于上面的贪心思路，我们维护一个数组`d[i]`，表示长度为`i`的最长上升子序列的末尾元素的最小值，用`len`记录目前最长上升子序列的长度。起始时`len`为1，`d[1]=nums[0]`。
> 同时我们可以注意到`d[i]`是关于`i`单调递增的。因为如果`d[j]>=d[i]`且`j<i`，我们考虑从长度为`i`的最长上升子序列的末尾删除`i-j`个元素，那么这个序列长度变为`j`，且第`j`个元素（末尾元素）必然小于`d[i]`，也就小于`d[j]`。那么我们就找到了一个长度为`j`的最长上升子序列，并且末尾元素比`d[i]`小，从而产生了矛盾。因此数组`d`的单调性得证。
> 我们依次遍历数组`nums`中的每个元素，并更新数组`d`和`len`的值。如果`nums[i]>d[len]`则更新`len=len+1`，否则在`d[1..len]`中找满足`d[i-1]<nums[j]<d[i]`的下标`i`，并更新`d[i]=nums[j]`。根据d数组的单调性，我们可以使用二分查找寻找下标`i`，优化时间复杂度。
> 最后整个算法流程为：
> - 设当前已求出的最长上升子序列的长度为`len`(初始时为1)，从前往后遍历数组`nums`，在遍历到`nums[i]`时：
> 	- 如果`nums[i]>d[len]`，则直接加入到`d`数组末尾，并更新`len=len+1`;
> 	- 否则，在`d`数组中二分查找，找到第一个比`nums[i]`小的数`d[k]`，并更新`d[k+1]=nums[i]`。

可以看出，这里改进之后，程序的思路跟一开始的动态规划方法已经没什么联系了。
评论区有一句比喻，把上面这个方法的思路说的很形象：
> 一个新员工一个老员工价值相当，老员工就可以走了，因为新员工被榨取的剩余空间更多。

这个比喻主要说的就是`d`数组的维护过程。我们相信这个数组中的最后一个一定是“上升速度最慢”的。如果我们确实遇到了一个上升速度“更慢”的，就把之前那个元素“开除”。当然要注意的是，当初跟着这个现在被“开除”的“员工”进来的“员工”，也会有被“开除”掉的风险，只要遇到了上升更慢且更长的子数组，总会有新来的“员工”把“空间不够”的老员工顶替掉。
而这个`len`数组最巧妙的一点在于，它在程序中只能增加而不能减少，相当于它的值记录了`d`数组所能到达的最大值，起到了动态规划方法的程序中`max_element`的作用。
如果实在不懂的话，可以参考这篇[博文](https://writings.sh/post/longest-increasing-subsequence-revisited)。
写出的程序如下：
```cpp
class Solution {
public:
    int lengthOfLIS(vector<int>& nums) {
        int len = 1, n = (int)nums.size();
        if (n == 0) {
            return 0;
        }
        vector<int> d(n + 1, 0);
        d[len] = nums[0];
        for (int i = 1; i < n; ++i) {
            if (nums[i] > d[len]) {
                d[++len] = nums[i];
            } else {
                int l = 1, r = len, pos = 0; // 如果找不到说明所有的数都比 nums[i] 大，此时要更新 d[1]，所以这里将 pos 设为 0
                while (l <= r) {
                    int mid = (l + r) >> 1;
                    if (d[mid] < nums[i]) {
                        pos = mid;
                        l = mid + 1;
                    } else {
                        r = mid - 1;
                    }
                }
                d[pos + 1] = nums[i];
            }
        }
        return len;
    }
};
```
该程序的时间复杂度为$O(n\log n)$，空间复杂度为$O(n)$。
