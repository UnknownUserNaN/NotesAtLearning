你这个学期必须选修 numCourses 门课程，记为 0 到 numCourses - 1 。

在选修某些课程之前需要一些先修课程。 先修课程按数组 prerequisites 给出，其中 prerequisites[i] = [ai, bi] ，表示如果要学习课程 ai 则 必须 先学习课程  bi 。

例如，先修课程对 [0, 1] 表示：想要学习课程 0 ，你需要先完成课程 1 。
请你判断是否可能完成所有课程的学习？如果可以，返回 true ；否则，返回 false 。

提示： prerequisites[i] 中的所有课程对 互不相同

---
按照图问题的一般思路，可以想到，这个问题实际上是在问我们能否在一个有向图中找到一条路径，使我们能够走过所有的结点。
既然是“能否”的问题，那么自然是使用深度优先更好。我们可以构造一个深度优先遍历的函数，为其输入一条由结点确定的路径，然后该函数尝试通过`prerequisites`中的信息，尝试从上游或者从下游延长这条路径。这个延长路径的过程可以参考回溯法完成。如果某一次遍历过程中，我们成功找到了一条长度等于结点数目的路径，便意味着我们找到了问题的解。
只不过可以预见的是，这个算法的时间复杂度可能有点高。还是先尝试把它写出来吧。结果发现在某些情境下，根本无法运行起来：

class Solution {
public:
    bool dfs(deque<int>& path, vector<vector<int>>& info, int numCourses) {
        if (path.size() == numCourses)
            return true;

        // 根据info中的信息延长path中的结点
        // 先寻找可以在path前面的结点和path后面的结点
        vector<int> prev, next;
        prev.push_back(-1);
        next.push_back(-1);
        for (auto each : info) {
            if (each[0] == path.front())
                prev.push_back(each[1]);
            else if (each[1] == path.back())
                next.push_back(each[0]);
        }

        // 尝试从上游和下游延长路径
        for (int p : prev) {
            for (int n : next) {
                if (p != -1)
                    path.push_front(p);
                if (n != -1)
                    path.push_back(n);
                if (p != -1 && n != -1) {
                    bool ret = dfs(path, info, numCourses);
                    if (ret)
                        return true;
                }
                if (p != -1)
                    path.pop_front();
                if (n != -1)
                    path.pop_back();
            }
        }
        return false;
    }

    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        deque<int> path = {0};
        return dfs(path, prerequisites, numCourses);
    }
};

官方题解首先明确了这个题目的本质：
> 本题是一道经典的「拓扑排序」问题。
> 给定一个包含n个节点的有向图G，我们给出它的节点编号的一种排列，如果满足：
> 	对于图G中的任意一条有向边(u,v)，u在排列中都出现在v的前面。
> 那么称该排列是图G的「拓扑排序」。根据上述的定义，我们可以得出两个结论：
> - 如果图G中存在环（即图G不是「有向无环图」），那么图G不存在拓扑排序。这是因为假设图中存在环c1,c2,...,cn,c1，那么c1在排列中必须出现在cn的前面，但cn同时也必须出现在c1的前面，因此不存在一个满足要求的排列，也就不存在拓扑排序；
> - 如果图G是有向无环图，那么它的拓扑排序可能不止一种。举一个最极端的例子，如果图G值包含n个节点却没有任何边，那么任意一种编号的排列都可以作为拓扑排序。
> 有了上述的简单分析，我们就可以将本题建模成一个求拓扑排序的问题了。

实际上到这里就可以发现，本题实际上并不需要找出一条贯穿全部结点的路径，而是一个比“路径”限定更为宽松的条件——拓扑排序。该排序必须在有向图中不存在环时才可进行，所以该算法也可以用来判断一个图中是否存在环。
那么如何进行拓扑排序呢？下面参考了[B站视频](https://www.bilibili.com/video/BV1Vh56zsE37/)，它给出了一套简单的流程。注意，下图中的“AOV网”可以通俗的理解为，结点代表一个事件的有向图；”入度“可以通俗地理解为指向一个结点的”箭头“的数量。
![[Pasted image 20250504005037.png]]
拓扑排序的核心在于“递归/循环”和“选择入度为0的点输出”。我们接下来要实现拓扑排序，实际上就可以按照上面的步骤来实现。
> 我们使用一个队列来进行广度优先搜索。初始时，所有入度为0的节点都被放入队列中，它们就是可以作为拓扑排序最前面的节点，并且它们之间的相对顺序是无关紧要的。
> 在广度优先搜索的每一步中，我们取出队首的节点u:
> - 我们将ｕ放入答案中;
> - 我们移除ｕ的所有出边，也就是将ｕ的所有相邻节点的入度减少1。如果某个相邻节点v的入度变为0，那么我们就将放入队列中。
> 在广度优先搜索的过程结束后。如果答案中包含了这n个节点，那么我们就找到了一种拓扑排序，否则说明图中存在环，也就不存在拓扑排序了。

官方题解指出，这实际上是基于广度优先思路的一种拓扑排序方法：
```cpp
class Solution {
private:
    vector<vector<int>> edges;
    vector<int> indeg;

public:
    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
	    // 遍历整个图
        edges.resize(numCourses); // 用于存储每个结点指向的结点
        indeg.resize(numCourses); // 用于储存每个结点的入度
        for (const auto& info: prerequisites) {
            edges[info[1]].push_back(info[0]);
            ++indeg[info[0]];
        }

        queue<int> q; // q中存储要操作的入度为0的点
        for (int i = 0; i < numCourses; ++i) {
            if (indeg[i] == 0) {
                q.push(i); // 选取入度为0的点加入结果（因为这种选择最没“风险”）
            }
        }

        int visited = 0;
        while (!q.empty()) { 
            ++visited;
            int u = q.front();
            q.pop(); // 如果要输出一种拓扑排序，就应在在此结点弹出队列前放入结果数组
            for (int v: edges[u]) {
                --indeg[v]; // 移除其后继点的入度
                if (indeg[v] == 0) {
                    q.push(v); // 增加新的待处理的入度为0的结点
                }
            }
        }

        return visited == numCourses;
    }
};
```
广度优先算法的思路非常直观。而官方题解还给出了一种深度优先的做法，思路看起来就没那么直观了。当然，其核心仍然是每次选取入度为0的点“输出”，如果已输出的点未能达到课程数目，那就说明该图中存在环。深度优先算法的特别之处在于，注意到了我们按着深度遍历的要求遍历结点时走过的路径，实际上就是符合拓扑排序的。我们可以在进行深度遍历的时候，“按层次”地将这些遍历的路径按前后关系”整合“进栈里（因为栈先进后出的结构能够反转箭头关系）。
只不过需要注意的是，该深度遍历的方法需要随意确定一个结点开始，而这个随意确定的结点不一定能够深度遍历完”全图“，因此我们还要在深度遍历的过程中记录已经”走过“的结点，如果遍历结束后，”走过“的结点数目小于总结点数，那我们还要在未遍历过的结点中再选择一个继续开始深度遍历。
官方题解对此算法的步骤分解得比较彻底：
> 对于图中的任意一个节点，它在搜索的过程中有三种状态，即：
> - 「未搜索」：我们还没有搜索到这个节点;
> - 「搜索中」：我们搜索过这个节点，但还没有回溯到该节点，即该节点还没有入栈，还有相邻的节点没有搜索完成;
> - 「已完成」：我们搜索过并且回溯过这个节点，即该节点已经入栈，并且所有该节点的相邻节点都出现在栈的更底部的位置，满足拓扑排序的要求。
> 通过上述的三种状态，我们就可以给出使用深度优先搜索得到拓扑排序的算法流程，在每一轮的搜索搜索开始时，我们任取一个「未搜索」的节点开始进行深度优先搜索。
> - ·我们将当前搜索的节点ｕ标记为「搜索中」，遍历该节点的每一个相邻节点v：
> 	- 如果为「未搜索」，那么我们开始搜索υ，待搜索完成回溯到ｕ;
> 	- 如果为「搜索中」，那么我们就找到了图中的一个环，因此是不存在拓扑排序的；
> 	- 如果v为「已完成」，那么说明已经在栈中了，而u还不在栈中，因此u无论何时入栈都不会影响到(u,v)之前的拓扑关系，以及不用进行任何操作。
> - ·当u的所有相邻节点都为「已完成」时，我们将ｕ放入栈中，并将其标记为「已完成」。
> 在整个深度优先搜索的过程结束后，如果我们没有找到图中的环，那么栈中存储这所有的n个节点，从栈顶到栈底的顺序即为一种拓扑排序。

写出的程序如下：
```cpp
class Solution {
private:
    vector<vector<int>> edges; // 以邻接链表的形式存储各结点指向的新结点
    vector<int> visited; // 记录已经“走过”的结点
    bool valid = true; // 指示该图中是否有环

public:
    void dfs(int u) {
        visited[u] = 1; // 将对应结点设定为正在遍历
        for (int v: edges[u]) {
            if (visited[v] == 0) { // 如果该结点指向的结点未被遍历过，则进一步进行深度遍历
                dfs(v);
                if (!valid) {
                    return;
                }
            }
            else if (visited[v] == 1) { // 遍历的过程中指回了路径自身，显然有环
                valid = false; // 该图不合理
                return;
            }
        }
        visited[u] = 2; // 将对应结点设定为已遍历过
    }

    bool canFinish(int numCourses, vector<vector<int>>& prerequisites) {
        edges.resize(numCourses);
        visited.resize(numCourses);
        for (const auto& info: prerequisites) {
            edges[info[1]].push_back(info[0]); // 根据题目条件存储图中各边
        }
        for (int i = 0; i < numCourses && valid; ++i) { // 防止某一节点无法遍历完全图，所以要保证每个结点都要做过一次DFS
            if (!visited[i]) {
                dfs(i);
            }
        }
        return valid; // 该图只要遍历完成后不存在环，即表示该图存在拓扑排序
    }
};
```
假若该图有$n$个结点、$m$条边，不难发现上面的算法都“至少”要遍历一次各个结点和各条边，因此上述两种算法的时间复杂度为$O(m+n)$；空间复杂度更好分析，因为分别使用了“邻接链表”存储了各条边和各个结点的状态，因此空间复杂度为$O(m+n)$。
