给你一个整数数组 `nums` ，请你找出一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

**子数组**是数组中的一个连续部分。

\*\*进阶：\*\*如果你已经实现复杂度为 `O(n)` 的解法，尝试使用更为精妙的 **分治法** 求解。

***

子串问题都可以考虑用滑动窗口的模型往上套。本题的“进阶”也提示了本文存在`$O(n)$`的解法，那么本题最优的解法肯定是只需要遍历一遍数组的。只需遍历一遍数组的方法往往都需要考虑双指针，也就是左右指针。

既然要使用左右指针，那么显然就要考虑如何移动左右指针。这里可以考虑类似贪心算法的思路，每一步都只选择目前的最优解，也就是，哪个指针能使当前窗口的值增大就移动哪一个指针。除非右指针与左指针重合了，这时候必须要移动右指针了。

调试了半天之后，终于写出了下面的程序：

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        // 设置左右指针
        int left = 0, right = 1;
        int sum = nums[0]; // 覆盖的应当是下标为[left, right-1]的总和
        int max_sum = nums[0];

        while (right <= nums.size()) {
            // 根据情况移动左右指针
            if (right >= nums.size()) {
                // 右指针已经移动到数组最末尾，这时只能移动左指针
                while (left < right - 1) {
                    sum -= nums[left];
                    left++;
					max_sum = max(max_sum, sum);
                }
                break;
            } else if (left >= right - 1) {
                // 左指针与右指针重合甚至已经越过，这时必须要移动右指针
                sum += nums[right];
                right++;
            } else {
                // 两指针均有空间可以移动，则看哪个移动之后可以最可能让数组增大
                if (nums[right] > -nums[left]) {
                    // 右指针对数组增大影响更好
                    sum += nums[right];
                    right++;
                } else {
                    // 左指针对数组增大影响更好
                    sum -= nums[left];
                    left++;
                }
            }

            // 更新最大的和
            max_sum = max(max_sum, sum);
        }

        return max_sum;
    }
};
```

这条程序能通过绝大部分的测试用例，但可惜并不能通过全部。这是因为上面基于贪心的算法还是有些问题的。比如官方给的如下的测试用例：

`[1,2,-1,-2,2,1,-2,1,4,-5,4]`

该用例中，取得最大值的窗口为`[2,1,-2,1,4]`，此时和为6；但根据上述的程序，最终窗口只会在`[1,4]`处取得最大值，此时和为5。这启发我们，按照偏贪心的想法是无法遍历到所有最大值所在的区间的。

没办法，只能换一个思路。这里又想起了之前的前缀和公式，这个公式在此类连续子数组求值时非常好用。先将整个数组依次求前缀和，那么题目要求的找到和最大的子数组，就转化成在前缀和数组中找到两个值，使他们的差最大。最朴素的想法自然是直接去求该前缀和数组的极差，即最大值和最小值的差，但这个差并不总是符合题意的，因为如果最大值出现在最小值之前，那么按照本题的情景，这个极差实际上是取不到的。

这里又想到了之前的一个思路，遍历前缀和数组中的每个数，设法找到其后数组的最大值，然后记录这个差值，在遍历的过程中不断维护这个差值的最大值。不过，找到其后的最大值是比较困难的，但是，遍历每个数组，设法找到其后遍历过的极小值是容易的。因此，我们可以改为遍历前缀和数组中的每个数，对每个数都设法找到其之前的最小值，使其两者的差尽可能地达到最大。

基于上面的思路，就可以写出如下的程序：

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        // 特殊情况
        if (nums.size() == 1)
            return nums[0];

        // 求前缀和，找到两个值y1<y2，使得其差y2-y1最大
        int cumsum = 0;     // 前缀和
        int min_cumsum = 0; // 过程中遇到的前缀和最小值
        int max_diff = INT_MIN;
        for (int i = 0; i < nums.size(); i++) {
            cumsum += nums[i];
            if (i == 0)
                max_diff = cumsum;
            else
                max_diff = max(max_diff, cumsum - min_cumsum);
            min_cumsum = min(min_cumsum, cumsum);
        }

        return max_diff;
    }
};
```

果然思路领先之后就是降维打击了，这次的程序经过几次debug后，直接AC了。

但是看了官方题解之后，惊讶地发现官方题解的思路与我的不同，官方题解使用了**动态规划**的做法。由于这个做法太过严谨，这里直接把官方题解的思路和代码贴在这里：

> 假设 nums 数组的长度是 n，下标从 0 到 n−1。
>
> 我们用 f(i) 代表以第 i 个数结尾的「连续子数组的最大和」，那么很显然我们要求的答案就是：
>
> ```math
> \displaystyle \max_{0\leq i\leq n-1} f(i)
> ```
>
> 因此我们只需要求出每个位置的 f(i)，然后返回 f 数组中的最大值即可。那么我们如何求 f(i) 呢？我们可以考虑 nums\[i] 单独成为一段还是加入 f(i−1) 对应的那一段，这取决于 nums\[i] 和 f(i−1)+nums\[i] 的大小，我们希望获得一个比较大的，于是可以写出这样的动态规划转移方程：
>
> ```math
> f(i)=\max\left\{f(i-1)+nums[i],nums[i] \right\}
> ```
>
> 不难给出一个时间复杂度 O(n)、空间复杂度 O(n) 的实现，即用一个 f 数组来保存 f(i) 的值，用一个循环求出所有 f(i)。考虑到 f(i) 只和 f(i−1) 相关，于是我们可以只用一个变量 pre 来维护对于当前 f(i) 的 f(i−1) 的值是多少，从而让空间复杂度降低到 O(1)，这有点类似「滚动数组」的思想。

基于此，可以得到下面的程序：

```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int pre = 0, maxAns = nums[0];
        for (const auto &x: nums) {
            pre = max(pre + x, x);
            maxAns = max(maxAns, pre);
        }
        return maxAns;
    }
};
```

实际上，观察最后写出的程序，实际上官方题解的做法跟前缀和的思路写出的程序是高度相似的。

值得一提的是，官方题解还提供了本题采用分治法优化的思路。但由于此题的分治法要讲述的内容实在太多，这里不再赘述了。

总结一遍做题的过程，可以发现，这种连续子数组的问题，考虑**前缀和公式**会非常大程度地简化运算。
