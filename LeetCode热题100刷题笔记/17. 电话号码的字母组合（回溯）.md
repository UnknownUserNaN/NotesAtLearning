给定一个仅包含数字 2-9 的字符串，返回所有它能表示的字母组合。答案可以按 任意顺序 返回。

给出数字到字母的映射如下（与电话按键相同）。注意 1 不对应任何字母。

提示：digits[i] 是范围 ['2', '9'] 的一个数字。

---
按照回溯的一般算法，这个问题依旧很简单。设想一个深度为输入字符串digits深度的树，每个深度为$i$的结点表示该字符串中对应$i$位置的数字所可能对应的字符。回溯法相当于对该树进行了深度遍历，当遍历到叶子结点时，即将组成的结果加入到结果数组中。
很容易写出下面的程序：
```cpp
class Solution {
public:
    unordered_map<char, vector<char>> dict = {
        {'2', {'a', 'b', 'c'}}, {'3', {'d', 'e', 'f'}},
        {'4', {'g', 'h', 'i'}}, {'5', {'j', 'k', 'l'}},
        {'6', {'m', 'n', 'o'}}, {'7', {'p', 'q', 'r', 's'}},
        {'8', {'t', 'u', 'v'}}, {'9', {'w', 'x', 'y', 'z'}}};

    void backtracking(vector<string>& ret, string& digits, string& s) {
        if (s.size() == digits.size()) {
            // 达到目标长度，即将其添入结果数组
            ret.push_back(s);
            return;
        }

        for (int i = 0; i < dict[digits[s.size()]].size(); i++) {
            s.push_back(dict[digits[s.size()]][i]);
            backtracking(ret, digits, s);
            s.pop_back();
        }
    }

    vector<string> letterCombinations(string digits) {
        if (digits.size() == 0)
            return {};
        
        vector<string> ret;
        string s = "";
        backtracking(ret, digits, s);
        return ret;
    }
};
```
该算法的时间复杂度与该树的结点数目有关系，不难分析出其时间复杂度大致为$O(3^n)$，$n$为输入的字符串长度。额外空间复杂度大致为$O(n)$。当然更准确的分析是，分别考虑字符串中数字7和9的个数，和非上述两者的数字的个数，然后分别计算这两者对应的结点个数。
