给你一个整数数组 `nums`，返回 数组 `answer` ，其中 `answer[i]` 等于 `nums` 中除 `nums[i]` 之外其余各元素的乘积 。

题目数据 **保证** 数组 `nums`之中任意元素的全部前缀元素和后缀的乘积都在  **32 位** 整数范围内。

请 \*\*不要使用除法，\*\*且在 `O(n)` 时间复杂度内完成此题。

\*\*进阶：\*\*你可以在 `O(1)` 的额外空间复杂度内完成这个题目吗？（ 出于对空间复杂度分析的目的，输出数组 \*\*不被视为 \*\*额外空间。）

***

此题的主要难点在于，本题不能使用除法，否则此题会非常简单。

查看本题的提示，发现提示中提到了前缀“积”和后缀“积”。基于这个提示就很容易想到，本题要求的除自身以外的数组的乘积，实际上就是当前元素的前缀积和后缀积的积。因此，可以分别求出该数组的前缀积和后缀积，最后按下标依次相乘即可。

```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        // 分别正向和后向遍历数组，求前缀积和后缀积
        vector<int> prefix_prod(nums.size());
        prefix_prod[0] = 1;
        for (int i = 1; i < nums.size(); i++)
            prefix_prod[i] = prefix_prod[i - 1] * nums[i - 1];

        vector<int> suffix_prod(nums.size());
        suffix_prod[nums.size() - 1] = 1;
        for (int i = nums.size() - 2; i >= 0; i--)
            suffix_prod[i] = suffix_prod[i + 1] * nums[i + 1];

        // 基于前缀积和后缀积得到结果
        vector<int> ans(nums.size());
        for (int i = 0; i < nums.size(); i++)
            ans[i] = prefix_prod[i] * suffix_prod[i];

        return ans;
    }
};
```

这样的思路虽然很简单，但是也不难发现这个算法的空间复杂度是`$O(2n)=O(n)$`，显然达不到算法要求的`$O(1)$`的复杂度。因此算法还有更多的可以优化的空间。

既然题目说输出的数组不被视为额外空间，那么我们可以带有一些“耍赖”性质地，将输出数组提前声明出来，来作为额外的空间。

```cpp
class Solution {
public:
    vector<int> productExceptSelf(vector<int>& nums) {
        vector<int> ans(nums.size());
        ans[0] = 1;
        for (int i = 1; i < nums.size(); i++)
            ans[i] = ans[i - 1] * nums[i - 1];

        int suffix_prod = 1;
        for (int i = nums.size() - 1; i >= 0; i--) {
            ans[i] = ans[i] * suffix_prod;
            suffix_prod *= nums[i];
        }

        return ans;
    }
};
```

这样就可以将算法的空间复杂度也压到`$O(1)$`了。事实上，这也是官方题解的思路。

值得一提的是，在官方题解的评论区里，有人给出了下面基于双指针思路的一种做法：

```cpp
vector<int> productExceptSelf(vector<int>& nums) {
    vector<int> answer(nums.size(), 1);
    int left = 0, right = nums.size() - 1;
    int lp = 1, rp = 1;
    while (right >= 0 && left < nums.size()) {
        answer[right] *= rp;
        answer[left] *= lp;
        lp *= nums[left++];
        rp *= nums[right--];
    }
    return answer;
}
```

其实本质上也是跟前面求前缀积和后缀积的思路一样，只不过更为“优雅”一些，输出数组不被“耍赖”地用来储存计算的中间结果。

上面全部的思考过程提示我们，遇到**数组求和、求积**之类的问题，将问题转化到**前缀和（积）和后缀和（积）**的求取上，问题往往会变得容易一些。
