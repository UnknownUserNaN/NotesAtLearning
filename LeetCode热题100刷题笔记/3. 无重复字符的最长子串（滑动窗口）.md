给定一个字符串 `s` ，请你找出其中不含有重复字符的 \*\*最长 子串 \*\*的长度。

***

由于这道题归在了”滑动窗口“这一类的问题之下，因此这算是给此类问题的一个较强的提示。

这里直接使用左指针和右指针表示滑动窗口，滑动窗口移动的方式按以下方式进行：

1.当当前窗口中有重复字符时，则左指针和右指针都向右移动一位，在保持当前窗口长度不变的情况下移动窗口，试图让窗口中没有重复字符。

2.当当前窗口中没有重复字符时，则只有右指针向右移动一位，让当前窗口中包含更多的字符。

不断重复执行上述逻辑，直至右指针指到字符串末尾为止。

基于上述思路，我第一次通过字符串子串的方式编写的代码如下：

```cpp
class Solution {
public:
    bool containRepeat(string s) {
        set<char> chars;

        // 遍历字符串中的每个字符，如果在集合中遇见过则立刻返回true
        for (char c : s) {
            if (chars.find(c) != chars.end()) {
                return true;
            }
            chars.insert(c);
        }
        return false;
    }

    int lengthOfLongestSubstring(string s) {
        if(s.empty()) return 0;

        // 设置滑动窗口，使用左指针和右指针表达
        int left = 0, right = 1;
        int maxlen = 1;

        // 主循环
        while (right < s.size()) {
            if (containRepeat(s.substr(left, right - left))) {
                // 当前窗口中有重复字符，则窗口长度不变，整体向右移动
                left++;
                right++;
            } else {
                // 当前窗口中不含重复字符，则整个窗口向右移动
                maxlen = max(maxlen, right - left);
                right++;
            }
        }
        
		// 处理最后一次的循环
        if (!containRepeat(s.substr(left, right - left))) {
            maxlen = max(maxlen, right - left);
        }

        return maxlen;
    }
};
```

这一次直接AC了，但是时间复杂度和空间复杂度都很大。ChatGPT的意见是，由于每次的判断中使用了substr方法和containRepeat函数，导致产生了许多不必要的开销。ChatGPT推荐使用set或者unordered\_set来实现这个滑动窗口。

使用set来记录窗口内内容是简单的，但是set本身其中就不会有重复字符，因此使用set进行改写之后，上述逻辑会有一些细微的差别。这里直接用ChatGPT推荐的做法：

```cpp
class Solution {

                charSet.erase(s[left]);
                left++;
            }

```

这里可以看出，这里不再在每次读入一个新字符时就验证一遍当前子串是否有重复字符，而是每次读入一个字符时，如果发现有重复，即立刻删除与该字符重复的字符。这个过程可能会重复多次，窗口长度可能也会不断缩短。

但是上面这个程序的复杂度虽然较之前有很大提升，但是仍然达不到该算法题目的一般水平。于是这里直接看了题解，查看哪里有一些细微差别：

```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        // 哈希集合，记录每个字符是否出现过
        unordered_set<char> occ;
        int n = s.size();
        // 右指针，初始值为 -1，相当于我们在字符串的左边界的左侧，还没有开始移动
        int rk = -1, ans = 0;
        // 枚举左指针的位置，初始值隐性地表示为 -1
        for (int i = 0; i < n; ++i) {
            if (i != 0) {
                // 左指针向右移动一格，移除一个字符
                occ.erase(s[i - 1]);
            }
            while (rk + 1 < n && !occ.count(s[rk + 1])) {
                // 不断地移动右指针
                occ.insert(s[rk + 1]);
                ++rk;
            }
            // 第 i 到 rk 个字符是一个极长的无重复字符子串
            ans = max(ans, rk - i + 1);
        }
        return ans;
    }
};
```

这里需要先讲一下官方题解的思路，尤其是题解讲解了使用“滑动窗口”这种思路的原因。根本在于，本题限制的“无重复字符最长子串”，当确定了左侧起始的字符时，当前位置的“无重复字符最长子串”是固定的。因此，实际上只需要逐个枚举这个“左侧起始字符”，然后设法构造出这个“无重复字符最长子串”即可。

这里，官方题解使用了unordered\_set来存储了每个窗口内的内容。每次移动“左指针”时，只需要通过erase方法删除之前的字符；而通过移动“右指针”改变窗口长度时，则需要通过count方法不断检查窗口内是否有重复字符，如有则继续右移（因此采用循环不断右移），否则则转而移动左指针。

在每次循环中，只需要不断记录每个位置的窗口长度，最后逐个对比即可。

至于C++的语法方面，值得注意的是：

1.与set和unordered\_set的find和count方法，前者返回相应元素的迭代器，如无则返回end()迭代器；后者则返回集合中相应元素的数目（虽然在不存在重复元素的集合中，数目只能是1或0）。

2.string生成子串（字符串切片），是通过它自身的substr方法实现的。该方法的第一个参数确定起始的切片位置，第二个参数为切片长度。
