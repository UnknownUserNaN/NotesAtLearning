给你一个字符串数组，请你将 **字母异位词** 组合在一起。可以按任意顺序返回结果列表。

**字母异位词** 是由重新排列源单词的所有字母得到的一个新单词。

***

最开始想到使用字符串转成的集合作为每个异位词的唯一标识：

```cpp
class Solution {
public:
    set<char> str2charset(string str){
        set<char> ret;
        for(char c: str){
            ret.insert(c);
        }
        return ret;
    }

    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        //采用存储每个字母的集合(set)作为异位词的唯一标识
        unordered_map<set<char>, vector<string>> hashmap;
        for(int i=0; i<strs.size(); i++){
            set<char> charset = str2charset(strs[i]);
            hashmap[charset].push_back(strs[i]);
        }
        
        // 准备输出
        vector<vector<string>> ret;
        for(auto iter: hashmap){
            ret.push_back(iter->second);
        }

        return ret;
    }
};
```

但是ChatGPT指出，set\<char>并不能作为unordered\_map的健，这是因为`set<char>` 是一种容器，而 C++ 中的 `unordered_map` 要求**键类型支持哈希操作**。然而，标准库中的 `set` 并没有定义哈希函数，因此不能直接用作 `unordered_map` 的键。此外题目要求按照字母异位词进行分组，而不仅仅是集合中的字符相同。比如 "abc" 和 "cab" 应归为同一组，而 "abc" 和 "aabbcc" 不应归为同一组。**使用 `set<char>` 会丢失字符的频次信息，导致分组不正确。**

既然ChatGPT指出了代码的问题，并推荐我们使用改进为排序后的字符串，那么我们可以按照这样的思路改进：

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        //采用存排序后的字符串作为异位词的唯一标识
        unordered_map<string, vector<string>> hashmap;
        for(int i=0; i<strs.size(); i++){
            string sorted_str = strs[i];
            sort(sorted_str.begin(), sorted_str.end());
            hashmap[sorted_str].push_back(strs[i]);
        }
        
        // 准备输出
        vector<vector<string>> ret;
        for(auto iter: hashmap){
            ret.push_back(iter.second);
        }

        return ret;
    }
};
```

上面的做法与官方的某个做法不谋而合。但同时，官方还给出了另一种做法，即将字符串中**各字符的频次信息**作为异位词的唯一标识：

```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        // 自定义对 array<int, 26> 类型的哈希函数
        auto arrayHash = [fn = hash<int>{}] (const array<int, 26>& arr) -> size_t {
            return accumulate(arr.begin(), arr.end(), 0u, [&](size_t acc, int num) {
                return (acc << 1) ^ fn(num);
            });
        };

        unordered_map<array<int, 26>, vector<string>, decltype(arrayHash)> mp(0, arrayHash);
        for (string& str: strs) {
            array<int, 26> counts{};
            int length = str.length();
            for (int i = 0; i < length; ++i) {
                counts[str[i] - 'a'] ++;
            }
            mp[counts].emplace_back(str);
        }
        vector<vector<string>> ans;
        for (auto it = mp.begin(); it != mp.end(); ++it) {
            ans.emplace_back(it->second);
        }
        return ans;
    }
};
```

总体思路大同小异，但是其中的一些C++需要注意：

1.定义了arrayHash匿名函数（lambda表达式）：由于这里必须要将array\<int, 26>作为哈希表的键值，但是C++的哈希表不支持这种复杂容器作为键值，需要提供自定义的哈希函数（即将每个数据类型映射到唯一整数的函数）。当然其实现有些过于复杂，这里不多作赘述，总之这里通过一系列精妙的数学方法为每个键值找到了匹配的键值。

2.使用了accumulate算法：这是一个 STL 算法，用于对 `arr` 中的所有元素进行累加操作。它从 `arr.begin()` 到 `arr.end()` 遍历数组，每次将当前累计值 `acc` 和当前元素 `num` 传递给 lambda 函数，并将结果作为新的累计值。比如可以使用该函数实现sum求和函数：

`int sum = accumulate(arr.begin(), arr.end(), 0);`

也可以实现product连乘函数：

`int sum = accumulate(arr.begin(), arr.end(), 1, multiplies<int>());`

3.使用了emplace\_back/emplace\_front方法：传统的push类方法是先创建出一个新元素后再插入到容器之中，而该方法则是直接在容器中创建符合要求的元素。该方法运行起来通常更为高效。
