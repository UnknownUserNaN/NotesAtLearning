给你一个整数数组 `nums`，有一个大小为 `k`\* \*的滑动窗口从数组的最左侧移动到数组的最右侧。你只可以看到在滑动窗口内的 `k` 个数字。滑动窗口每次只向右移动一位。

返回 \*滑动窗口中的最大值 \*。

***

最简单的思路是，将滑动窗口不断从左向右移动，然后在每次滑动的过程中再逐个比较当前窗口中的数值的最大值。但是这个想法显然不行，因为这个算法的时间复杂度是`$O(mn)$`，其中`$m$`是滑动窗口的大小，`$n$`是整个整数数组的大小。

由于之前做过了第560题，借鉴之前的经验，这里考虑利用滑动过程中的中间值。我们只关心滑动过程中的窗口中的最大值，因此，只要最大值没有滑出窗口，我们就只令滑入窗口的新值与原窗口中的最大值进行比较。如果新值大于原最大值，则原最大值可抛弃；如果新值小于最大值，那么我们仍需关注原最大值。如果原最大值滑出窗口，则不得不再设法找到窗口中的最小值。虽然这个方法有些复杂，但是节省了许多不必要的排序，因此实际的时间复杂度会比上面的`$O(mn)$`有一定降低（当然极端情况下两者的时间复杂度相同）。

该题目的提示提到可以使用deque，也就是双端链表来刻画窗口。基于这一点提示和上述思路，可以写出下面的代码：

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        // 刻画滑动窗口
        deque<int> window;

        // 为滑动窗口填入初始值，并设定好最大值
        vector<int> ans;
        int num_max;
        for (int i = 0; i < k; i++) {
            window.push_back(nums[i]);
            if (i == 0) {
                num_max = nums[i];
            } else {
                num_max = nums[i] > num_max ? nums[i] : num_max;
            }
        }
        ans.push_back(num_max);

        // 滑动窗口开始滑动，期间需要关注最大值是否滑出窗口
        if (nums.size() == 1)
            return ans; // 特殊情况
        for (int i = 1; i < nums.size() - k + 1; i++) {
            // 关注滑入和滑出的值
            int num_in = nums[i + k - 1];
            int num_out = nums[i - 1];
            window.push_back(num_in);
            window.pop_front();

            // 最大值判断
            if (num_in > num_max) {
                num_max = num_in; // 如果滑入值大于原最大值，则直接改变最大值
            } else {
                // 滑入值小于原最大值，则观察原最大值是否已滑出窗口
                if (num_out == num_max) {
                    // 已滑出窗口，则需要重新寻找当前窗口中的最大值
                    for (int j = 0; j < k; j++) {
                        if (j == 0) {
                            num_max = nums[i + j];
                        } else {
                            num_max = nums[i + j] > num_max ? nums[i + j] : num_max;
                        }
                    }
                }
                // 最大值未滑出窗口，则num_max维持不变
            }

            // 将结果保存入数组
            ans.push_back(num_max);
        }

        return ans;
    }
};
```

但是，很遗憾运行时间超时了，停在一个数组数值不断递减的测试中，应该是因为最大值被不断地移出窗口，导致窗口在不断地重新寻找当前数组中的最大值。看来算法还需要再优化。

这里最终还是看了官方题解的思路。官方用了一个类似于“大根堆”的思路，直接叙述有点复杂，我们先考虑一个没有滑动窗口存在的版本：

假若我们想要寻找一整个数组中的最大值，我们可以这样做：首先，建立一个存储窗口中最大值索引的双向队列。然后，按顺序遍历数组中的每个值，如果该值大于此前的遍历过的值，则说明此前遍历过的值都不是最大值，于是将该值之前的索引全部弹出（注意是向后弹出）。然后，无论该值是否大于此前遍历过的值，都将该值对应的下标直接压入队列（在后入队）。

在这种没有滑动窗口存在的情况下，按照上述步骤遍历完数组中的所有值，最终排在队列最前面的值一定是这个数组中的最大值。排在这个最大值后面的元素（或索引），大致按照其”适合“成为数组最大值的”程度”排序（当然，这个“程度”是有方向性的）。

虽然这种做法虽然很有意思，但显然不够“经济”，因为我们额外需要一个队列（实际上也可以是栈）来存储各个下标，这浪费了很多空间。如果这个数组的最大值出现在数组的开头，那么这个算法会将这个最大值后面的所有数值全部排入队列。这将耗费许多的空间。

但是，这样的排序方式是有益的，尤其是在本题设计的“滑动窗口”情境中，它可以很方便地扩展。在“滑动窗口”的情况下，即使我们提前遇到了整个数组的最大值，这个最大值也会因为滑出窗口而无效。为了让这些滑出窗口的数值无效化，我们便需要将这些排在队列前方的元素向前弹出（在前出队）。这就是我们需要一个**双向**队列的原因。

这样，在每次的遍历中，我们只需要做上面的两件事：一，按照之前的规则将”可能的最大值“排入队列；二，检查排在队列前方的最大值是否因为滑出窗口而无效了。

基于上面的这些思路，便可以得到下面的代码：

```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        // 记录那些“局部最大值”的索引值
        deque<int> q;

        // 填入初始值
        vector<int> ans;
        for (int i = 0; i < k; i++) {
            while(!q.empty() && nums[i] >= nums[q.back()]){
                // 如果遍历到的值比其前面的值大，那么前面的值一定不是最大值，便可以直接弹出队列
                // 这个过程可能会持续很多次，因此需要使用循环
                q.pop_back();
            }
            q.push_back(i);
        }
        ans.push_back(nums[q.front()]);

        // 窗口开始滑动
        for (int i = k; i < nums.size(); i++) {
            while(!q.empty() && nums[i] >= nums[q.back()]){
                // 如果遍历到的值比其前面的值大，那么前面的值一定不是最大值，便可以直接弹出队列
                // 这个过程可能会持续很多次，因此需要使用循环
                q.pop_back();
            }
            q.push_back(i);

            // 观察此前的局部最大值是否已经滑出窗口
            while(q.front() <= i-k){
                q.pop_front(); // 将此前的局部最大值弹出
            }

            // 将结果保存入数组
            ans.push_back(nums[q.front()]);
        }

        return ans;
    }
};
```
