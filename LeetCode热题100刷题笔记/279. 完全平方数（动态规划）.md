给你一个整数 n ，返回 和为 n 的完全平方数的最少数量 。

完全平方数 是一个整数，其值等于另一个整数的平方；换句话说，其值等于一个整数自乘的积。例如，1、4、9 和 16 都是完全平方数，而 3 和 11 不是。

---
采用动态规划的思路的话，本题的思路很简单。拿到一个数`n`，假设本题所求的“和为`n`的完全平方数的最少数量”可以用动态规划数组表示为`dp[n-1]`，那么`dp[n-1]`可以由此前的若干种状态加上对应的完全平方数得到。比如假设`n==12`，那么`dp[11]`可以由`dp[10]`加1得到，也可以由`dp[7]`加4得到，也可以由`dp[2]`加9得到，`dp[11]`的具体取值，应当在这三种方案中取最小值，也就是`dp[11]=min(dp[10], dp[7], dp[2])+1`。具体可以由哪些状态转移得到，取决于该数之下有多少个完全平方数，即输入一个整数`n`，它的下面应当有`(int)sqrt(n)`个完全平方数。
对输入的一个整数`n`，我们不能总是认为用较大的完全平方数组成它所用的数量最少。比如题目中给出的`n==12`的例子，我们会下意识认为先用9来组成它能使用最少的完全平方数。但事实并非如此，使用9的话，那么整个和的序列应当为`[9, 1, 1, 1]`，而实际上最短的序列是`[4, 4, 4]`。所以，我们最好还是遍历每个它能转移而来的状态。
基于此，可以写出如下的代码：
```cpp
class Solution {
public:
    int numSquares(int n) {
        vector<int> dp(n);
        for (int i = 0; i < n; i++) {
            int a = (int)sqrt(i + 1);
            if (a * a == i + 1)
                // 该数是完全平方数
                dp[i] = 1;
            else {
                // 该数不恰好是完全平方数
                // 遍历每个其可以转移而来的状态，选择其中的最小值
                int min_val = INT_MAX;
                for (int j = 1; j <= a; j++)
                    min_val = min(min_val, dp[i - j * j]);
                dp[i] = min_val + 1;
            }
        }
        return dp[n - 1];
    }
};
```
不过比较遗憾的一点是，该算法的时间复杂度达到了$O(n\sqrt n)=O(n^{\frac{3}{2}})$，略高；空间复杂度达到了$O(n)$，如果采用递归方法的话可能会更低一点。
最终，在[官方题解](https://leetcode.cn/problems/perfect-squares/solutions/822940/wan-quan-ping-fang-shu-by-leetcode-solut-t99c/)中找到了更好的方案。不出意外的，这个问题更好的解法需要诉诸数学。
> 数学中的“四平方和定理“指出，任意一个正整数都可以被表示为至多四个正整数的平方和。这给出了这个问题的上界。
> 同时四平方和定理包含了一个更强的结论：当且仅当$n=4^k\times(8m+7)$时，$n$可以被表示为至多三个正整数的平方和。因此，当$n=4^k\times(8m+7)$时，n只能被表示为四个正整数的平方和。此时我们可以直接返回4。
> 当$n\neq4^k\times(8m+7)$时，我们需要判断到底多少个完全平方数能够表示$n$，我们知道答案只会是1,2,3中的一个，所以只需要枚举。
> - 答案为1时，则必有n为完全平方数，这很好判断;
> - 答案为2时，则有$n=a^2+b^2$，我们只需要枚举所有的$a(1≤a≤√n)$，判断$n-a^2$是否为完全平方数
> - 即可;
> - 答案为3时，我们很难在一个优秀的时间复杂度内解决它，但我们只需要检查答案为1或2的两种情况。
> 即可利用排除法确定答案。

基于此写出的代码如下：
```cpp
class Solution {
public:
    // 判断是否为完全平方数
    bool isPerfectSquare(int x) {
        int y = sqrt(x);
        return y * y == x;
    }

    // 判断是否能表示为 4^k*(8m+7)
    bool checkAnswer4(int x) {
        while (x % 4 == 0) {
            x /= 4;
        }
        return x % 8 == 7;
    }

    int numSquares(int n) {
        if (isPerfectSquare(n)) {
            return 1;
        }
        if (checkAnswer4(n)) {
            return 4;
        }
        for (int i = 1; i * i <= n; i++) {
            int j = n - i * i;
            if (isPerfectSquare(j)) {
                return 2;
            }
        }
        return 3;
    }
};
```
这样的算法，时间复杂度为$O(\sqrt{n})$，空间复杂度更是降到了$O(1)$，非常优秀。
