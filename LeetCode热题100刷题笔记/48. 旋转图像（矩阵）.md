给定一个 n × n 的二维矩阵 matrix 表示一个图像。请你将图像顺时针旋转 90 度。

你必须在 原地 旋转图像，这意味着你需要直接修改输入的二维矩阵。请不要 使用另一个矩阵来旋转图像。

---
还没做，不过目测这题需要使用线性代数。先从线性代数的思路考虑一遍这个问题。
向右旋转90°，对应着基向量组从$I=\left[\begin{matrix}1 & 0 \\ 0 & 1 \end{matrix}\right]$到$T=\left[\begin{matrix} 0 & 1\\ -1 &0 \end{matrix}\right]$的变换。这个变换特殊之处在于，原来在$x$轴上的基向量在旋转后跑到了$y$轴上，原来在$y$轴上的跑到了$x$轴上。矩阵上的体现就是从原来的单位矩阵变成了“反对角矩阵”。
题目要求使用“原地”算法。一种典型的原地算法就是一维数组的“倒序”，这是因为它可以通过关于对称轴对称的两个元素的交换实现。到了二维的矩阵（本题只涉及方阵），我们显然也可以找一条对称轴然后对对称轴两侧的元素做交换操作。
一种最容易想到的是矩阵的上下对换（翻转）和左右对换（翻转），这分别对应着正方形的竖直对称轴和水平对称轴，也分别对应着$T_{ud}=\left[\begin{matrix}1&0\\0&-1\end{matrix}\right]$和$T_{lr}=\left[\begin{matrix}-1&0\\0&1\end{matrix}\right]$的线性变换。如果我们希望通过这两个线性变换的“复合”得到向右旋转90°的效果，显然是不可能的。因为很显然，这两者是对角矩阵，而旋转90°所对应的矩阵是“反对角”的。具体而言，这两个变换只会让原来在$x$轴上的基向量继续在$x$轴上，原来在$y$轴的继续在$y$轴上，根本无法实现“跑到另一个坐标轴上“的效果。
那么有没有一条能实现“跑到另一个坐标轴上”的对称轴呢？显然是有的。正方形不止前面提到的竖直对称轴和水平对称轴，还有它的两条对角线也可作为对称轴，这两条对角线上的对称轴分别对应着$T_{tp}=\left[\begin{matrix}0&1\\1&0\end{matrix}\right]$和$T_{ntp}=\left[\begin{matrix}0&-1\\-1&0\end{matrix}\right]$这两个线性变换。这两个变换恰好能使原来在$x$轴上的基向量在旋转后跑到了$y$轴上，原来在$y$轴上的跑到了$x$轴上，也就是让对角矩阵变成了“反对角矩阵”。
那么很显然，要得到变换$T$，需要前面这两种对换操作的复合。也就是说，实际上我们可以发现，$T=T_{ud}T_{tp}=T_{lr}T_{ntp}$。要得到向右旋转90°的效果，我们要么先做一次$T_{tp}$，再做一次$T_{ud}$；要么就先做一次$T_{ntp}$，再做一次$T_{lr}$。
为了能够利用C++中的内置算法（也就是algorithm库中的reverse函数），我们选择使用第二种方法编写程序：
```cpp
class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int n=matrix.size(); // 题目只涉及方阵

        // 先做一次T_{ntp}
        for(int i=0; i<n; i++){
            for(int j=0; j<i; j++){
                int temp = matrix[i][j];
                matrix[i][j] = matrix[j][i];
                matrix[j][i] = temp;
            }
        }

        // 利用内置算法，再做一次T_{lr}
        for(int i=0; i<n; i++){
            reverse(matrix[i].begin(), matrix[i].end());
        }
    }
};
```
这算是最“优雅”的解法了。官方题解的解法三与上面的大同小异，只不过改为使用了utility库中的swap函数：
```cpp
class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int n = matrix.size();
        // 水平翻转
        for (int i = 0; i < n / 2; ++i) {
            for (int j = 0; j < n; ++j) {
                swap(matrix[i][j], matrix[n - i - 1][j]);
            }
        }
        // 主对角线翻转
        for (int i = 0; i < n; ++i) {
            for (int j = 0; j < i; ++j) {
                swap(matrix[i][j], matrix[j][i]);
            }
        }
    }
};
```
值得一提的是，官方题解中还有一种“分区”交换值的算法。简单而言，是将方阵分割成四个需要交换取值的分区，然后每个分区中的元素顺次交换它们的值。具体可以参考[官方题解](https://leetcode.cn/problems/rotate-image/solutions/526980/xuan-zhuan-tu-xiang-by-leetcode-solution-vu3m/)中的方法二。
```cpp
class Solution {
public:
    void rotate(vector<vector<int>>& matrix) {
        int n = matrix.size();
        for (int i = 0; i < n / 2; ++i) {
            for (int j = 0; j < (n + 1) / 2; ++j) {
                int temp = matrix[i][j];
                matrix[i][j] = matrix[n - j - 1][i];
                matrix[n - j - 1][i] = matrix[n - i - 1][n - j - 1];
                matrix[n - i - 1][n - j - 1] = matrix[j][n - i - 1];
                matrix[j][n - i - 1] = temp;
            }
        }
    }
};
```
