给你两个单词 word1 和 word2， 请返回将 word1 转换成 word2 所使用的最少操作数  。

你可以对一个单词进行如下三种操作：

- 插入一个字符
- 删除一个字符
- 替换一个字符

提示：word1 和 word2 由小写英文字母组成

---
既然本题提醒了多维动态规划，那我们先就试着拆分题目中的问题，看看能不能把它拆成形式相同但规模更小的子问题。
中间我尝试了一些思路，但是始终没能将完整的思路想出来。只能想到按`dp[m][n]`的形式建立`(m+1)*(n+1)`大小的数组，其中`m`是`word1`的长度，`n`是`word2`的长度，然后我们逐步从`dp[0][0]`遍历到`dp[m][n]`获得答案。
但是各个数组之间状态如何实现转移？我不清楚，这里只能去求助[官方题解](https://leetcode.cn/problems/edit-distance/solutions/188223/bian-ji-ju-chi-by-leetcode-solution/)了。
官方题解一上来的思路非常清晰。首先，题目中给出了三种可选的操作，这三种操作可以分别作用到两个字符串上，所以总共有6个操作。但我们不难发现，给一个字符串插入（或删除）字符相当于给另一个字符串删除（或插入）字符，所以给`word1`（或`word2`)删除字符的操作实际上可以等价为给`word2`（或`word1`）插入字符的操作，这样4个操作就可以简化为2个。另外，替换`word1`的字符也相当于替换`word2`的字符，这两者是“双向奔赴”的，所以还有2个操作就可以简化为1个。这样，6个操作就可以简化到下面的3个：
- 在`word1`中插入一个字符；
- 在`word2`中插入一个字符；
- 修改单词`word1`的一个字符。
这样一来，把原问题转化为规模较小的子问题就变得方便了。之所以最终简化剩下插入操作，是因为插入这个过程非常符合我们从0规模慢慢扩展到目标规模的动态规划过程，这一点在后面的分析中可以体会到。
随后官方题解的一段推理我觉得非常精彩：
> 我们用A=horse，B=ros作为例子，来看一看是如何把这个问题转化为规模较小的若干子问题的。
> - 在单词A中插入一个字符：如果我们知道horse到ro的编辑距离为a，那么显然horse到ros的编辑距离不会超过a+1。这是因为我们可以在次操作后将horse和r变为相同的字符串，只需要额外的1次操作，在单词的末尾添加字符，就能在a+1次操作后将horse和r变为相同的字符串;
> - 在单词B中插入一个字符：如果我们知道hors到ros的编辑距离为a，那么显然 horse到 ros的编辑距离不会超过b＋1，原因同上;
> - 修改单词A的一个字符：如果我们知道hors到ro的编辑距离为，那么显然horse到ros的编辑距离不会超过c+1，原因同上。
> 那么从horse变成ros的编辑距离应该为min(a + 1, b + 1, c + 1)
> 注意：为什么我们总是在单词A和B的末尾插入或者修改字符，能不能在其它的地方进行操作呢？答案是可以的，但是我们知道，操作的顺序是不影响最终的结果的。例如对于单词cat，我们希望在和之间添加字符并且将字符修改为字符，那么这两个操作无论为什么顺序，都会得到最终的结果cdab。
> 你可能觉得horse到ro这个问题也很难解决。但是没关系，我们可以继续用上面的方法拆分这个问题，对于这个问题拆分出来的所有子问题，我们也可以继续拆分，直到：
> - 字符串A为空，如从 转换到ro，显然编辑距离为字符串B的长度，这里是2;
> - 字符串B为空，如从horse转换到 ，显然编辑距离为字符串的长度，这里是5。
> 因此，我们就可以使用动态规划来解决这个问题了。我们用D\[i]\[j]表示的前i个字母和的前j个字母之间的编辑距离。

这个考虑动态规划问题的思路非常值得借鉴。首先确定问题的规模，然后确定如何描述各个规模的状态，再确定各个状态之间有哪些可能的转移方式（这时非常需要假设某一“前状态”的取值为x），最后再确定边界条件。
基于上面的一套推理，我们可以得出下面的状态转移方程。可以看出，下面$\min$函数中的三个分项，分别对应着上面的三个操作。
- 如果`word1`与`word2`的最后一个字母相同：$$dp[i][j]=\min(dp[i][j-1]+1, dp[i-1][j]+1, dp[i-1][j-1])$$因为此时从`dp[i-1][j-1]`转移到`dp[i][j]`不需要修改最后一个字符。
- 如果`word1`与`word2`的最后一个字母不同：$$dp[i][j]=\min(dp[i][j-1]+1, dp[i-1][j]+1, dp[i-1][j-1]+1)$$
有了状态转移方程了，也就容易写出程序了：
```cpp
class Solution {
public:
    int minDistance(string word1, string word2) {
        int n = word1.length();
        int m = word2.length();

        // 有一个字符串为空串
        if (n * m == 0) return n + m;

        // DP 数组
        vector<vector<int>> D(n + 1, vector<int>(m + 1));

        // 边界状态初始化
        for (int i = 0; i < n + 1; i++) {
            D[i][0] = i;
        }
        for (int j = 0; j < m + 1; j++) {
            D[0][j] = j;
        }

        // 计算所有 DP 值
        for (int i = 1; i < n + 1; i++) {
            for (int j = 1; j < m + 1; j++) {
                int left = D[i - 1][j] + 1;
                int down = D[i][j - 1] + 1;
                int left_down = D[i - 1][j - 1];
                if (word1[i - 1] != word2[j - 1]) left_down += 1;
                D[i][j] = min(left, min(down, left_down));

            }
        }
        return D[n][m];
    }
};
```
写出了程序，剩下的复杂度分析也就非常简单了。观察循环算法和动态规划数组的大小就能发现，该问题的时间复杂度和空间复杂度都是$O(mn)$。
