给你一个 只包含正整数 的 非空 数组 nums 。请你判断是否可以将这个数组分割成两个子集，使得两个子集的元素和相等。

---
仍然是创建动态规划数组`dp[i]`，其中的每个元素表示与题目有关的、规模更小的子问题的量。具体是哪个量，我们根据情境稍作研究后确定。
我们容易确定，在`i==0`时，此时一定会分成单个元素的集合与另一个空集，因此此时两个集合之间的差一定为`nums[0]`。然后是`i==1`的情况，如果`nums[1]==nums[0]`，那么这时就可以将两个集合分为等和子集；反之，则不能将其分割为等和子集。我们此时可以留意到一个细节，在`dp[i]`的子问题情况下，如果我们新加入的`nums[i]`元素恰好可以“填补”此前两个集合之间的“差距”，那么我们便可以确认在此时可以分割出等和子集。
我们不妨逆向思考一下。我们希望在`i==n-1`时划分出等和子集，也就是两子集之差为`0`。那么，我们就需要在`i==n-2`时，划分出两个子集之差为`nums[n-1]`的情况。进一步地，在`i==n-3`时，我们要划分出两个子集之差为`nums[n-1]+nums[n-2]`或者`abs(nums[n-1]-nums[n-2])`的情况，这分别对应着使差距扩大的选择（即将新元素划入取值更大的数组）和使差距缩小的选择（即将新元素划入取值更小的数组）。
到这里，有了一个不使用动态规划的解法：我们枚举每个元素，每个元素都有使此前的两数集之差增大或减小的可能，我们枚举这些差，遍历完整个数组后检查这些差中是否有0即可。但是这个解法的时间复杂度和空间复杂度实在太高，分别是$O(n^2)$和$O(n)$。
```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        if (nums.size() == 1)
            return false;

        unordered_set<int> diff = {0};
        for (int i = 0; i < nums.size(); i++) {
            unordered_set<int> next_diff;
            for (int each : diff) {
                next_diff.insert(each + nums[i]);
                next_diff.insert(each > nums[i] ? each - nums[i]
                                                : nums[i] - each);
            }
            diff = next_diff;
        }
        return find(diff.begin(), diff.end(), 0) != diff.end();
    }
};
```
这个算法还能再怎么优化呢？我想不到了，只能去看看[官方题解](https://leetcode.cn/problems/partition-equal-subset-sum/solutions/442320/fen-ge-deng-he-zi-ji-by-leetcode-solution/)怎么做的。
官方题解上来搞了一个小小的幽默：
> 本题是经典的「NP 完全问题」，也就是说，如果你发现了该问题的一个多项式算法，那么恭喜你证明出了 P=NP，可以期待一下图灵奖了。
> 正因如此，我们不应期望该问题有多项式时间复杂度的解法。我们能想到的，例如基于贪心算法的「将数组降序排序后，依次将每个元素添加至当前元素和较小的子集中」之类的方法都是错误的，可以轻松地举出反例。因此，我们必须尝试非多项式时间复杂度的算法，例如时间复杂度与元素大小相关的动态规划。

随后，官方题解说明此问题可以抽象成“0-1背包问题”。
> 这道题可以换一种表述：给定一个只包含正整数的非空数组 nums[0]，判断是否可以从数组中选出一些数字，使得这些数字的和等于整个数组的元素和的一半。因此这个问题可以转换成「0−1 背包问题」。这道题与传统的「0−1 背包问题」的区别在于，传统的「0−1 背包问题」要求选取的物品的重量之和不能超过背包的总容量，这道题则要求选取的数字的和恰好等于整个数组的元素和的一半。类似于传统的「0−1 背包问题」，可以使用动态规划求解。

问题转化为我们能否从数组中选出一些元素，使这些元素的和等于全部元素和的一半。关于这个问题，有一些非常简单的情况我们可以直接给出答案：
- 如果数组中只有一个元素，那么显然不可分，直接返回false；
- 如果数组的全部元素和为奇数，那么显然不可分，直接返回false；
- 如果数组的全部元素和为偶数，但是存在一个大于全部元素和的一半的元素，此时也可以直接返回false。
所以我们接下来讨论的都是数组长度大于1、全部元素和为偶数且不存在“过于大”的极端值的情况。对于这个已经“剪过枝”的情况，官方题解再采用动态规划方法求解：
> 创建**二维数组** dp，包含 n 行 target+1 列（target=sum/2)，其中 dp\[i]\[j] 表示从数组的 [0,i] 下标范围内选取若干个正整数（可以是 0 个），是否存在一种选取方案使得被选取的正整数的和等于 j。初始时，dp 中的全部元素都是 false。
> 在定义状态之后，需要考虑边界情况。以下两种情况都属于边界情况：
> - 如果不选取任何正整数，则被选取的正整数之和等于 0。因此对于所有 0≤i<n，都有 dp\[i]\[0]=true。
> - 当 i\==0 时，只有一个正整数 nums[0] 可以被选取，因此 dp\[0]\[nums[0]]=true。
> 对于 i>0 且 j>0 的情况，如何确定 dp\[i]\[j] 的值？需要分别考虑以下两种情况。
> - 如果 j≥nums[i]，则对于当前的数字 nums[i]，可以选取也可以不选取，两种情况只要有一个为 true，就有 dp\[i]\[j]=true。
> 	- 如果不选取 nums[i]，则 dp\[i]\[j]=dp\[i−1]\[j]；
> 	- 如果选取 nums[i]，则 dp\[i]\[j]=dp\[i−1]\[j−nums[i]]。
> - 如果 j<nums[i]，则在选取的数字的和等于 j 的情况下无法选取当前的数字 nums[i]，因此有 dp\[i]\[j]=dp\[i−1]\[j]。

此时的状态转移方程为：$$
dp[i][j]=\begin{cases}
dp[i-1][j]||dp[i-1][j-nums[i]],&j\geq nums[i]\\
dp[i-1][j], &j<nums[i]
\end{cases}
$$
显然，题中问题的答案即为`dp[n-1][target]`。
基于此，可以写出下面的代码：
```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int n = nums.size();
        if (n < 2) {
            return false;
        }
        int sum = accumulate(nums.begin(), nums.end(), 0);
        int maxNum = *max_element(nums.begin(), nums.end());
        if (sum & 1) {
            return false;
        }
        int target = sum / 2;
        if (maxNum > target) {
            return false;
        }
        
        vector<vector<int>> dp(n, vector<int>(target + 1, 0));
        for (int i = 0; i < n; i++) {
            dp[i][0] = true;
        }
        dp[0][nums[0]] = true;
        for (int i = 1; i < n; i++) {
            int num = nums[i];
            for (int j = 1; j <= target; j++) {
                if (j >= num) {
                    dp[i][j] = dp[i - 1][j] | dp[i - 1][j - num];
                } else {
                    dp[i][j] = dp[i - 1][j];
                }
            }
        }
        return dp[n - 1][target];
    }
};
```
当然，我们不难发现，`dp`中的新元素无论如何都要依赖`i-1`时的元素，所以实际上我们并不需要`i`这个维度，`dp`这个数组实际上只需要设定成一维的就可以。
优化后的程序如下：
```cpp
class Solution {
public:
    bool canPartition(vector<int>& nums) {
        int n = nums.size();
        if (n < 2) {
            return false;
        }
        int sum = 0, maxNum = 0;
        for (auto& num : nums) {
            sum += num;
            maxNum = max(maxNum, num);
        }
        if (sum & 1) {
            return false;
        }
        int target = sum / 2;
        if (maxNum > target) {
            return false;
        }
        
        vector<int> dp(target + 1, 0);
        dp[0] = true;
        for (int i = 0; i < n; i++) {
            int num = nums[i];
            for (int j = target; j >= num; --j) {
                dp[j] |= dp[j - num];
            }
        }
        return dp[target];
    }
};
```
这样就把程序的时间复杂度降到了$O(n\times target)$级，空间复杂度降到了$O(target)$级。
有一个解决动态规划问题的心得：如果实在没有动态规划的思路，试着把描述问题规模的量都试着减小，建立一个多维的动态规划数组，写出状态转移方程之后再尝试优化时间和空间复杂度。
