给你一个长度为 n 的链表，每个节点包含一个额外增加的随机指针 random ，该指针可以指向链表中的任何节点或空节点。

构造这个链表的 深拷贝。 深拷贝应该正好由 n 个 全新 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 next 指针和 random 指针也都应指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。复制链表中的指针都不应指向原链表中的节点 。

例如，如果原链表中有 X 和 Y 两个节点，其中 X.random --> Y 。那么在复制链表中对应的两个节点 x 和 y ，同样有 x.random --> y 。

返回复制链表的头节点。

用一个由 n 个节点组成的链表来表示输入/输出中的链表。每个节点用一个 [val, random_index] 表示：

val：一个表示 Node.val 的整数。
random_index：随机指针指向的节点索引（范围从 0 到 n-1）；如果不指向任何节点，则为  null 。
你的代码 只 接受原链表的头节点 head 作为传入参数。

---
最简单的思路是遍历两次输入的链表。第一次遍历，按顺序建立链表，但是忽略掉其中的random项，一边建立新链表一边记录每个结点的指针。第二次遍历，处理新链表的random项，建立其连接。
第二步在想法上非常简单，但是实现起来有一些困难。旧链表的random项是指针，但是我们无从知道这些指针究竟指向的是链表的第几个。没办法，我们只能遍历前面建立的旧结点的指针，然后对应上它在链表中的位置，最后对应上新链表中的指针。这一步本可以使用类似哈希表的方式一一对应的，但是C++的哈希表无法直接建立这样的键值对，需要自己手动编写哈希函数。这就有点复杂了。
基于此就可以写出代码：
```cpp
/*
// Definition for a Node.
class Node {
public:
    int val;
    Node* next;
    Node* random;

    Node(int _val) {
        val = _val;
        next = NULL;
        random = NULL;
    }
};
*/

class Solution {
public:
    Node* copyRandomList(Node* head) {
        // 第一次遍历：建立新链表的每个结点
        vector<Node*> old_ptrs, new_ptrs;
        Node* new_prehead = new Node(0); // 哨兵空结点
        Node *p1 = head, *p2 = new_prehead;
        while (p1 != nullptr) {
            p2->next = new Node(p1->val);
            old_ptrs.push_back(p1);
            new_ptrs.push_back(p2->next);
            p1 = p1->next;
            p2 = p2->next;
        }

        // 第二次遍历：处理新链表的random项
        p1 = head;
        p2 = new_prehead->next;
        while (p1 != nullptr) {
            if (p1->random == nullptr)
                p2->random = nullptr;
            else {
                int idx = 0;
                while (p1->random != old_ptrs[idx]) {
                    idx++;
                }
                p2->random = new_ptrs[idx];
            }
            p1 = p1->next;
            p2 = p2->next;
        }

        Node* ret = new_prehead->next;
        delete new_prehead;
        return ret;
    }
};
```
[官方题解](https://leetcode.cn/problems/copy-list-with-random-pointer/solutions/889166/fu-zhi-dai-sui-ji-zhi-zhen-de-lian-biao-rblsf/)中的回答就采用了经典的哈希表来代替了上面查询的操作。不过具体实现上，官方题解采用了递归的方法来实现。也就是说，通过头结点的next和random结点延伸出去，依次建立新链表中的各个结点，并将其指针储存在全局存储的哈希表中。如果哈希表中有相应结点，则直接返回相应的指针；如果没有相应结点，则建立相应结点后再返回其指针。
这是一个非常精妙的算法，代码的长度也很短，实现起来的思路也非常清晰。
```cpp
class Solution {
public:
    unordered_map<Node*, Node*> cachedNode;

    Node* copyRandomList(Node* head) {
        if (head == nullptr) {
            return nullptr;
        }
        if (!cachedNode.count(head)) {
            Node* headNew = new Node(head->val);
            cachedNode[head] = headNew;
            headNew->next = copyRandomList(head->next);
            headNew->random = copyRandomList(head->random);
        }
        return cachedNode[head];
    }
};
```
但是很明显地，上面的算法都需要花费$O(n)$的空间，这显然不经济。[官方题解](https://leetcode.cn/problems/copy-list-with-random-pointer/solutions/889166/fu-zhi-dai-sui-ji-zhi-zhen-de-lian-biao-rblsf/)中又提到了一种名为“结点拆分”的方法。这种方法能在保留原来的$O(n)$时间的复杂度的情况下，将空间消耗降低到$O(1)$。其具体步骤如下：
> 我们首先将该链表中每一个节点拆分为两个相连的节点，例如对于链表 A→B→C，我们可以将其拆分为 A→A'→B→B'→C→C'。对于任意一个原节点 S，其拷贝节点 S' 即为其后继节点。
> 这样，我们可以直接找到每一个拷贝节点 S 的随机指针应当指向的节点，即为其原节点 S 的随机指针指向的节点 T 的后继节点 T '。需要注意原节点的随机指针可能为空，我们需要特别判断这种情况。
> 当我们完成了拷贝节点的随机指针的赋值，我们只需要将这个链表按照原节点与拷贝节点的种类进行拆分即可，只需要遍历一次。同样需要注意最后一个拷贝节点的后继节点为空，我们需要特别判断这种情况。

这个思路非常推荐参考官方题解中的图像理解，更能领会到官方算法中的巧妙之处。具体而言，该方法通过“老结点后面插入对应新结点”的方法，创造了老与新之间的对应关系，代替了哈希表方法中的哈希表。这里启发我们，有时可以**利用原数据结构的特点创造对应关系，而不一定非要利用哈希表**实现。
官方算法的程序如下：
```cpp
class Solution {
public:
    Node* copyRandomList(Node* head) {
        if (head == nullptr) {
            return nullptr;
        }
        for (Node* node = head; node != nullptr; node = node->next->next) {
            Node* nodeNew = new Node(node->val);
            nodeNew->next = node->next;
            node->next = nodeNew;
        }
        for (Node* node = head; node != nullptr; node = node->next->next) {
            Node* nodeNew = node->next;
            nodeNew->random = (node->random != nullptr) ? node->random->next : nullptr;
        }
        Node* headNew = head->next;
        for (Node* node = head; node != nullptr; node = node->next) {
            Node* nodeNew = node->next;
            node->next = node->next->next;
            nodeNew->next = (nodeNew->next != nullptr) ? nodeNew->next->next : nullptr;
        }
        return headNew;
    }
};
```
但是值得一提的是，此种方法比哈希表法的运行时间略高，前者是$O(3n)=O(n)$，后者则约为$O(n)$。不难看出这是因为“结点拆分“法对原链表进行了三次遍历：
1. 遍历一次原链表，在每个“老结点”后方都增加一个“新结点”，并插入到老结点之间；
2. 再遍历一次原链表，根据”老结点“后方是对应“新结点”的特性，将老结点的random指针“复制”到新结点上（老结点的random指针的后继就是对应的新结点）；
3. 再遍历一次原链表，将老结点和新结点依次“解开”，各自构成新的链表。
