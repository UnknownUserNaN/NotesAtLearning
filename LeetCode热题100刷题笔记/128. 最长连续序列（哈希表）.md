给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)`\* \*的算法解决此问题。

***

我的做法一开始并没有用到哈希表，跑测试意外地用时和内存都很低：

```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        // 边界情况处理
        if(nums.size() == 0) return 0;
        else if(nums.size() == 1) return 1;
 
        // 先对输入进行排序，这样数值连续的序列在数组中位置也是连续的了
        sort(nums.begin(), nums.end());

        // 再对数组中的各个连续序列进行长度的比较
        int max_length = 1;
        int current_length = 1;
        for(int i=1; i<nums.size(); i++){
            if(nums[i] == nums[i-1]+1){
                // 递增
                current_length++;
            }
            else if(nums[i] == nums[i-1]){
                // 与前者相等
                continue;
            }
            else{
                // 与前者不连续
                current_length = 1;
            }

            if(current_length > max_length){
                max_length = current_length;
            }
        }

        return max_length;
    }
};
```

官方的做法更为巧妙。核心思路与上面大同小异，但是通过对集合中更小值的搜索避开了排序这一步骤（排序的复杂度是`$O(n\log n)$`，而集合或者哈希表中查询的复杂度**理论上**是`$O(1)$`)。通过检查集合中是否有前驱的存在，从而确定当前的数值是否可以作为迭代的起始点。后面的思路便都大同小异了。

```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        int ans = 0;
        unordered_set<int> st(nums.begin(), nums.end()); // 把 nums 转成哈希集合
        for (int x : st) {
            if (st.contains(x - 1)) {
                continue;
            }
            // x 是序列的起点
            int y = x + 1;
            while (st.contains(y)) { // 不断查找下一个数是否在哈希集合中
                y++;
            }
            // 循环结束后，y-1 是最后一个在哈希集合中的数
            ans = max(ans, y - x); // 从 x 到 y-1 一共 y-x 个数
        }
        return ans;
    }
};
```

几个值得注意的点：

1.**实现哈希表不一定要通过unordered\_map来实现**。看该题，似乎也可以通过unordered\_set实现。

2.在创建复杂容器时可以通过指定数据类型来快速转化，比如这里的`unordered_set<int> st(nums.begin(), nums.end());`。

3.理论上讲，类似查询之类的操作，其复杂度都可以视为`$O(1)$`。
