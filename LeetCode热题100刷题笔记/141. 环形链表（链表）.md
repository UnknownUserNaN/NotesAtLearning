给你一个链表的头节点 head ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：pos 不作为参数进行传递 。仅仅是为了标识链表的实际情况。

如果链表中存在环 ，则返回 true 。 否则，返回 false 。

进阶：你能用 O(1)（即，常量）内存解决此问题吗？

---
由于在刷到这个题之前就了解过这个问题的解决方法，因此做到这个题其实有种被“剧透”的感觉。这个问题的解决方案就是经典的“快慢指针法”。两个指针，慢指针一次走一步，快指针一次走两步。如果链表中存在环，那么快指针必将与慢指针相遇，也就是“套圈”了，说明链表中有环；如果快指针或者慢指针中有一者走到了链表的尽头（即NULL），也就说明这个链表没有环。
基于这个思路能很快写出代码：
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    ListNode* detectCycle(ListNode* head) {
        ListNode *slow = head, *fast = head;
        bool flag = false;
        while (fast != nullptr) {
            slow = slow->next;
            if (fast->next == nullptr) {
                return nullptr;
            }
            fast = fast->next->next;
            if (fast == slow) {
                flag = true;
                break;
            }
        }
    }
}
```
当然，评论区还给出了一种“歪门邪道法”，纯为了通过测试而用的：

> 原理上官方的是正路。歪路就是构造一个假节点，把所有访问过的表格的next都指向假节点，并且在循环内判断是否访问到构造的假节点。
> 还有一个最简单的歪路就是把遍历过的node的value改成1000001，判断下一个node的value是1000001就表示有环。
