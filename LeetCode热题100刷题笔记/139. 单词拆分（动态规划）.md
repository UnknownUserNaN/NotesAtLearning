给你一个字符串 s 和一个字符串列表 wordDict 作为字典。如果可以利用字典中出现的一个或多个单词拼接出 s 则返回 true。

注意：不要求字典中出现的单词全部都使用，并且字典中的单词可以重复使用。

提示：
- s 和 wordDict[i] 仅由小写英文字母组成
- wordDict 中的所有字符串 互不相同

---
这道题虽然归在了动态规划的分组之下，但我疑心这道题似乎使用回溯法更好写，但是回溯法的时间复杂度实在是太高了，大概率不符合要求。
乍看之下，本题似乎很难使用动态规划方法解决。但是，我们不难注意到，假若一个字符串是可以进行拆分的，那么它在去掉一个子字符串后一定也是可以合理的。比如当`s="catsanddogs"`，`wordDict=['cats', 'and', 'dogs']`时，原字符串恰好可以被字典拆分，所以当字符串中去掉末尾的`"dogs"`之后，字符串显然还能继续拆分。我们便可以利用这个递归性质。
如果我们将字符串长度`n`作为题中问题规模的指标的话，那么我们最终要求的问题的答案就是`dp[n]`。可以注意到，只要`wordDict`中存在一个单词能让`dp`从一个为true的状态”转移“到另一个为true的状态，就可以保证我们最终在`dp[n]`得到问题的答案。具体的判断规则是：如果字典中存在与字符串（设字符串长度为`n`）末端匹配的单词（设单词长度为`l`），并且`dp[n-l]==true`，那么便可以确定`dp[n]==true`。
基于此，我们可以写出下面的程序：
```cpp
class Solution {
public:
    bool match(string& s, int pos, string& word) {
        // 检测s的末端是否能与word匹配
        int i = pos - 1, j = word.size() - 1;
        while (j >= 0) {
            if (s[i] != word[j])
                return false;
            i--;
            j--;
        }
        return true;
    }

    bool wordBreak(string s, vector<string>& wordDict) {
        vector<bool> dp(s.size() + 1);
        dp[0] = true;
        for (int i = 1; i <= s.size(); i++) {
            bool ans = false;
            for (string word : wordDict) {
                if (word.size() > i)
                    continue;
                if (dp[i - word.size()] && match(s, i, word)){
                    ans = true;
                    break;
                }
            }
            dp[i] = ans;
        }
        return dp[s.size()];
    }
};
```
上面程序的时间复杂度为$O(mn^2)$，略高，其中$m$为字典中词的数量，$n$为字符串的长度。空间复杂度为$O(n)$，还算合理。
[官方题解](https://leetcode.cn/problems/word-break/solutions/302471/dan-ci-chai-fen-by-leetcode-solution/)的做法要更合理一点，使用集合来存储各词，在字符串中枚举分割的位置，如果该位置处的`dp`取值为`true`且该位置之后的子串恰好能在集合中找到，就说明该字符串是可以被分割的。这个设计使得时间复杂度得以降到$O(n^2)$。
```cpp
class Solution {
public:
    bool wordBreak(string s, vector<string>& wordDict) {
        auto wordDictSet = unordered_set <string> ();
        for (auto word: wordDict) {
            wordDictSet.insert(word);
        }

        auto dp = vector <bool> (s.size() + 1);
        dp[0] = true;
        for (int i = 1; i <= s.size(); ++i) {
            for (int j = 0; j < i; ++j) {
                if (dp[j] && wordDictSet.find(s.substr(j, i - j)) != wordDictSet.end()) {
                    dp[i] = true;
                    break;
                }
            }
        }

        return dp[s.size()];
    }
};
```
但比较有意思的是，我把这段程序拿来提交，结果这个程序的运行时间反而比我自己写的程序要高。也许是官方题解做法中的子串方法提升了实际处理的时间。
官方题解最后提到本题实际上用Trie字典树的方法做会更快，不过这里属于是NLP的专业内容了，这里就没必要继续展开了。
