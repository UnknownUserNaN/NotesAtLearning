给你一个整数数组 `nums` 和一个整数 `k` ，请你统计并返回 *该数组中和为 `k`*\* \*\*的子数组的个数 \*。

子数组是数组中元素的连续非空序列。

***

最简单的思路肯定是沿用前面滑动窗口的想法。枚举每个窗口开始的位置，然后不断扩大窗口，直至窗口内数值的总和达到要求。但由于这里的数组里有可能出现负数，因此这里窗口扩大的过程不得不一直扩展到数组最右侧。这显然是低效的。因此还需要优化。

但是该怎么优化呢？这里我查看了题目的提示，最后一条提示中提到了一条重要的性质，即从第`$i$`个元素到第`$j$`个元素的和，相当于从第`$0$`个元素到第`$j$`个元素的和，减去第`$0$`个元素到第`$i$`个元素的和。如果我们能够把输入数组的**累加和**求出来，那么找出所有满足要求的子数组，便转化为在累加和数组中找到对应的两个数，使其差满足要求。

想到这里，问题就简单了许多。接下来我们只需要遍历数组，遍历到一个数值`$x$`时，只需要在该数值的右侧寻找有多少个数值`$x+k$`即可。这一步可以借助count函数完成。

基于上面的这个想法就能写出下面的代码：

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        // 建立累加和数组
        vector<int> cumsum;
        cumsum.push_back(0); // 这里填入的是累加和S_0
        for (int i = 0; i < nums.size(); i++) {
            cumsum.push_back(cumsum[i] + nums[i]); // 这里填入的是累加和S_n(n>1)
        }

        // 按照累加和的性质，遍历该数组
        // 若遍历到的值为x，则应在数组中寻找y=x+k
        int counter = 0;
        for (int i = 0; i < cumsum.size(); i++) {
            counter += count(cumsum.begin() + i + 1, cumsum.end(), cumsum[i] + k);
        }

        return counter;
    }
};
```

这次的提交直接AC了。但遗憾的是，无论是耗时还是耗内存，这个方法都比较高。看来还需要找方法继续优化。

这时候我就想到了提示三，也就是倒数第二条提示。这条提示说，可以将各累加和的频次存储到哈希表里，因为这里我们并不关注究竟是哪两个数值可以求差得到指定值，只想知道有几个满足需求的值。因此这里没必要使用count函数寻找这个值，而可以通过预先建立各累加和的频次哈希表。

但是有了这个思路，我却没有写出代码来。这是因为我发现这里建立的频次哈希表实际上是整个累加和数组的频次统计，但是遍历数组时，我们需要的是出现在右侧的累加和数值的频次信息，而非整个数组的频次信息。这里究竟该怎么解决呢？我百思不得其解，最终只能求助于题解。

题解对这个问题的解答是，我不去寻找出现在右侧的数值`$x+k$`，而是在遍历过之后，寻找出现在左侧的数值`$x$`，也可以记成`$y-k$`。一边求和，一边统计频次信息并建立哈希表。

```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        // 建立累加和数组并寻找出现在左侧的值x=y-k
        int counter = 0;
        int cumsum = 0;
        unordered_map<int, int> cumsum_freq;
        cumsum_freq[0]++;
        for (int i = 0; i < nums.size(); i++) {
            // 一边算累加和
            cumsum += nums[i];

            // 一边根据已有的左侧信息，寻找有没有符合要求的累加和
            counter += cumsum_freq[cumsum - k];

            // 一边记录频次信息
            cumsum_freq[cumsum]++;
        }

        return counter;
    }
};
```

总结一下这题做完后的经验：

1.当涉及到数组中某几个相邻数的求和时，可以考虑前缀和公式`$S_{ij}=S_j-S_i$`，有一种茅塞顿开的感觉。

2.某些中间量可以在数组遍历的时候顺带求得，比如在求数组和的时候，某些中间量可以顺带应用一下。

3.有些时候，当顺着一个方向遍历找某一个值很困难时，可以考虑换一个方向。

4.真正优秀的算法都是靠不断找问题、不断迭代出来的。
