给定一个链表的头节点  head ，返回链表开始入环的第一个节点。 如果链表无环，则返回 null。

如果链表中有某个节点，可以通过连续跟踪 next 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 pos 来表示链表尾连接到链表中的位置（索引从 0 开始）。如果 pos 是 -1，则在该链表中没有环。注意：pos 不作为参数进行传递，仅仅是为了标识链表的实际情况。

不允许修改 链表。

进阶：你是否可以使用 O(1) 空间解决此题？

---
核心思路仍然是在快慢指针法上改进。当快指针与慢指针相遇时，快指针肯定已经“套”了慢指针几圈，两者之间的路程差一定是环长度的倍数。而我们又知道，快指针的速度是慢指针的两倍，因此当两者相遇时，快指针走过的路程一定是慢指针的两倍。
但是基于这些信息，我们能如何求出问题的答案呢？我不知道，于是又只能去求助官方题解。
官方题解在上面的思考上更进一步了。首先，他们把这个情景的示意图画了出来：
![[Pasted image 20250402131256.png]]
快慢指针在图中的紫色点相遇。假设相遇时快指针的路程是$l_f$，慢指针的路程是$l_s$，那么根据前面的分析，显然有$l_f=2l_s$。
而根据图中的表示，我们可以发现$l_f$和$l_s$可以用上面图中的量表示。也就是说，$l_s=a+b$，$l_f=a+n(b+c)+b$，其中$n$是快指针“套”的圈数。
如果能求出$a$，我们就能知道进入环路的点在哪里。于是整理上式，可得：$a=c+(n-1)(b+c)$。
式中的$b+c$是环的长度，也就是一圈的长度。这个式子启发我们，如果在链表的起点和相遇点（紫色点）同时释放两个“速度”相同的指针，那么它们必然会在入环点相遇（即使过程中会经历套圈）。
基于这样的推理，我们就很容易写出程序了：
```cpp
class Solution {
public:
    ListNode* detectCycle(ListNode* head) {
        ListNode *slow = head, *fast = head;
        bool flag = false;
        while (fast != nullptr) {
            slow = slow->next;
            if (fast->next == nullptr) {
                return nullptr;
            }
            fast = fast->next->next;
            if (fast == slow) {
                flag = true;
                break;
            }
        }

        if (flag == false)
            return nullptr;
        else {
            ListNode* ptr = head;
            while (ptr != slow) {
                ptr = ptr->next;
                slow = slow->next;
            }
            return ptr;
        }
    }
};
```
当然，如果系统最终没有检查链表内容的话，其实也可以用之前的“歪门邪道法”，通过速度更快。
