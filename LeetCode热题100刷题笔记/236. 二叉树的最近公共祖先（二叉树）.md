给定一个二叉树, 找到该树中两个指定节点的最近公共祖先。

百度百科中最近公共祖先的定义为：“对于有根树 T 的两个节点 p、q，最近公共祖先表示为一个节点 x，满足 x 是 p、q 的祖先且 x 的深度尽可能大（一个节点也可以是它自己的祖先）。”

提示：
所有 Node.val 互不相同 。

---
很多二叉树问题都可以递归地考虑，本题也是如此。我们直接将目标问题拆解成左右子树上的子问题。从根节点出发，如果其任何一个孩子都不是p或者q，则转而在该节点的左右子树中执行上述过程；如果其有一个孩子为p或q，则说明该根节点参与了p和q结点之间的连接。这一个分支实际上还可以进一步划归，如果我们发现有一个结点r，它的其中一个孩子是p，那么p和q最近的公共祖先实际上就是r和q最近的公共祖先。
基于上面的思路可以很快地写出程序：
```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */
class Solution {
public:
    TreeNode* origin;

    TreeNode* trace(TreeNode* root, TreeNode* p, TreeNode* q) {
        if ((root->left == p && root->right == q) ||
            (root->left == q && root->right == p))
            // 该结点的左右孩子就是p和q
            return root;
        else if ((root->left == p && root->right != q) ||
                 (root->left != q && root->right == p))
            // 该结点有一个孩子是p，此时可以划归成root与q的公共祖先问题
            return trace(origin, root, q); // 这里可以优化
        else if ((root->left == q && root->right != p) ||
                 (root->left != p && root->right == q))
            // 该结点有一个孩子是q，此时可以划归成root与p的公共祖先问题
            return trace(origin, root, p); // 这里可以优化
        else {
            // 该结点的两个孩子都不是p或者q，此时其必有一个子树中可以找到该节点
            TreeNode *ret_left = nullptr, *ret_right = nullptr;
            if (root->left != nullptr)
                ret_left = trace(root->left, p, q);
            if (root->right != nullptr)
                ret_right = trace(root->right, p, q);
            return ret_left == nullptr ? ret_right : ret_left;
        }
    }

    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        origin = root;
        return trace(root, p, q);
    }
};
```
但是真正运行之后才发现，上面的程序存在两个问题：
1. 上面的程序中，当找到p和q中其中一者时，立刻返回到整棵树的起点搜索剩余的元素。这种方法在实现上比较简单，但是这会造成许多重复性的遍历。因为遍历实际上完全没必要从整棵树的根开始，而只需要在“上一个分叉口“进行即可。不过这个问题不算关键，只是会拖慢运行效率。
2. 这个问题就比较关键了。运行测试用例发现，上面的程序无法处理当q（或p）是p（或q）的后代时的情况。这是因为，上面的程序在遇到p和q其中一者时，便默认该结点和其全部后代中不存在另一者，而是转而在其他的子树中寻找对应节点。这显然是非常不正确的。
无法，只得退回到最原始的形态。采用深度优先方式遍历寻找p和q，并记录其从整棵树的根到该结点的路径，然后比较这两个路径在何时产生分歧。这样的程序可以很容易地写出来：
```cpp
class Solution {
    vector<TreeNode*> p_path, q_path;

public:
    void dfs(TreeNode* root, TreeNode* p, TreeNode* q,
             vector<TreeNode*> path) {
        if (root == nullptr)
            return;

        path.push_back(root);
        if (root == p)
            p_path = path;
        else if (root == q)
            q_path = path;
        
        dfs(root->left, p, q, path);
        dfs(root->right, p, q, path);
    }

    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        vector<TreeNode*> path;
        dfs(root, p, q, path);

        int i = 0;
        int min_len = min(p_path.size(), q_path.size());
        while (i < min_len && p_path[i] == q_path[i])
            i++;
        return p_path[i - 1];
    }
};
```
但是这个做法相当简单粗暴。虽然时间复杂度会达到$O(n)$的水平，但是空间复杂度却是近乎指数级的膨胀，因为函数`dfs`的参数path特意没有使用引用传递，而是值传递，这意味着每次进行一次递归，都要创建一个path的拷贝。这样，深度为$N$的结点会创建长度为$N+1$的path数组，而这样深度为$N$的结点大致有$2^N$个，因此遍历整棵树的空间复杂度即达到$O(N2^N)$的层级（等差乘等比数列的求和），这个空间消耗是非常惊人的。
当然，上面的做法也可以进行一定程度上的优化。比如利用题目中各结点的存储值均不相同的条件，设置一个哈希表（因为可以实现$O(1)$时间的查找），存储某一节点的父节点。在深度遍历整棵树的时候，我们不断“丰富”这个哈希表。找到p和q之后，再按照这个哈希表不断自叶子向根”跳转“直至找到两者的第一个公共结点。找公共结点的方法可以参考之前判断链表是否相交的思路。这样就免去了使用数组path和它的拷贝记录遍历路径的尴尬，空间复杂度可以降低很多：
```cpp
class Solution {
    unordered_map<int, TreeNode*> parent;

public:
    void dfs(TreeNode* root, TreeNode* prev) {
        if (root == nullptr)
            return;

        parent[root->val] = prev;
        dfs(root->left, root);
        dfs(root->right, root);
    }

    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 深度遍历整个树，记录每个结点的父节点
        dfs(root, nullptr);

        // 采用类似于寻找链表相交的方法，寻找两个指针的第一个公共结点
        TreeNode *pp = p, *pq = q;
        while (pp != pq) {
            pp = parent[pp->val] == nullptr ? p : parent[pp->val];
            pq = parent[pq->val] == nullptr ? q : parent[pq->val];
        }
        return pp;
    }
};
```
不过很遗憾，这个写法又不出意外地超时了。因为这个寻找公共结点的做法虽然空间复杂度很低，但是在p和q深度差距较大时，这个循环的时间复杂度会远高于$O(N)\approx O(\log n)$（这里$N$是树的深度，使用$N\approx \log n$来近似估计）。无法，只得“空间换时间”：
```cpp
/**
 * Definition for a binary tree node.
 * struct TreeNode {
 *     int val;
 *     TreeNode *left;
 *     TreeNode *right;
 *     TreeNode(int x) : val(x), left(NULL), right(NULL) {}
 * };
 */
class Solution {
    unordered_map<int, TreeNode*> parent;

public:
    void dfs(TreeNode* root, TreeNode* prev) {
        if (root == nullptr)
            return;

        parent[root->val] = prev;
        dfs(root->left, root);
        dfs(root->right, root);
    }

    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        // 深度遍历整个树，记录每个结点的父节点
        dfs(root, nullptr);

        // 记录下两结点到达根节点的路径
        vector<TreeNode*> path_p, path_q;
        TreeNode *pp = p, *pq = q;
        while (pp != nullptr) {
            path_p.emplace_back(pp);
            pp = parent[pp->val];
        }
        while (pq != nullptr) {
            path_q.emplace_back(pq);
            pq = parent[pq->val];
        }

        // 从根向叶子遍历两个路径，直至遇到最后一个相同的结点
        reverse(path_p.begin(), path_p.end());
        reverse(path_q.begin(), path_q.end()); // 改用双向队列或从数组末尾开始遍历可以降低这部分倒序的时间
        int i = 0;
        int min_len = min(path_p.size(), path_q.size());
        while (i < min_len && path_p[i] == path_q[i])
            i++;

        return path_p[i - 1];
    }
};
```
这样，时间复杂度可以大致降低到$O(n)$的水平，空间复杂度也基本可以维持在$O(n)$的水平。
做到这里，难道我们一开始的基于递归的思路就没有可能了吗？显然不是。[官方题解](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/solutions/238552/er-cha-shu-de-zui-jin-gong-gong-zu-xian-by-leetc-2/)给出了一个更为缜密的思考：

> 我们递归遍历整棵二叉树，定义 $f_x$ 表示 x 节点的子树中是否包含 p 节点或 q 节点，如果包含为 true，否则为 false。那么符合条件的最近公共祖先 x 一定满足如下条件：
> $(f_{lson} \&\& f_{rson}) ∣∣ ((x = p ∣∣ x = q) \&\& (f_{lson}​ ∣∣ f_{rson}))$
> 其中 $f_{lson}$ 和 $f_{rson}$ 分别代表 x 节点的左孩子和右孩子。初看可能会感觉条件判断有点复杂，我们来一条条看，$f_{lson}$ 和 $f_{rson}$  说明左子树和右子树均包含 p 节点或 q 节点，如果左子树包含的是 p 节点，那么右子树只能包含 q 节点，反之亦然，因为 p 节点和 q 节点都是不同且唯一的节点，因此如果满足这个判断条件即可说明 x 就是我们要找的最近公共祖先。
> 再来看第二条判断条件，这个判断条件即是考虑了 x 恰好是 p 节点或 q 节点且它的左子树或右子树有一个包含了另一个节点的情况，因此如果满足这个判断条件亦可说明 x 就是我们要找的最近公共祖先。
> 你可能会疑惑这样找出来的公共祖先深度是否是最大的。其实是最大的，因为我们是自底向上从叶子节点开始更新的，所以在所有满足条件的公共祖先中一定是深度最大的祖先先被访问到，且由于$f_x$ 本身的定义很巧妙，在找到最近公共祖先 x 以后，$f_x$ 按定义被设置为 true ，即假定了这个子树中只有一个 p 节点或 q 节点，因此其他公共祖先不会再被判断为符合条件。

该思路的核心是，注意到**符合要求的结点x，其左子树和右子树中必然各自存在p或q，或者自身必然是p或q且其中一个子树中存在p或q**，也就是将子问题从一开始我们思路中的”**左右孩子是否是p或q**“转化成了”**左右子树中是否包含p或q**“。
官方题解的写法是这样的：
```cpp
class Solution {
public:
    TreeNode* ans;
    bool dfs(TreeNode* root, TreeNode* p, TreeNode* q) {
        if (root == nullptr) return false;
        bool lson = dfs(root->left, p, q);
        bool rson = dfs(root->right, p, q);
        if ((lson && rson) || ((root->val == p->val || root->val == q->val) && (lson || rson))) {
            ans = root;
        } 
        return lson || rson || (root->val == p->val || root->val == q->val);
    }
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        dfs(root, p, q);
        return ans;
    }
};
```
评论区有人给出了思路类似，但更为简洁的写法：
> 如果当前节点是 null 或者是目标节点之一（p 或 q），直接返回当前节点。
> 否则递归左右子树：
> 	左子树返回值为 l，右子树返回值为 r。
> 	根据左右子树的返回值判断：
> 		如果左子树返回 null，说明 p 和 q 都在右子树中，返回右子树的结果。
> 		如果右子树返回 null，说明 p 和 q 都在左子树中，返回左子树的结果。
> 		如果左右子树都不为 null，说明当前节点就是最近公共祖先，返回当前节点。

```cpp
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        if (root == nullptr || root == p || root == q)
            return root;

        TreeNode *ret_left, *ret_right;
        ret_left = lowestCommonAncestor(root->left, p, q);
        ret_right = lowestCommonAncestor(root->right, p, q);

        if (ret_left == nullptr)
            // 左子树中不包含p和q，说明p和q都在右子树中，返回右子树的结果
            return ret_right;
        else if (ret_right == nullptr)
            // 右子树中不包含p和q，说明p和q都在左子树中，返回左子树的结果
            return ret_left;
        else
            // 左右子树中都分别包含p和q，说明当前结点就是所求的结点
            return root;
    }
};
```
这个做法最精妙的一点就是将所有操作都整合到了题目所设计的函数中。函数的设计思路与官方题解的想法类似，但是这里更为巧妙的是，作者设法让题目所要求的函数直接承担了官方题解中dfs函数的作用。具体的返回值也是类似的，只不过作者使用返回值是否为空指针代替了子树中是否包含p或q这个布尔值的结果。如果该节点的左右子树中不包含p或q，则设法令其返回null；如果包含，则返回自身。这个包含的结果会随着深度遍历的过程不断“回传”并上调，最终在整个的根节点处完成“整合”。
[另一篇题解](https://leetcode.cn/problems/lowest-common-ancestor-of-a-binary-tree/solutions/240096/236-er-cha-shu-de-zui-jin-gong-gong-zu-xian-hou-xu/)给出了更为详细的分析（这个递推的分析思路很值得学习）：

> 考虑通过递归对二叉树进行先序遍历，当遇到节点 p 或 q 时返回。从底至顶回溯，当节点 p,q 在节点 root 的异侧时，节点 root 即为最近公共祖先，则向上返回 root 。
> 递归解析：
> 1. 终止条件：
> 	当越过叶节点，则直接返回 null ；
> 	当 root 等于 p,q ，则直接返回 root ；
> 2. 递推工作：
> 	开启递归左子节点，返回值记为 left ；
> 	开启递归右子节点，返回值记为 right ；
> 3. 返回值：根据 left 和 right ，可展开为四种情况：
> 	a. 当 left 和 right 同时为空 ：说明 root 的左 / 右子树中都不包含 p,q ，返回 null ；
> 	b. 当 left 和 right 同时不为空 ：说明 p,q 分列在 root 的 异侧 （分别在 左 / 右子树），因此 root 为最近公共祖先，返回 root ；
> 	c. 当 left 为空 ，right 不为空 ：p,q 都不在 root 的左子树中，直接返回 right 。具体可分为两种情况：
> 		p,q 其中一个在 root 的 右子树 中，此时 right 指向 p（假设为 p ）；
> 		p,q 两节点都在 root 的 右子树 中，此时的 right 指向 最近公共祖先节点 ；
> 	d. 当 left 不为空 ， right 为空 ：与情况 c. 同理；

严格按照上面的分析，写出来的代码是这样的：
```cpp
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        if(root == nullptr || root == p || root == q) return root;
        TreeNode *left = lowestCommonAncestor(root->left, p, q);
        TreeNode *right = lowestCommonAncestor(root->right, p, q);
        if(left == nullptr && right == nullptr) return nullptr; // a.
        if(left == nullptr) return right; // c.
        if(right == nullptr) return left; // d.
        return root; // b. if(left != null and right != null)
    }
};
```
当然后来发现情况a实际上可以整合到c和d的分支中，于是就可以写出刚才那种简洁的写法：
```cpp
class Solution {
public:
    TreeNode* lowestCommonAncestor(TreeNode* root, TreeNode* p, TreeNode* q) {
        if(root == nullptr || root == p || root == q) return root;
        TreeNode *left = lowestCommonAncestor(root->left, p, q);
        TreeNode *right = lowestCommonAncestor(root->right, p, q);
        if(left == nullptr) return right;
        if(right == nullptr) return left;
        return root;
    }
};
```
整个算法的时间复杂度和空间复杂度都可以达到$O(n)$级别。
