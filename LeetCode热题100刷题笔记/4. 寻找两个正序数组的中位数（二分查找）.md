给定两个大小分别为 m 和 n 的正序（从小到大）数组 nums1 和 nums2。请你找出并返回这两个正序数组的 中位数 。

算法的时间复杂度应该为 O(log (m+n)) 。

---
需要注意的是，我们需要返回的是这两个数组“合并”后的数组的中位数。因此，一种最简单的方法显然是直接合并这两个数组，但这显然不经济，因为这需要额外的空间。
我们需要回想中位数的性质。如果某数x是一个正序数组中的中位数，那么它一定处在这个数组最中间的位置，也就是其前方的数字个数一定等于其后方的数字个数。
这个性质应当在多个正序数组中也是成立的。如果数x是题中这两个正序数组的中位数，那么排在它“前面”的数字个数应当与排在它“后面”的数字个数相等。稍微定量一点考虑，假设在数组nums1中在x“前面”的数有a个，在其“后面”的数有b个，在数组nums2中在x“前面”的数有c个，在其“后面”的数有d个，那么应当有`a+c=b+d`。
但是，满足`a+c=b+d`的情况有许多个。可以想象在两个数组中都各自存在一个切分点，切分点分别将数组分别分成长度为a和b，以及c和d的两个小块。这两个切分点的位置可以随意移动。可以想象这两个切分点一开始处于这两个数组的正中间，这样有`a=b`和`c=d`，因此显然满足`a+c=b+d`，现在我们令某一个切分点向左移动若干个单位，然后令另一个向右移动同样的单位，这样实际上仍然满足`a+c=b+d`。这样做可以得到许多个合理的切分点，那这些切分点中究竟哪一者是符合要求的呢？
到这里我就想不到如何继续推理下去了，最后在[官方题解](https://leetcode.cn/problems/median-of-two-sorted-arrays/solutions/258842/xun-zhao-liang-ge-you-xu-shu-zu-de-zhong-wei-s-114/)的帮助下才进行了下去。
这里需要注意到中位数的另一个性质，即它将一个正序数组分割成两部分之后，一定有一个子数组中的所有数都大于等于另一个子数组中的所有数，也就是一定有一个子数组的最小值大于等于另一个子数组中的最大值。回到我们两个数组分割的情境中，随意放置两个符合`a+c=b+d`的分割点，两个分割点分别把两个数组分成了长度为a和b，以及c和d的四个子数组，其中我们可以确知，b数组的值将永远大于等于a数组，d数组的值将永远大于等于c数组。但是，一般情况下，d数组的值不一定都大于等于a数组，同样的，b数组的值不一定都大于等于c数组。然而，如果我们取到了中位数作为分割点，那么显然就能有d数组的值大于等于a数组的值，b数组的值大于等于c数组的值，这就是我们判断的条件。
我们已经形成了判断的初步直觉，现在我们来稍微“数学”地来完善一下上面的过程：
设`nums1`的长度为`m`，`nums2`的长度为`n`；两数组中存在的分割点分别为`i`和`j`，`i`将数组`nums1`分成`[0,i]`和`[i+1,m-1]`两个部分，`j`将数组`nums2`分成`[0,j]`和`[j+1,n-1]`两个部分。
`m`和`n`的取值不同，`i`和`j`的关系也略有差别。`m+n`为偶数时，中位数一定可以将合并后的数组分成等长的两个部分，所以四个子数组中，处于“左侧”的元素总数应当等于处于“右侧”的元素总数，也就是应有`(i+1)+(j+1)=(m-1-i)+(n-1-j)`，化简可得`i+j=(m+n-4)/2`。而当`m+n`为奇数时，中位数会将合并后的数组分成长度仅相差1的两个部分，我们不妨设左侧的部分长度更长，然后沿用上面的条件，可以类似地得出`(i+1)+(j+1)=(m-1-i)+(n-1-j)+1`，化简可得`i+j=(m+n-3)/2`。运用这两个等式，我们就可以通过一个分割点的位置得知另一个分割点的位置。
现在来检查各个子数组之间的大小关系。按照前面的推理，当`i`和`j`恰好能得到中位数的分割时，应当有`nums1[i+1]>=nums2[j]`和`nums2[j+1]>=nums1[i]`同时成立。如果发现两者中有一个条件不满足，就意味着我们要对`i`和`j`进行调整。比如我们发现`nums1[i+1]<nums2[j]`，这意味着我们要令`nums[i+1]`更大，`nums2[j]`更小，于是就需要我们设法增大`i`，减小`j`。具体增大`i`或者减小`j`的方法可以参考二分法，增大`i`，就令`i`下一次取为右侧区间的中点，`j`的取值靠上述等式确定。不断参照二分法进行调整，直至上述两个条件同时满足为止。
循环停止后，该如何确定中位数？对于`m+n`为偶数的情况，中位数一定是“左侧”区间中最大的数与“右侧”区间中最小的数的平均值。而对于`m+n`为奇数的情况，中位数一定是“左侧“区间中最大的数，因为我们事先假设了”左侧“区间的长度比”右侧“更长。这两个最值又该如何确定？实际上，”左侧“区间中最大的数就是`max(nums1[i],nums2[j])`，”右侧“区间中最小的数就是`min(nums1[i+1],nums2[j+1])`。
当然，还有比较恼人的边界条件，很有可能在某一次迭代中，`i`或者`j`移动到了数组的最尽头，这时某一个数组的“左侧”数组或“右侧”数组的长度就将变为0，也就是`i==-1||j==-1`，或者`i==m-1||j==n-1`，这时我们需要进行额外的特殊处理。当某数组左侧的子数组长度变为0时，其最大值将被视为负无穷，即`INT_MIN`；当某数组右侧的子数组长度变为0时，其最大值将被视为正无穷，即`INT_MAX`。
基于上面这些推理，可以写出下面的程序。说实在的，程序并不是那么“美观”。
```cpp
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        int m = nums1.size(), n = nums2.size();
        int l1 = -1, r1 = m; // 使用开区间表示
        int left1, left2, right1, right2;

        while (true) {
            int i = l1 + (r1 - l1) / 2; // 将nums1分成(-1, i]和(i, m)两部分
            int j;                      // j的取值通过等式推导
            if ((m + n) % 2 == 0) {
                // m+n是偶数
                j = (m + n - 4) / 2 - i; // 将nums2分成(-1, j]和(j, n)两部分
                j = j > n - 1 ? n - 1 : j <= -1 ? -1 : j; // 防止j的取值超出范围
                i = (m + n - 4) / 2 - j; // 如果j的位置调整过，i的位置也要进行调整
            } else {
                // m+n是奇数
                j = (m + n - 3) / 2 - i;
                j = j > n - 1 ? n - 1 : j <= -1 ? -1 : j;
                i = (m + n - 3) / 2 - j;
            }

            // 结合边界条件确定各子数组的边界值
            left1 = i <= -1 ? INT_MIN : nums1[i];
            left2 = j <= -1 ? INT_MIN : nums2[j];
            right1 = i >= m - 1 ? INT_MAX : nums1[i + 1];
            right2 = j >= n - 1 ? INT_MAX : nums2[j + 1];

            if (right1 < left2)
                // 需要设法令right1增大，i要增大
                // i = i+(r1-i)/2;
                l1 = i;
            else if (right2 < left1)
                // 需要设法令right2增大，j要增大，i要减小
                // i = l1+(i-l1)/2;
                r1 = i;
            else
                break;
        }

        if ((m + n) % 2 == 0)
            // m+n是偶数，此时应当返回左数组的最大值和右区间最小值的平均值
            return max(left1, left2) / 2.0 + min(right1, right2) / 2.0;
        else
            // m+n是奇数，此时应当返回左数组的最大值
            return max(left1, left2);
    }
};
```
官方题解的程序就写得简洁许多，也很容易能看出二分查找的框架。比上面我自己写的程序，有两点不太一样：
一是假设了`nums1`的长度必然大于`nums2`的长度，如果输入的二者不满足这个条件，则采用“递归”的方法，交换两个参数后再输出。这个预设能方便程序的编写。上面的程序没有满足这个预设，导致调整了`j`的位置之后还要再调整`i`的位置。
二是官方题解还将某些需要分情况讨论的问题合并成了一个。其中一个将`m+n`为奇数还是偶数的分支处理合并成了一个，二是将上面程序中讨论`right1<left2`和`right2<left1`的情形合并成了一个。这样写起来程序运行会更快。
```cpp
class Solution {
public:
    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        if (nums1.size() > nums2.size()) {
            return findMedianSortedArrays(nums2, nums1);
        }
        
        int m = nums1.size();
        int n = nums2.size();
        int left = 0, right = m;
        // median1：前一部分的最大值
        // median2：后一部分的最小值
        int median1 = 0, median2 = 0;

        while (left <= right) {
            // 前一部分包含 nums1[0 .. i-1] 和 nums2[0 .. j-1]
            // 后一部分包含 nums1[i .. m-1] 和 nums2[j .. n-1]
            int i = (left + right) / 2;
            int j = (m + n + 1) / 2 - i;

            // nums_im1, nums_i, nums_jm1, nums_j 分别表示 nums1[i-1], nums1[i], nums2[j-1], nums2[j]
            int nums_im1 = (i == 0 ? INT_MIN : nums1[i - 1]);
            int nums_i = (i == m ? INT_MAX : nums1[i]);
            int nums_jm1 = (j == 0 ? INT_MIN : nums2[j - 1]);
            int nums_j = (j == n ? INT_MAX : nums2[j]);

            if (nums_im1 <= nums_j) {
                median1 = max(nums_im1, nums_jm1);
                median2 = min(nums_i, nums_j);
                left = i + 1;
            } else {
                right = i - 1;
            }
        }

        return (m + n) % 2 == 0 ? (median1 + median2) / 2.0 : median1;
    }
};
```
根据官方题解的程序，容易分析出该方法的时间复杂度为$O(\log\min(m,n))$，因为我们主要是在较短的数组中执行二分查找；其空间复杂度为$O(1)$。
值得一提的是，官方题解中还有一种基于二分查找的、时间复杂度为$O(\log(m+n))$的算法。其基本思路是，将寻找合并数组中的中位数转化为寻找合并后长度为$L=m+n$的数组中第$L/2$小的数字。如何找到这个数字呢？这里也使用了一种“奇巧”的想法，分别找到`nums1`中第$L/4-1$个元素a，`nums2`中第$L/4-1$个元素b。可以确定的是，a与b中的较小者在合并后的数组中，其前方最多也只有$2\times(L/4-1)=L/2-2$个元素，因此两者中的较小者显然不是中位数。这个信息可以帮助我们缩小查找中位数的范围，因为这样较小者所在的数组之中的前$L/4-1$个数就一定不是中位数了。
```cpp
class Solution {
public:
    int getKthElement(const vector<int>& nums1, const vector<int>& nums2, int k) {
        /* 主要思路：要找到第 k (k>1) 小的元素，那么就取 pivot1 = nums1[k/2-1] 和 pivot2 = nums2[k/2-1] 进行比较
         * 这里的 "/" 表示整除
         * nums1 中小于等于 pivot1 的元素有 nums1[0 .. k/2-2] 共计 k/2-1 个
         * nums2 中小于等于 pivot2 的元素有 nums2[0 .. k/2-2] 共计 k/2-1 个
         * 取 pivot = min(pivot1, pivot2)，两个数组中小于等于 pivot 的元素共计不会超过 (k/2-1) + (k/2-1) <= k-2 个
         * 这样 pivot 本身最大也只能是第 k-1 小的元素
         * 如果 pivot = pivot1，那么 nums1[0 .. k/2-1] 都不可能是第 k 小的元素。把这些元素全部 "删除"，剩下的作为新的 nums1 数组
         * 如果 pivot = pivot2，那么 nums2[0 .. k/2-1] 都不可能是第 k 小的元素。把这些元素全部 "删除"，剩下的作为新的 nums2 数组
         * 由于我们 "删除" 了一些元素（这些元素都比第 k 小的元素要小），因此需要修改 k 的值，减去删除的数的个数
         */

        int m = nums1.size();
        int n = nums2.size();
        int index1 = 0, index2 = 0;

        while (true) {
            // 边界情况
            if (index1 == m) {
                return nums2[index2 + k - 1];
            }
            if (index2 == n) {
                return nums1[index1 + k - 1];
            }
            if (k == 1) {
                return min(nums1[index1], nums2[index2]);
            }

            // 正常情况
            int newIndex1 = min(index1 + k / 2 - 1, m - 1);
            int newIndex2 = min(index2 + k / 2 - 1, n - 1);
            int pivot1 = nums1[newIndex1];
            int pivot2 = nums2[newIndex2];
            if (pivot1 <= pivot2) {
                k -= newIndex1 - index1 + 1;
                index1 = newIndex1 + 1;
            }
            else {
                k -= newIndex2 - index2 + 1;
                index2 = newIndex2 + 1;
            }
        }
    }

    double findMedianSortedArrays(vector<int>& nums1, vector<int>& nums2) {
        int totalLength = nums1.size() + nums2.size();
        if (totalLength % 2 == 1) {
            return getKthElement(nums1, nums2, (totalLength + 1) / 2);
        }
        else {
            return (getKthElement(nums1, nums2, totalLength / 2) + getKthElement(nums1, nums2, totalLength / 2 + 1)) / 2.0;
        }
    }
};
```
