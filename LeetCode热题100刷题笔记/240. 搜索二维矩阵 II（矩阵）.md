编写一个高效的算法来搜索 m x n 矩阵 matrix 中的一个目标值 target 。该矩阵具有以下特性：

每行的元素从左到右升序排列。
每列的元素从上到下升序排列。

---
看到这个问题，首先想到的是经典的二分法。但是后来一想，这个问题中涉及的是二维情形的二分法，处理起来较为复杂。最主要的就是要处理搜索区域的四个角点，逻辑较为复杂，所以暂时没想出什么比较好的思路，最后只能看[官方题解](https://leetcode.cn/problems/search-a-2d-matrix-ii/solutions/1062538/sou-suo-er-wei-ju-zhen-ii-by-leetcode-so-9hcx/)。
官方的题解中介绍了三种方法：第一种是暴力解法，时间复杂度是$O(mn)$；第二种是逐行或逐列使用二分法，时间复杂度降到$O(m\log n)$或$O(n\log m)$；第三种就是复杂度较低的“Z字形搜索法”，将时间复杂度降到了$O(m+n)$。
具体的思路与一开始自己想的很类似，区别在于搜索的起始点不同。通常情况下的搜索起始点往往选在矩阵的左上角$(0,0)$或者右下角$(m-1, n-1)$，而此种办法则是从右上角$(0,n-1)$开始搜索。在每一步的搜索过程中，如果我们位于位置$(x,y)$，那么我们希望在以整个矩阵的左下角为搜索区域的左下角、以 $(x,y)$ 为搜索区域的右上角的矩阵中进行搜索，即行的范围为$[x,m−1]$，列的范围为 $[0,y]$。
> 这样，在每一次的循环中：
> 1. 如果 matrix[x,y]=target，说明搜索完成；
> 2. 如果 matrix[x,y]>target，由于每一列的元素都是升序排列的，那么在当前的搜索矩阵中，所有位于第 y 列的元素都是严格大于 target 的，因此我们可以将它们全部忽略，即将 y 减少 1；
> 3. 如果 matrix[x,y]<target，由于每一行的元素都是升序排列的，那么在当前的搜索矩阵中，所有位于第 x 行的元素都是严格小于 target 的，因此我们可以将它们全部忽略，即将 x 增加 1。
 在搜索的过程中，如果我们超出了矩阵的边界，那么说明矩阵中不存在 target。
 
 对称地，上面的搜索步骤也可以从矩阵的左下角开始。
 为什么不从矩阵的左上角或者右下角开始搜索？对比上面的搜索过程就能发现，如果从矩阵的左上角（或者右下角）开始，那么一旦发现matrix[x,y]<target（或者matrix[x,y]>target），我们将无法确定target是在当前位置的右侧还是下侧（或者左侧还是上侧）。从左下角或者右上角开始搜索，使得大小关系可以跟行进方向一一对应起来，这种“主动破坏对称性”的方法很值得学习。
 基于这个思路，很容易写出如下的代码：
```cpp
class Solution {
public:
    bool searchMatrix(vector<vector<int>>& matrix, int target) {
        int m = matrix.size();
        int n = matrix[0].size();

        // 开始按题解方法搜索
        int x = 0, y = n - 1;
        while (x < m && y >= 0) { // 搜索位置在矩阵内
            if (matrix[x][y] == target)
                return true;
            else if (matrix[x][y] > target)
                y--;
            else if (matrix[x][y] < target)
                x++;
        }
        return false;
    }
};
```
值得一提的是，官方题解下面的评论提到了一种解释上面方法的另一种思路，即：
>这个把他看做一个二叉搜索树，其中右上角点是根节点，往下是右子树，往左是左子树。
