数字 n 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且 有效的 括号组合。

---
很幸运这个问题归在了“回溯”这个板块之下，否则我还真不容易想到这个问题的解法。
首先思考，给定括号对数，如何按一定规律构造出出所有可能的组合？实际上，对于要插入的第$i$组括号，我们可以考虑在插入它的左半括号之前，先将其之前的若干个括号闭合掉，也就是插入对应的右半括号。对每一组的括号插入都重复如此步骤，并记录下此时未能闭合的括号数。在插入完所有组的左半括号之后，再一次性把所有未能闭合的括号都一次性闭合掉，然后将字符串插入结果数组。
这个思路可能稍有些绕，但是写成程序之后还是比较清晰的：
```cpp
class Solution {
public:
    void backtracking(vector<string>& ret, string& s, int cur, int left,
                      int n) {
        if (cur == n) {
            for (int j = 0; j < left; j++)
                s += ")";
            ret.push_back(s);
            for (int j = 0; j < left; j++)
                s.pop_back();
            return;
        }

        for (int i = 0; i <= left; i++) {
            for (int j = 0; j < i; j++)
                s += ")";

            s += "(";
            backtracking(ret, s, cur + 1, left - i + 1, n);

            s.pop_back();
            for (int j = 0; j < i; j++)
                s.pop_back();
        }
    }

    vector<string> generateParenthesis(int n) {
        vector<string> ret;
        string s;
        backtracking(ret, s, 0, 0, n);
        return ret;
    }
};
```
[官方题解](https://leetcode.cn/problems/generate-parentheses/solutions/192912/gua-hao-sheng-cheng-by-leetcode-solution/)的写法则更为简洁。具体来说，它的主体思路是“如果左括号数量不大于 n，我们可以放一个左括号。如果右括号数量小于左括号的数量，我们可以放一个右括号。”程序的其余部分依然是标准的回溯模板：
```cpp
class Solution {
    void backtrack(vector<string>& ans, string& cur, int open, int close, int n) {
        if (cur.size() == n * 2) {
            ans.push_back(cur);
            return;
        }
        if (open < n) {
            cur.push_back('(');
            backtrack(ans, cur, open + 1, close, n);
            cur.pop_back();
        }
        if (close < open) {
            cur.push_back(')');
            backtrack(ans, cur, open, close + 1, n);
            cur.pop_back();
        }
    }
public:
    vector<string> generateParenthesis(int n) {
        vector<string> result;
        string current;
        backtrack(result, current, 0, 0, n);
        return result;
    }
};
```
这个程序的时间复杂度分析比较麻烦，官方题解直接省略推导过程指出这个算法的渐进复杂度与卡特兰数有关，大致为$\displaystyle O(\frac{4^n}{\sqrt{n}}))$。空间复杂度大致为$O(n)$。
官方题解还提供了一种基于递归的思路。它的时间复杂度与回溯法相同，但是空间复杂度能降低到$\displaystyle O(\frac{4^n}{\sqrt{n}}))$。
> 任何一个括号序列都一定是由 ‘(’ 开头，并且第一个 ‘(’ 一定有一个唯一与之对应的 ‘)’。这样一来，每一个括号序列可以用 (a)b 来表示，其中 a 与 b 分别是一个合法的括号序列（可以为空）。
> 那么，要生成所有长度为 2n 的括号序列，我们定义一个函数 generate(n) 来返回所有可能的括号序列。那么在函数 generate(n) 的过程中：
> 	我们需要枚举与第一个 ‘(’ 对应的 ‘)’ 的位置 2i+1；
> 	递归调用 generate(i) 即可计算 a 的所有可能性；
> 	递归调用 generate(n−i−1) 即可计算 b 的所有可能性；
> 	遍历 a 与 b 的所有可能性并拼接，即可得到所有长度为 2n 的括号序列。
> 为了节省计算时间，我们在每次 generate(i) 函数返回之前，把返回值存储起来，下次再调用 generate(i) 时可以直接返回，不需要再递归计算。

```cpp
class Solution {
    shared_ptr<vector<string>> cache[100] = {nullptr};
public:
    shared_ptr<vector<string>> generate(int n) {
        if (cache[n] != nullptr)
            return cache[n];
        if (n == 0) {
            cache[0] = shared_ptr<vector<string>>(new vector<string>{""});
        } else {
            auto result = shared_ptr<vector<string>>(new vector<string>);
            for (int i = 0; i != n; ++i) {
                auto lefts = generate(i);
                auto rights = generate(n - i - 1);
                for (const string& left : *lefts)
                    for (const string& right : *rights)
                        result -> push_back("(" + left + ")" + right);
            }
            cache[n] = result;
        }
        return cache[n];
    }
    vector<string> generateParenthesis(int n) {
        return *generate(n);
    }
};
```
这里需要注意的是上面程序中的特别写法，也就是用上了shared_ptr实现缓存（cache），来存储运算过程中的某些可能需要重复利用的计算结果。下面直接让D老师总结使用这一方法的作用：
> 在LeetCode的“括号生成”题解中，`shared_ptr`的使用主要服务于**记忆化（缓存）**和**高效共享数据**两个核心目的。以下是详细解释：
> 
> 1. **记忆化（缓存）的实现**
> - **`cache`数组**：类成员`shared_ptr<vector<string>> cache[100]`用于存储已计算的结果。例如，`cache[n]`保存`n`对括号生成的所有有效组合。
> - **避免重复计算**：递归过程中，若`cache[n]`已存在（非`nullptr`），直接返回缓存结果，避免重复展开递归。
> 
> 2. **`shared_ptr`的作用**
> - **内存管理**：`shared_ptr`是智能指针，通过引用计数自动释放内存，避免手动管理导致的泄漏。
> - **共享数据**：递归过程中，不同分支可能多次请求同一`n`的结果（如`generate(0)`会被多次调用）。使用`shared_ptr`允许不同调用共享同一份数据，避免拷贝`vector<string>`的开销。
> - **生命周期控制**：缓存的结果可能被多个调用方同时引用（如`cache[1]`被多个更大的`n`依赖）。`shared_ptr`确保所有引用释放后才销毁对象，避免悬空指针。
> 
> 3. **为何不用`unique_ptr`或原始指针？**
> - **`unique_ptr`**：所有权唯一，无法被多个调用方共享（如`cache[n]`存入后，返回时会转移所有权，导致缓存失效）。
> - **原始指针**：需手动管理内存，易出错且代码复杂。
> - **`shared_ptr`**：天然支持共享所有权，完美适配缓存场景。
> 
> 4. **代码流程示例**
> - **首次计算`n=2`**：
>   1. 分解为`i=0`和`n-i-1=1`。
>   2. 递归调用`generate(0)`和`generate(1)`。
>   3. `generate(0)`缓存结果`[""]`，`generate(1)`缓存结果`["()"]`。
>   4. 组合生成`["()()", "(())"]`，存入`cache[2]`。
> - **后续调用`n=2`**：直接返回`cache[2]`，无需计算。
> 
> 5. **性能与安全**
> - **减少拷贝**：直接传递指针，避免复制`vector`（尤其是结果较多时）。
> - **自动释放**：程序结束时，所有`shared_ptr`引用计数归零，缓存数据自动释放。
> 
> 总结
> `shared_ptr`在此代码中通过共享和自动管理内存，实现了高效且安全的记忆化递归，是优化时间与空间复杂度的关键设计。
