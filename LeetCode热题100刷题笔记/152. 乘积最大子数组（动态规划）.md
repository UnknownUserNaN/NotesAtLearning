给你一个整数数组 nums ，请你找出数组中乘积最大的非空连续 子数组（该子数组中至少包含一个数字），并返回该子数组所对应的乘积。

测试用例的答案是一个 32-位 整数。

提示：nums 的任何子数组的乘积都 保证 是一个 32-位 整数

---
此题与“300. 最长递增子数组”的思路类似。仍然是使用动态规划思路。建立数组`dp[i]`，其元素的值表示遍历至此的子数组的乘积最大子数组的积。可以看出，`dp[i]`只能通过`dp[i-1]`的状态转移得到，因为题目中要求的是连续的子数组。那么转移的条件是什么呢？显然是当前位置的数组在求积之后能使之前`dp[i-1]`的结果变得更大。如果当前位置的结果不会使其变得更大，即`nums[i]<1`，那我们不如将这个“状态传导”的过程“截断”，直接令`dp[i]=nums[i]`。随后我们遍历整个`dp`数组，找到其中的最大值返回即可。
可以写出下面的程序：
```cpp
class Solution {
public:
    int maxProduct(vector<int>& nums) {
        if (nums.size() == 1)
            return nums[0];

        vector<int> dp(nums.size());
        dp[0] = nums[0];
        for (int i = 1; i < nums.size(); i++) {
            if (dp[i - 1] * nums[i] > dp[i - 1])
                dp[i] = dp[i - 1] * nums[i];
            else
                dp[i] = nums[i];
        }
        return *max_element(dp.begin(), dp.end());
    }
};
```
结果不出意外地，从下面这个用例上掉下来了：
```cpp
[-2, 3, -4]
```
按上面的程序，程序会直接在遍历到3时进行“截断”。因为程序认为，3并不能使此前的片段积-2增大。由于负数的出现，会导致之前的最大变成最小，导致后续可能出现的最大变成负数。草率地在某一个节点进行截断，这是相当“贪心”的一种行为，因为如果后面遇到了更多的负数，那么“负负得正”，反而会使得片段积增大。
那么这里我们该如何改进呢？[官方题解](https://leetcode.cn/problems/maximum-product-subarray/solutions/250015/cheng-ji-zui-da-zi-shu-zu-by-leetcode-solution/)的思路是，将遇到的正数和负数分开讨论：
考虑当前位置如果是一个负数的话，那么我们希望以它前一个位置结尾的某个段的积也是个负数，这样就可以负负得正，并且我们希望这个积尽可能「负得更多」，即尽可能小。如果当前位置是一个正数的话，我们更希望以它前一个位置结尾的某个段的积也是个正数，并且希望它尽可能地大。于是这里我们可以再维护一个$f_{min}(i)$，它表示以第$i$个元素结尾的乘积最小子数组的乘积，那么我们可以得到这样的动态规划转移方程：
$$\begin{aligned}
f_{max}(i)&=\max_{i=1}^{n}\left\{f_{max}(i-1)\cdot a_i, f_{min}(i-1)\cdot a_i, a_i\right\}\\
f_{min}(i)&=\min_{i=1}^{n}\left\{f_{max}(i-1)\cdot a_i, f_{min}(i-1)\cdot a_i, a_i\right\}
\end{aligned}
$$
最终的输出结果就是在$f_{max}$数组中找最大值即可。
```cpp
class Solution {
public:
    int maxProduct(vector<int>& nums) {
        vector <long> maxF(nums.begin(),nums.end()), minF(nums.begin(), nums.end());
        for (int i = 1; i < nums.size(); ++i) {
            maxF[i] = max(maxF[i - 1] * nums[i], max((long)nums[i], minF[i - 1] * nums[i]));
            minF[i] = min(minF[i - 1] * nums[i], min((long)nums[i], maxF[i - 1] * nums[i]));
            if(minF[i]<INT_MIN) {
                minF[i]=nums[i];
            }
        }
        return *max_element(maxF.begin(), maxF.end());
    }
};
```
该程序的时间复杂度为$O(n)$，空间复杂度为$O(n)$。
值得一提的是，评论区有人给出了空间复杂度降到$O(1)$的算法：
```cpp
class Solution {
public:
    int maxProduct(vector<int> nums) {
        int product = 1, n = nums.size();
        int max_val = nums[0];

        for (int i = 0; i < n; i++) {
            product *= nums[i];
            max_val = max(max_val, product);
            if (nums[i] == 0) {
                product = 1;
            }
        }

        product = 1;
        for (int i = n - 1; i >= 0; i--) {
            product *= nums[i];
            max_val = max(max_val, product);
            if (nums[i] == 0) {
                product = 1;
            }
        }

        return max_val;
    }
};
```
有人解释了他的做法：
> 按照我的理解，解释一下大佬的算法：
> 首先假设，不会出现 0 。
> - a、整个数列中偶数个负数，结果一定是全部数相乘
> - b、整个数列中奇数个负数，结果一定是某一个负数的左右子数列相乘，所以正反都按照暴力解决，所以正反求解，一定有结果。（因为此时，正向会被求成负数的区间，在逆向后一定会被求成正数，此时正反两次就可以取到正确的子区间最大值）
> 接着，假设会出现0。把整个数列，想象成按照0分割的多个子数列，按照不会出现0去暴力解决，最终汇总所有子数列的最大值。
