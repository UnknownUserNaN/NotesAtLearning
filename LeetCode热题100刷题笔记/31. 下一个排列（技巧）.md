整数数组的一个 排列  就是将其所有成员以序列或线性顺序排列。

例如，arr = [1,2,3] ，以下这些都可以视作 arr 的排列：[1,2,3]、[1,3,2]、[3,1,2]、[2,3,1] 。
整数数组的 下一个排列 是指其整数的下一个字典序更大的排列。更正式地，如果数组的所有排列根据其字典顺序从小到大排列在一个容器中，那么数组的 下一个排列 就是在这个有序容器中排在它后面的那个排列。如果不存在下一个更大的排列，那么这个数组必须重排为字典序最小的排列（即，其元素按升序排列）。

例如，arr = [1,2,3] 的下一个排列是 [1,3,2] 。
类似地，arr = [2,3,1] 的下一个排列是 [3,1,2] 。
而 arr = [3,2,1] 的下一个排列是 [1,2,3] ，因为 [3,2,1] 不存在一个字典序更大的排列。
给你一个整数数组 nums ，找出 nums 的下一个排列。

必须 原地 修改，只允许使用额外常数空间。

这个题也是尝试了一些思路，但是最终因为达不到常数空间的要求而放弃了。最开始的思路是，将数组$a_i$的每个排列计算出对应的字典序，即$\displaystyle Order=\sum_{i=0}^{N-1}{idx[a_i]\times n^i}$，其中$idx$是该元素由小到大排序后形成的元素到下标的哈希表，$n$为将各个$a_i$在该级数中错开的数，通常取$n=\max(a_i)+1$。但是这个想法有些太过繁琐了，所以这里只好再次寻求答案的帮助：
这里直接贴上[灵茶山艾府大佬的解析](https://leetcode.cn/problems/next-permutation/solutions/3621022/jiao-ni-cong-ling-kai-shi-si-kao-zhe-ti-9qfrq/)思路。首先，大佬使用”寻找下一个整数”的例子做了与本题情境类似的引入：
> 整数 13999 的「下一个整数」是多少？
> 你可以轻松地回答出：14000。
> 怎么算的？拆解步骤：
> - 从右向左，找第一个小于 9 的数，即 3。
> - 把 3 加一，变成 4。
> - 把 4 右边的数都变成 0（最小的个位数）。
> 本题要计算「下一个排列」，这类似于计算「下一个不含重复数字的整数」，计算框架应该和「下一个整数」是类似的。

那么题中的下一个排列该怎么做？实际上类比上面的做法，就有些呼之欲出了。
首先，先由右向左遍历给出的数组，找到第一个“不够大”的数。这个“不够大”的数该如何定义？我们可以先从一个例子入手，考虑序列`[1, 4, 3, 5, 2]`，这个序列的”下一个排列“是`[1, 4, 5, 3, 2]`。我们从右往左遍历时发现，第四个数3可以”变得“更大。为什么可以变得更大？因为它的右边存在着比它还大的数。越大的数组越靠右，那么这个数组的字典序就倾向于越小。所以我们要设法把这个可以增大的数与在它右侧、且比它更大的数进行交换。基于这个例子，我们便可以体会出在各个“不够大”的数该如何定义了，它就是我们从右向左遍历时，遇到的第一个小于它右侧数字的数（更小的数字占据了更“重要”的位置，我们要设法把更“重要”的数字与其进行交换）。
与哪个数字进行交换呢？显然是在该数字（记为x）右侧的比它更大的数字。但如果x右侧有很多个数字比它更大呢？这时，为了尽可能得到其字典序恰好大于当前序列的“下一个序列”，我们需要将x与其右侧最小的大于它的数字进行交换，以保证字典序的增加尽量地小。我们将这个数字记为y。
现在我们交换x和y，事情就结束了吗？恐怕没有。我们还需要进行一些“后处理”。为了保证交换后的“影响”，也就是字典序的增加尽量地小，我们要保证这个交换后的数字，其后的子序列的字典序尽可能地小。所以，我们还要对交换后的y其后的子序列进行从小到大的排序。
当然需要注意的是，题目还规定了一种特殊情况。如果输入的序列已经是字典序最大的序列了，即按从大到小排序的了，这时我们要返回它的反转，即字典序最小的、从小到大排序的序列。字典序最大的序列在我们进行第一步寻找x的过程中，我们会发现我们从右向左遍历时找不到，遍历的下标会一直减小，直到变为-1。我们便可以利用这一点做特殊处理，如果发现下标变为-1，便直接对输入的数组做倒序。
这样，我们便形成了最终的算法。这个算法需要进行两次最差为$O(n)$级时间的遍历，然后进行一次最差为$O(n\log n)$级时间的排序。当然值得注意的是，这里的排序实际上可以更换为$O(n)$时间的倒序，因为交换完x和y之后，y右侧的子序列一定是从大到小排列的。这里的证明可以参见原题解。
大佬给出的程序更为精简，也更为巧妙：
```cpp
class Solution {
public:
    void nextPermutation(vector<int>& nums) {
        int n = nums.size();

        // 第一步：从右向左找到第一个小于右侧相邻数字的数 nums[i]
        int i = n - 2;
        while (i >= 0 && nums[i] >= nums[i + 1]) {
            i--;
        }

        // 如果找到了，进入第二步；否则跳过第二步，反转整个数组
        if (i >= 0) {
            // 第二步：从右向左找到 nums[i] 右边最小的大于 nums[i] 的数 nums[j]
            int j = n - 1;
            while (nums[j] <= nums[i]) {
                j--;
            }
            // 交换 nums[i] 和 nums[j]
            swap(nums[i], nums[j]);
        }

        // 第三步：反转 nums[i+1:]（如果上面跳过第二步，此时 i = -1）
        reverse(nums.begin() + i + 1, nums.end());
    }
};
```
该算法的时间复杂度为$O(3n)=O(n)$，空间复杂度为$O(1)$。
