一个机器人位于一个 m x n 网格的左上角 （起始点在下图中标记为 “Start” ）。

机器人每次只能向下或者向右移动一步。机器人试图达到网格的右下角（在下图中标记为 “Finish” ）。

问总共有多少条不同的路径？

---
这是一个非常经典的数学问题——求卡特兰数。我们建立与题中网格同形的动态规划数组`dp[i][j]`，表示到达此格的方法数。可以看出，除非此格位于边角，否则每个格子都可以由其左侧和上侧的格子的状态转移而来，于是可以确定`dp[i][j]=dp[i-1][j]+dp[i][j-1]`这个状态转移方程，而当该格子位于边角时，则根据情况，只令其由其左侧或上侧的状态转移而来。
基于此，很容易写出下面的程序：
```cpp
class Solution {
public:
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m, vector<int>(n));
        for (int i = 0; i < m; i++) {
            for (int j = 0; j < n; j++) {
                if (i == 0 && j == 0)
                    dp[i][j] = 1;
                else if (i == 0)
                    dp[i][j] = dp[i][j - 1];
                else if (j == 0)
                    dp[i][j] = dp[i - 1][j];
                else
                    dp[i][j] = dp[i][j - 1] + dp[i - 1][j];
            }
        }
        return dp[m - 1][n - 1];
    }
};
```
上面算法的时间复杂度为$O(mn)$，空间复杂度为$O(mn)$。
当然，实际上也很容易知道，这个问题其实可以转化为组合数学问题。到达右下角的格点，我们必须要向右走m-1次，向下走n-1次，因此我们的路径实际上是这(m+n-2)种走法的组合，总数也就是$C_{m+n-2}^{m-1}$。所以，我们也可以直接编程求解这个组合数。
下面直接贴上[官方题解](https://leetcode.cn/problems/unique-paths/solutions/514311/bu-tong-lu-jing-by-leetcode-solution-hzjf/)的程序：
```cpp
class Solution {
public:
    int uniquePaths(int m, int n) {
        long long ans = 1;
        for (int x = n, y = 1; y < m; ++x, ++y) {
            ans = ans * x / y;
        }
        return ans;
    }
};
```
这样就把时间复杂度降到了$O(m)$（实际上还可以再优化到$O(\min{m, n})$），空间复杂度降到了$O(1)$。
