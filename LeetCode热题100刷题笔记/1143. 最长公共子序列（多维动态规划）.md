给定两个字符串 text1 和 text2，返回这两个字符串的最长 公共子序列 的长度。如果不存在 公共子序列 ，返回 0 。

一个字符串的 子序列 是指这样一个新的字符串：它是由原字符串在不改变字符的相对顺序的情况下删除某些字符（也可以不删除任何字符）后组成的新字符串。

例如，"ace" 是 "abcde" 的子序列，但 "aec" 不是 "abcde" 的子序列。
两个字符串的 公共子序列 是这两个字符串所共同拥有的子序列。

---
本题是多维动态规划的经典题目。动态规划的特点是，要研究的问题与一个规模更小的子问题相关。初看，动态规划的思路与递归比较类似，但是细节上也有一些区别。这里我们直接看D老师的总结：
> 1. 核心思路的差异
	递归（Recursion）
		自顶向下分解问题：从原始问题出发，不断将问题分解为更小的子问题，直到达到递归基（最小可解问题）。
		无记忆性：每次遇到相同的子问题都会重新计算，导致大量重复计算（如斐波那契数列的递归实现）。
		代码直观：更贴近问题本身的数学定义，但效率可能极低。
	动态规划（Dynamic Programming, DP）
		自底向上或记忆化搜索：先解决最小的子问题，逐步构建到原问题的解；或通过缓存（记忆化）避免重复计算。
		有记忆性：通过表格或数组存储子问题的解，确保每个子问题只计算一次。
		优化重叠子问题：专为有重叠子问题和最优子结构的问题设计，显著提升效率。
> 2. 运行效率对比
	递归的缺陷
		指数级时间复杂度：例如斐波那契数列的递归实现时间复杂度为$O(2^n)$，大量重复计算。
		栈溢出风险：递归深度过大可能导致栈溢出（如计算 fib(100)）。
	动态规划的优势
		多项式时间复杂度：通过存储子问题解，将时间降至 $O(n)$（如斐波那契数列的DP实现）。
		空间换时间：牺牲额外空间（如数组或哈希表）存储中间结果，大幅提升效率。
>3. 适用场景
	递归适用场景
		无重叠子问题：例如树的遍历（前序/中序/后序），每个子问题独立。
		问题分治：如归并排序、快速排序，子问题无重复。
		代码简洁性优先：当效率不是首要考虑时。
	动态规划适用场景
		重叠子问题：如斐波那契数列、背包问题、最长公共子序列（LCS）。
		最优子结构：全局最优解可通过子问题最优解组合得到。
		需要高效解：对时间复杂度敏感的问题。

按我的理解，二者的一个核心差异是，递归擅长处理无重叠子问题的问题，而动态规划则擅长处理有重叠子问题的问题。因为递归本质上是一个树形结构，有重叠子问题意味着这至少是一个图结构，所以递归处理有重叠子问题的问题时，效率会低，反而动态规划问题更善于解决这类问题，因为它使用了额外空间存储子问题的解，很大程度上方便了图式的调用。
回到这个问题，我们首先要对题目要求的问题进行拆分和划归。假设两个字符串的长度分别是m和n，那么不难发现，长度分别为m和n的字符串的解与长度分别为m-1和n-1的字符串有关。
假设我们已知更小规模的同类问题的解，我们考虑长度分别为m和n的字符串时的情况，此时较之前的子问题，两个字符串各自增加了一个新的字符。我们分别将这两个字符分别记为`s1[m]`和`s2[n]`。如果这两个字符相同，那么很显然，这说明两个字符串的公共子序列长度增加了一位；而如果这两个字符不同，则说明这两个字符串的公共子序列长度较之前没有增加，也就是无论是从第一个字符串上增加`s1[m]`还是在第二个字符串上增加`s2[n]`，公共子序列长度都不会增加。
动态规划做法更倾向于我们写出严谨的数学方程来表示，这通常被称为“状态转移方程”。假若用$dp[m][n]$来表示本题所研究问题的解，那么不难按照上面的思路写出如下的状态转移方程：
$$
dp[m][n]=\left\{
\begin{aligned}
dp[m-1][n-1]+1&, \ \ s1[m]=s2[n]\\
\max\left({dp[m][n-1], dp[m-1][n]}\right)&,\ \ s1[m]\neq s2[n]
\end{aligned}
\right.$$
基于这个状态转移方程就很容易写出下面的程序。不过[官方题解](https://leetcode.cn/problems/longest-common-subsequence/solutions/696763/zui-chang-gong-gong-zi-xu-lie-by-leetcod-y7u0/)的做法比较严谨，还认真讨论了边界情况下的取值，即$m=0$和$n=0$时的情况，毫无疑问这两类边界情况下的子问题解都为0，即$dp[0][n]=0$和$dp[m][0]=0$。
```cpp
class Solution {
public:
    int longestCommonSubsequence(string text1, string text2) {
        int m = text1.length(), n = text2.length();
        vector<vector<int>> dp(m + 1, vector<int>(n + 1));
        for (int i = 1; i <= m; i++) {
            char c1 = text1.at(i - 1);
            for (int j = 1; j <= n; j++) {
                char c2 = text2.at(j - 1);
                if (c1 == c2) {
                    dp[i][j] = dp[i - 1][j - 1] + 1;
                } else {
                    dp[i][j] = max(dp[i - 1][j], dp[i][j - 1]);
                }
            }
        }
        return dp[m][n];
    }
};
```
不难发现，这样的问题时间复杂度和空间复杂度都达到了$O(mn)$的水平。
