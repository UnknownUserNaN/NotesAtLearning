给你一个单链表的头节点 head ，请你判断该链表是否为回文链表。如果是，返回 true ；否则，返回 false 。
进阶：你能否用 O(n) 时间复杂度和 O(1) 空间复杂度解决此题？

---
一个最简单粗暴的方法是，直接遍历整个链表，将其内容全部转存到数组中之后再在数组中使用双指针法。如果这个链表是双向链表，那这个问题会变得很简单，从两端开始遍历直到合拢即可。只可惜本题提供的是单向链表，并且不希望你用高于$O(1)$的空间解决此问题。
没什么好的想法，只能去求助[官方题解](https://leetcode.cn/problems/palindrome-linked-list/solutions/457059/hui-wen-lian-biao-by-leetcode-solution/)了。
官方题解的想法与最开始的数组转存法非常类似，但关键在于，单向链表没有像数组一样允许双向访问的机制。官方题解的思路是改造原链表，使其符合这一机制。也就是说，我们直接反转链表的后半部分，这样就可以实现头、尾双指针的遍历。
但是这样会修改原链表的结构。没关系，遍历完之后恢复回来就好了。
基于这个思路可以写出以下的代码：
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        // 寻找链表的结尾，并记录链表的长度
        ListNode* tail = head;
        int len = 1;
        while (tail->next != NULL) {
            tail = tail->next;
            len++;
        }

        // 确定需要反转的链表部分的结尾
        ListNode* mid = head;
        for (int i = 0; i < len / 2; i++) {
            mid = mid->next;
        }

        // 翻转后半部分的链表
        ListNode* p1 = mid;
        ListNode* p2 = NULL;
        while (p1 != NULL) {
            ListNode* temp = p1->next;
            p1->next = p2;
            p2 = p1;
            p1 = temp;
        }

        // 头尾同时遍历，对照检查
        p1 = head;
        p2 = tail;
        while (p2 != NULL) {
            if (p1->val != p2->val)
                return false;
            p1 = p1->next;
            p2 = p2->next;
        }
        return true; // 由于题目不检查链表结构，所以就不还原回原来的链表结构了
    }
};
```
但是这个程序写得相当不简洁，下面是官方题解中的写法：
```cpp
class Solution {
public:
    bool isPalindrome(ListNode* head) {
        if (head == nullptr) {
            return true;
        }

        // 找到前半部分链表的尾节点并反转后半部分链表
        ListNode* firstHalfEnd = endOfFirstHalf(head);
        ListNode* secondHalfStart = reverseList(firstHalfEnd->next);

        // 判断是否回文
        ListNode* p1 = head;
        ListNode* p2 = secondHalfStart;
        bool result = true;
        while (result && p2 != nullptr) {
            if (p1->val != p2->val) {
                result = false;
            }
            p1 = p1->next;
            p2 = p2->next;
        }        

        // 还原链表并返回结果
        firstHalfEnd->next = reverseList(secondHalfStart);
        return result;
    }

    ListNode* reverseList(ListNode* head) {
        ListNode* prev = nullptr;
        ListNode* curr = head;
        while (curr != nullptr) {
            ListNode* nextTemp = curr->next;
            curr->next = prev;
            prev = curr;
            curr = nextTemp;
        }
        return prev;
    }

    ListNode* endOfFirstHalf(ListNode* head) {
        ListNode* fast = head;
        ListNode* slow = head;
        while (fast->next != nullptr && fast->next->next != nullptr) {
            fast = fast->next->next;
            slow = slow->next;
        }
        return slow;
    }
};
```
然而，虽然该方法能将额外空间降低到$O(1)$，但是官方题解也指出了此种方法的弊端：

>该方法虽然可以将空间复杂度降到 O(1)，但是在并发环境下，该方法也有缺点。在并发环境下，函数运行时需要锁定其他线程或进程对链表的访问，因为在函数执行过程中链表会被修改。
