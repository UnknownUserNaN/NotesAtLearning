给定一个长度为 `n` 的 **0 索引**整数数组 `nums`。初始位置为 `nums[0]`。

每个元素 `nums[i]` 表示从索引 `i` 向后跳转的最大长度。换句话说，如果你在 `nums[i]` 处，你可以跳转到任意 `nums[i + j]` 处:

*   `0 <= j <= nums[i]` 
*   `i + j < n`

返回到达 `nums[n - 1]` 的最小跳跃次数。生成的测试用例可以到达 `nums[n - 1]`。

***

这题可以沿用45题的思路，考虑正向遍历的算法。首先设置两个指针，以确定这一步最远和最近的范围。最开始这个范围肯定是限定在最开始的第一步上，然后根据第一步的限定范围，推出第二步的范围……依此类推，直至发现最远的范围可以覆盖到数组末端为止。根据上面的思路，可以写出下面的程序：

```cpp
class Solution {
public:
    int jump(vector<int>& nums) {
        int step = 0;
        int left = 0, next_left = 0;
        int right = 0, next_right = 0;

        while (right < nums.size() - 1) {
            // 确定每一步的延伸范围
            for (int i = left; i <= right; i++) {
                next_right = max(nums[i] + i, next_right);
                if (nums[i] != 0)
                    next_left = min(1 + i, next_left); // 最少走1步，防止无效遍历
                else
                    next_left = min(i, next_left);
            }

            // 确定下一步的浮动范围
            left = next_left;
            right = next_right;
            step++;
        }

        return step;
    }
};
```

虽然这次提交直接AC了，但很遗憾的是，这个程序并不是最简的状态，尤其是时间复杂度上实际上是略高于`$O(n)$`的，因为实际上整个程序运行的过程中，某些元素并不止被遍历了一次。因此还需要进一步地优化。

为什么我们会有多余的遍历呢？这主要是因为我们每一次的循环中，除了能到达的最远的位置，我们也没有“放弃”不是最远位置的点。实际上我们只需要注意到，假设我们在某一步的遍历过程中找到了“最远”的位置，那么只要这一步里能到达的“最远”位置上的数字不是0，就说明在遍历到达这一最远位置之前，在这一步的范围内都不会到达更远的地方——除非此前的“最远”的地方并不是真正的最远的地方。

基于这一点观察，我们便可以改写上面的遍历过程。我们可以从左往右遍历数组，并根据遍历到的内容不断维护能到达的最远地方。如果遍历的位置越过或者达到了此前最远的地方了，那么说明此时进入了新的一步，于是步数可以相应加一，直至可到达的最远位置大于一为止。

下面祭出官方题解中的代码：

```cpp
class Solution {
public:
    int jump(vector<int>& nums) {
        int maxPos = 0, n = nums.size(), end = 0, step = 0;
        for (int i = 0; i < n - 1; ++i) {
            if (maxPos >= i) {
                maxPos = max(maxPos, i + nums[i]);
                if (i == end) {
                    end = maxPos;
                    ++step;
                }
            }
        }
        return step;
    }
};
```
