给定一个包含 n + 1 个整数的数组 nums ，其数字都在 [1, n] 范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 nums 只有 一个重复的整数 ，返回 这个重复的数 。

你设计的解决方案必须 不修改 数组 nums 且只用常量级 O(1) 的额外空间。

提示：
- nums 中 只有一个整数 出现 两次或多次 ，其余整数均只出现 一次

进阶：
- 如何证明 nums 中至少存在一个重复的数字?
- 你可以设计一个线性级时间复杂度 O(n) 的解决方案吗？

---
本题可以从“136. 只出现一次的数字”中获取灵感。异或（xor）运算有一个非常重要的性质，就是它具有交换律，同时两个相同的元素做异或运算必然得0，所以在此题中对数组中所有数求异或和可以“自动”求得重复的数字，因为其他出现两次的数字会在异或运算中自动“消去”。
这启发我们试图利用异或运算来完成此题。但事实上，根本找不到一种合适的二进制运算来完成本题，因为二进制运算往往不具有“累积性”。比如我们试着对数组中每个数求前缀异或和，然后对比它与同位置的前缀和是否相同。在想象中，异或和具有相同“消去性”，因此在我们第二次遇到重复的数字时，前缀异或和应当是小于前缀和的。但实际上，异或和因为不具有进位性质，因此实际上当我们遇到了不同的数字时，前缀异或和就已经与前缀和不同了。
还是来看看[官方题解](https://leetcode.cn/problems/find-the-duplicate-number/solutions/261119/xun-zhao-zhong-fu-shu-by-leetcode-solution/)怎么做的吧！
官方题解给出了三种做法，首先是最经典的二分查找法。显然不能直接对数组应用二分查找，因为原数组不具有单调性。于是题解在这里设计了一个辅助数组`cnt[i]`，遍历`nums`数组，记录数组中小于等于当前遍历元素的数有多少个。很显然，这个数组是单调递增的。并且，由于除了要查找的元素它出现了多次以外，该数组其余元素都只出现了一次（甚至零次），因此正常情况下，`cnt`会随着`i`增加而逐步加1（或者0），直到遇到重复的元素，会突然增加大于1的量，随后又是稳步加一。由此，我们可以确定，在目标元素`target`之前的`cnt`中的元素，通常有`cnt[i]<=i`；在其后的元素，有`cnt[i]>i`。基于这个性质，我们可以进行二分查找。
当然，上面的做法需要使用数组`cnt`，这需要使用$O(n)$级的空间，不符合题意。因此实际实现中，我们需要将`cnt`设定为一个变量，需要的时候随时根据累加求取其在某个下标处的值即可，不过这样做的代价就是整个算法的时间复杂度从$O(n)$上升到$O(n\log n)$。
```cpp
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int n = nums.size();
        int l = 1, r = n - 1, ans = -1;
        while (l <= r) {
            int mid = (l + r) >> 1;
            int cnt = 0;
            for (int i = 0; i < n; ++i) {
                cnt += nums[i] <= mid;
            }
            if (cnt <= mid) {
                l = mid + 1;
            } else {
                r = mid - 1;
                ans = mid;
            }
        }
        return ans;
    }
};
```
然后是第二种做法，基于二进制的位运算。我们知道，输入的数组中只会出现$[1,n]$范围内的数，我们考虑这些数的二进制表示，分别记录它们在各个二进制位上的为“1”的数目，这个数组记为`y[j]`。然后考虑整个输入数组，由于其中存在重复的数字，因此其在二进制数上的“统计”（记为`x[i]`)显然不同于`y[i]`。这里实际上可以证明，有某些`i`满足`x[i]>y[i]`。我们收集这些满足`x[i]>y[i]`的`i`，然后将这些`i`对应的二进制位置为1，组成一个新的数。这个数就是重复的数字。
其实这个思路有点绕。还是直接贴上官方的代码吧：
```cpp
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int n = nums.size(), ans = 0;
        // 确定二进制下最高位是多少
        int bit_max = 31;
        while (!((n - 1) >> bit_max)) {
            bit_max -= 1;
        }
        for (int bit = 0; bit <= bit_max; ++bit) {
            int x = 0, y = 0;
            for (int i = 0; i < n; ++i) {
                if (nums[i] & (1 << bit)) {
                    x += 1;
                }
                if (i >= 1 && (i & (1 << bit))) {
                    y += 1;
                }
            }
            if (x > y) {
                ans |= 1 << bit;
            }
        }
        return ans;
    }
};
```
通过上面的算法能看出来，上面的做法的时间复杂度为$O(n\log n)$，空间复杂度为$O(1)$，不满足进阶中的复杂度需求。
那么有没有一种时间复杂度能达到$O(n)$的算法呢？有的。官方题解最终给出了一种“将上面的数组视为链表“的方法。具体的看待方法是，对每个元素`nums[i]`，其元素值表示其指向的下一个元素的下标。由于`nums`中只存在$[1, n]$上的数，因此这里我们完全不用担心这样构造链表的方法会”越界“。
现在我们考虑，正常情况之下，由于绝大部分的数字都会在数组中出现一次，因此数组的各个下标与数组元素之间可以形成一一映射的关系，因此这些元素组成的链表显然是没有环的。但是，如果这里面出现了重复的数字，也就是存在两个下标，`i`和`j`，有`nums[i]==nums[j]`，这样显然会导致一个这两个“链表结点”指向同一个结点，这就是出现了“环”的地方。Floyd算法，也就是“快慢指针法”，恰好擅长找到这个“环”出现的地方。
这里算法的思路可以联系“142. 环形链表II”来理解。当慢指针和快指针相遇时，可以判断该链表存在环。但我们还需要找到环所在的位置，所以需要将慢指针从初始处释放，快指针继续“行走”，但改为一次走一步，最终两者再次相遇的地方就是链表成环的位置。
写出的程序如下：
```cpp
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int slow = 0, fast = 0;
        do {
            slow = nums[slow]; // 慢指针走一步
            fast = nums[nums[fast]]; // 快指针走两步
        } while (slow != fast); 
        slow = 0;
        while (slow != fast) {
            slow = nums[slow];
            fast = nums[fast];
        }
        return slow;
    }
};
```
这是一个非常精妙的算法。它可以用常数级的空间和$O(n)$级的时间来解决这个特殊数组的问题。
