1. **JDBC：**（Java DataBase Connectivity），就是使用Java语言操作关系型数据库的一套API。 
	但是使用JDBC来操作数据库，会比较繁琐，所以现在在企业项目开发中，一般都会使用基于JDBC的封装的高级框架，比如：Mybatis、MybatisPlus、Hibernate、SpringDataJPA。

2. JDBC执行DML和DQL的示例程序（直接运行在Junit的测试环境中）：
	```java
	public class JDBCTest {  
    @Test  
    // 执行DML更新语句  
    public void testUpdate() throws ClassNotFoundException, SQLException {  
        // 1. 注册驱动  
//        Class.forName("com.mysql.cj.jdbc.Driver"); // 这一行动态引入实际上没必要执行了，因为新版本的驱动会自动注册  
  
        // 2. 获取数据库连接  
        String url = "jdbc:mysql://localhost:3306/web01";  
        String username = "root";  
        String password = "";  
        Connection connection = DriverManager.getConnection(url, username, password);  
  
        // 3. 获取SQL语句执行对象  
        Statement statement = connection.createStatement();  
  
        // 4. 执行SQL语句  
        int i = statement.executeUpdate("update user set age = 25 where id = 1");  
        System.out.println("影响的SQL记录数为"+i);  
  
        // 5. 释放资源  
        statement.close();  
        connection.close();  
    }  
  
    @Test  
    // 执行DQL查询语句  
    public void testSelect() throws ClassNotFoundException, SQLException {  
        // 1. 注册驱动  
//        Class.forName("com.mysql.cj.jdbc.Driver");  
  
        // 2. 获取数据库连接  
        String url = "jdbc:mysql://localhost:3306/web01";  
        String username = "root";  
        String password = "";  
        Connection connection = DriverManager.getConnection(url, username, password);  
  
        // 3. 获取SQL语句执行对象  
        Statement statement = connection.createStatement();  
  
        // 4. 执行SQL语句  
        String sql = "select id, username, password, name, age from user where username = ? and password = ?"; // 这里的问号是占位符，这整个语句是一个预编译SQL语句  
        PreparedStatement preparedStatement = connection.prepareStatement(sql); // 将上述字符串设定为预编译SQL语句  
        preparedStatement.setString(1, "lvbu"); // 设置第一个占位符的值  
        preparedStatement.setString(2, "123456"); // 设置第二个占位符的值  
        // 预编译SQL语句更加安全，因为预编译SQL语句在编译的时候已经将SQL语句进行解析，将SQL语句中的占位符替换为真正的值，从而避免了SQL注入攻击  
  
        // 5. 处理结果集合  
        ResultSet results = preparedStatement.executeQuery();  
        while (results.next()) { // 调用next()方法，逐个遍历查询结果表中的每条数据，这里是一个非常经典的写法  
            int id = results.getInt(1);  
            String user = results.getString(2);  
            String pwd = results.getString(3);  
            String name = results.getString(4);  
            int age = results.getInt(5);  
            System.out.println(id+" "+user+" "+pwd+" "+name+" "+age);  
        }  
  
        // 6. 关闭资源  
        // 这里的写法其实不够安全，因为一旦上面的SQL语句执行失败，那么下面的资源就会无法释放，所以这里应该使用try-catch-finally  
        preparedStatement.close();  
        connection.close();  
        results.close();  
    }  
	}
	```
	- 预编译SQL语句的好处：
		1. 可以防止SQL注入，更加安全
		2. 方便了SQL执行器的优化过程，执行效率更高
