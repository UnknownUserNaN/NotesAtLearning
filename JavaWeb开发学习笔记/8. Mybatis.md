1. MyBatis是一款优秀的 **持久层** **框架**，用于简化JDBC的开发。
	- 持久层：指的是就是数据访问层(DAO)，是用来操作数据库的。
	- 框架：是一个半成品软件，是一套可重用的、通用的、软件基础代码模型。在框架的基础上进行软件开发更加高效、规范、通用、可拓展。
2. Mybatis+SpringBoot的简单程序创建步骤：
	1. 准备工作：
		1. 创建SpringBoot工程，引入Mybatis的相关依赖
		2. 准备MySQL数据库，以及Java中的实体类
			实体类编写示例：
			```java
			@Data  
			@NoArgsConstructor  
			@AllArgsConstructor  // 这里调用了Lombok中的注解来快速生成实体类的有参和无参构造函数
			public class User {  
			    private Integer id;  
			    private String username;  
			    private String password;  
			    private String name;  
			    private Integer age;  
			}
			```
		3. 在SpringBoot项目的application.properties中编写数据库的连接信息
			数据库连接信息配置示例：
			```text
			# 配置数据库的连接信息  
			spring.datasource.url=jdbc:mysql://localhost:3306/web01  
			spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver  
			spring.datasource.username=root  
			spring.datasource.password=
			```
	2. 使用Mybatis的`@Mapper`注解，编写处理MySQL的接口，Mybatis会自动创建该接口的实现类（处理类），并将其动态插入到Spring的IOC容器中，成为Bean对象，业务代码调用时只需要通过`@Autowired`注解引入就可以了。
		接口编写示例：
		```java
			@Mapper // 这是Mybatis提供的注解，程序在运行时，Mybatis会自动地为该接口生成代理对象（也就是之前的Impl类），并注入到Spring的IOC容器中，成为bean对象  
			public interface UserMapper {  
			  
			    @Select("select * from user") // Mybatis提供的注解，表示该方法会执行SQL查询语句  
			    public List<User> findAll();  
			  
			}
		```
		业务代码编写示例：
		```java
		@SpringBootTest // 这是SpringBoot提供的单元测试注解，需要与引导包在同一个子包下  
		class Lesson05MybatisQuickStartApplicationTests {  
		  
		    @Autowired // 既然之前的Mapper已经自动加载到Spring的IOC容器中了，这里便可以直接将其动态加载进来  
		    private UserMapper userMapper;  
		      
		    @Test  
		    public void testFindAll() {  
		        List<User> users = userMapper.findAll();  
		        users.forEach(System.out::println); // 将返回结果中的每一个都调用该方法打印出来  
		        // 这里的双冒号是Java8提供的“方法引用”的语法，主要用于简化Lambda表达式的书写  
		        // 上面的代码等价于：users.forEach(user -> System.out.println(user));  
		    }  
		}
		```
1. JDBC VS Mybatis
	- JDBC程序的**缺点**：
		- url、username、password 等相关参数全部硬编码在java代码中。
		- 查询结果的解析、封装比较繁琐。
		- 每一次操作数据库之前，先获取连接，操作完毕之后，关闭连接。 频繁的获取连接、释放连接造成资源浪费。
	- Mybatis是如何解决这些问题的：
		- 数据库连接四要素(驱动、链接、用户名、密码)，都配置在springboot默认的配置文件 application.properties中
		- 查询结果的解析及封装，由mybatis自动完成映射封装，我们无需关注
		- 在mybatis中使用了数据库连接池技术，从而避免了频繁的创建连接、销毁连接而带来的资源浪费。
	- 对于Mybatis来说，我们在开发持久层程序操作数据库时，需要重点关注以下两个方面：
		- 数据库的连接配置
		- Mapper接口的编写
2. Mybatis中采用的数据库连接池：
	- 特点：
		- 数据库连接池是个容器，负责分配、管理数据库连接(Connection)
			- 程序在启动时，会在数据库连接池(容器)中，创建一定数量的Connection对象
		- 允许应用程序重复使用一个现有的数据库连接，而不是再重新建立一个
			- 客户端在执行SQL时，先从连接池中获取一个Connection对象，然后在执行SQL语句，SQL语句执行完之后，释放Connection时就会把Connection对象归还给连接池（Connection对象可以复用）
		- 释放空闲时间超过最大空闲时间的连接，来避免因为没有释放连接而引起的数据库连接遗漏
			- 客户端获取到Connection对象了，但是Connection对象并没有去访问数据库(处于空闲)，数据库连接池发现Connection对象的空闲时间 > 连接池中预设的最大空闲时间，此时数据库连接池就会自动释放掉这个连接对象
		- 数据库连接池的好处：
			- 资源重用（数据库连接可以复用）
			- 提升系统响应速度
			- 避免数据库连接遗漏（空闲过久的连接会回收）
	- 实现数据库连接池：
		- 官方(sun)提供了数据库连接池标准（javax.sql.DataSource接口）
		- 第三方组织必须按照DataSource接口实现，现在也更多使用第三方的数据库连接池。现在使用更多的是**Hikari(springboot默认)、Druid （阿里巴巴开发的开源项目，性能更优越）**
3. 基于MyBatis的增删改查操作（如何编写对应的Mapper）：
	1. 删除操作（Delete）：
		Mapper编写示例：
		```Java
		@Delete("delete from user where id = #{id}")  // 这里的#{id}是Mybatis提供的占位符，会自动将函数中的参数拼合到预编译SQL语句中
		public void deleteById(Integer id);
		```
		- 上面的函数可以有Integer类型的返回值，表示DML语句执行完毕影响的记录数（实际删除的记录数）。
		- Mybatis提供了两种占位符：一个是 `#{...}`，另一个是 `${...}`，区别如下：
		
		| 符号   | 说明                           | 场景            | 优缺点         |
		| ---- | ---------------------------- | ------------- | ----------- |
		| #{…} | 占位符。执行时，会将#{…}替换为?，生成预编译SQL  | 参数值传递         | 安全、性能高 （推荐） |
		| ${…} | 拼接符。直接将参数拼接在SQL语句中，存在SQL注入问题 | 表名、字段名动态设置时使用 | 不安全、性能低     |
	1. 新增操作（Insert）：
		Mapper编写示例：
		```java
		@Insert("insert into user(username,password,name,age) values(#{username},#{password},#{name},#{age})")  
		public void insert(User user); // 当SQL语句中的参数比较多时，Mybatis支持将函数参数的属性值映射到SQL语句中
		```
	2. 修改操作（Update）：
		Mapper编写示例：
		```Java
		@Update("update user set username = #{username},password = #{password},name = #{name},age = #{age} where id = #{id}") 
		public void update(User user);
		```
	3. 查询操作（Select）：
		Mapper编写示例：
		```java
		@Select("select * from user where username = #{username} and password = #{password}")  
		public User findByUsernameAndPassword(@Param("username") String username, @Param("password") String password);  
		// 虽然有多个参数的时候MyBatis可以根据参数顺序自动推断，但为了代码可读性和维护性，建议在多参数方法中始终使用@Param注解。  
		// 这个问题的根源是因为Java编译完程序之后，会自动去掉所有参数形参的名字，只保留数据类型，因此MyBatis无法通过形参名来确定参数的位置。
		```
		不过，基于官方骨架创建的springboot项目中，接口编译时会保留方法形参名，@Param注解可以省略 (#{形参名})。
1. MyBatis的另一种开发方式——XML映射配置：
	MyBatis有两种开发方式，一是基于前面所提到的注解形式，另一种则是基于XML的映射配置。
	1. XML映射配置文件规范：
		1. XML映射文件的名称与Mapper接口名称一致，并且将XML映射文件和Mapper接口放置在相同包下（同包同名）
			- 在Maven项目中，对应接口通常存放在java目录下，而XML映射文件通常要放在resources目录下，因此此时要格外注意两者包名的对应
			- 不过，Mybatis支持在properties设定文件中指定XML映射文件的位置，这样就不必严格保证XML配置文件必须同包同名了
				```text
				mybatis.mapper-locations=classpath:mapper/*.xml
				# 这里表示XML配置文件存放在mapper这个包下，然后统一查找对应名字的xml文件
				```
		1. XML映射文件的namespace属性为Mapper接口全限定名一致
		2. XML映射文件中sql语句的id与Mapper接口中的方法名一致，并保持返回类型一致
		![[Pasted image 20251204000339.png]]
	2. XML映射配置文件编写示例：
		```xml
		<?xml version="1.0" encoding="UTF-8" ?>
		<!DOCTYPE mapper
		        PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN"
		        "https://mybatis.org/dtd/mybatis-3-mapper.dtd">
		<mapper namespace="com.itheima.mapper.EmpMapper">
		
		    <!--无参Mapper的声明示例-->
		    <select id="findAll" resultType="com.itheima.pojo.User">
		        select * from user <!-- 这里放要执行的SQL语句 -->
		    </select>
		    
		    <!--有参Mapper的声明示例-->
		     <select id="findByUsernameAndPassword" resultType="org.example.lesson05_mybatisquickstart.pojo.User">
		        select * from user 
		        where username = #{username} and password = #{password}
		    </select>
		    
		</mapper>
		```
	1. 使用Mybatis的注解，主要是来完成一些简单的增删改查功能。**如果需要实现复杂的SQL功能，建议使用XML来配置映射语句。**

		![[Pasted image 20251204002154.png]]
		- yml配置文件的基本语法：
			- 大小写敏感
			- 数值前边必须有空格，作为分隔符
			- 使用缩进表示层级关系，缩进时，不允许使用Tab键，只能用空格（idea中会自动将Tab转换为空格）
			- 缩进的空格数目不重要，只要相同层级的元素左侧对齐即可
			- `#`表示注释，从这个字符一直到行尾，都会被解析器忽略
		- yml文件中常见的数据格式：
			- 对象/Map集合
			```YAML
			user:
			  name: zhangsan
			  age: 18
			  password: 123456
			```
			- 数组/List/Set集合
			```YAML
			hobby: 
			  - java
			  - game
			  - sport
			```
		- 在yml格式的配置文件中，如果配置项的值是以 0 开头的，值需要使用 '' 引起来，因为以0开头在yml中表示8进制的数据。
